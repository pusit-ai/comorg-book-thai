# 5.1 SEMICONDUCTOR MAIN MEMORY  

## 1. Organization  
หน่วยความจำหลักในปัจจุบันใช้ Semiconductor โดยมีหน่วยพื้นฐานคือ Memory Cell  
Memory cell สามารถเขียน (Write) และอ่าน (Read) ได้ โดยเก็บข้อมูลเป็น 0 และ 1  

---

## 2. DRAM and SRAM  
Semiconductor Memory แบ่งได้ 2 ประเภทหลัก  

### 2.1 RAM (Volatile)  
สามารถอ่านและเขียนได้ แต่ข้อมูลจะหายเมื่อไฟดับ  

DRAM เก็บข้อมูลในรูปประจุไฟฟ้า ต้อง Refresh ตลอดเวลา  
ราคาถูก ความจุสูง → ใช้เป็น Main Memory  

SRAM ใช้วงจร Flip-Flop ไม่ต้อง Refresh  
เร็วกว่าแต่ราคาแพง → ใช้เป็น Cache  

### 2.2 ROM (Non-Volatile)  
เก็บข้อมูลถาวร เน้นอ่าน ไม่เหมาะกับการเขียน  

---

## 3. Types of ROM  
ROM : เขียนมาจากโรงงาน แก้ไม่ได้  
PROM : เขียนได้ครั้งเดียว  
EPROM : ลบด้วยแสง UV และเขียนใหม่ได้  
EEPROM : ลบ/เขียนใหม่ด้วยไฟฟ้า (ระดับ Byte)  
Flash Memory : ลบเร็วกว่า และเก็บข้อมูลหนาแน่นกว่า  

---

## 4. Chip Logic (Organization & Chip Logic)  
การจัดเรียง : Memory Cell เรียงเป็น Row/Column (1 cell = 1 bit)  
การเข้าถึงข้อมูล : ใช้ Address และสัญญาณควบคุม เช่น WE, OE  
DRAM ใช้ RAS และ CAS เพื่อเลือก Row และ Column  
DRAM ต้อง Refresh ตลอดเวลา  

---

## 5. The Trade-off  
ไม่มีหน่วยความจำที่ดีที่สุด ต้องแลกกันระหว่าง Speed, Price, Capacity  

---

# บทที่ 5.2 เรื่อง Error Correction (การแก้ไขข้อผิดพลาดในหน่วยความจำ) 
ได้อธิบายถึงระบบหน่วยความจำเซมิคอนดักเตอร์ที่มักจะพบข้อผิดพลาด โดยสามารถแบ่งชนิดของข้อผิดพลาดออกเป็น 2 ประเภทหลัก ได้แก่:

## 1. Hard failures (ข้อผิดพลาดแบบถาวร): 
เป็นความบกพร่องทางกายภาพที่ทำให้เซลล์หน่วยความจำไม่สามารถจัดเก็บข้อมูลได้อย่างเชื่อถือได้ และจะค้างอยู่ที่ค่า 0 หรือ 1 เสมอ หรือสลับค่าไปมาแบบควบคุมไม่ได้ ซึ่งอาจเกิดจากการใช้งานในสภาพแวดล้อมที่รุนแรง ความบกพร่องจากกระบวนการผลิต หรือการเสื่อมสภาพจากการใช้งาน

---

## 2. Soft errors (ข้อผิดพลาดแบบชั่วคราว): 
เป็นเหตุการณ์สุ่มที่ไม่ทำลายเซลล์หน่วยความจำ แต่ทำให้ค่าข้อมูลในหน่วยความจำเปลี่ยนไป สาเหตุมักเกิดจากปัญหาของแหล่งจ่ายไฟ หรืออนุภาคแอลฟา (Alpha particles) จากการสลายตัวของสารกัมมันตรังสีที่มีอยู่ในวัสดุเกือบทุกชนิด

**กระบวนการตรวจจับและแก้ไขข้อผิดพลาด (Error-Correcting Codes):** เพื่อจัดการกับปัญหาดังกล่าว ระบบหน่วยความจำหลักสมัยใหม่จึงมีลอจิกสำหรับคอยตรวจจับและแก้ไขข้อมูลโดยมีกระบวนการทำงานคือ:

* **ตอนเขียนข้อมูล:** ข้อมูลขนาด M บิต จะถูกนำไปคำนวณผ่านฟังก์ชัน เพื่อสร้าง "รหัส (Code)" ตรวจสอบขนาด K บิต ดังนั้นขนาดของข้อมูลทั้งหมดที่ถูกจัดเก็บจริงลงในหน่วยความจำคือ M+K บิต
* **ตอนอ่านข้อมูล:** ระบบจะสร้างรหัส K บิตชุดใหม่ขึ้นมาจากข้อมูลที่อ่าน แล้วนำไปเปรียบเทียบกับรหัส K บิตเดิมที่เคยดึงออกมา

ซึ่งผลลัพธ์ที่เป็นไปได้มี 3 กรณี:
1. ไม่พบข้อผิดพลาด ระบบจะส่งข้อมูลออกไปใช้งานได้เลย
2. พบข้อผิดพลาดและสามารถแก้ไขได้ ข้อมูลและรหัสจะถูกส่งเข้าไปในตัวแก้ไข (Corrector) เพื่อปรับข้อมูล M บิตให้ถูกต้องก่อนส่งออกไป
3. พบข้อผิดพลาดแต่ไม่สามารถแก้ไขได้ ระบบจะทำการรายงานความผิดปกตินี้

**รหัสแฮมมิง (Hamming Code):** หนึ่งในวิธีแก้ไขข้อผิดพลาดที่เข้าใจง่ายที่สุดคิดค้นโดย Richard Hamming ระบบนี้จะเปรียบเทียบรหัสเดิมกับรหัสใหม่แบบบิตต่อบิตด้วยการทำ **Exclusive-OR (XOR)** ซึ่งผลลัพธ์ที่ได้เรียกว่า **Syndrome word** ขนาด K บิต

* จำนวนของบิตตรวจสอบ (K) จะต้องสอดคล้องกับอสมการ: $2^K - 1 \ge M + K$
* ตัวอย่างเช่น หากข้อมูลมีขนาด 8 บิต (M=8) จะต้องใช้บิตตรวจสอบ 4 บิต (K=4) เป็นอย่างน้อย เนื่องจาก $2^4 - 1 \ge 8 + 4$ (หรือ $15 \ge 12$)

**ลักษณะการอ่านค่า Syndrome Word:**
* **ถ้าค่าเป็น 0 ทั้งหมด:** หมายถึงไม่พบข้อผิดพลาด
* **ถ้ามีเลข 1 เพียงบิตเดียว:** หมายความว่ามีข้อผิดพลาดเกิดขึ้นที่ตัวบิตตรวจสอบ (Check bits) เอง ซึ่งระบบไม่ต้องทำการแก้ไขข้อมูล
* **ถ้ามีเลข 1 มากกว่าหนึ่งบิต:** ค่าตัวเลขของ Syndrome จะระบุตำแหน่งของบิตข้อมูลที่ผิดพลาด เพื่อให้ระบบทำการกลับค่า (Invert) บิตในตำแหน่งนั้นให้กลับมาถูกต้อง โดยบิตตรวจสอบ (Check bits) จะถูกกำหนดให้อยู่ในตำแหน่งที่เป็นเลขยกกำลังของ 2 เสมอ (เช่น ตำแหน่งที่ 1, 2, 4, และ 8)

**ระบบ SEC-DED (Single-Error-Correcting, Double-Error-Detecting):** โดยทั่วไปแล้ว ในระบบหน่วยความจำมักจะนิยมติดตั้งรหัสที่เรียกว่า **SEC-DED** ซึ่งมีความสามารถในการ **"แก้ไขข้อผิดพลาดได้ 1 บิต และตรวจจับข้อผิดพลาดที่เกิดพร้อมกัน 2 บิตได้"**

เพื่อป้องกันการที่ระบบวิเคราะห์ผิดพลาดเมื่อมีข้อมูลเสีย 2 บิตพร้อมกัน SEC-DED จึงต้องเพิ่มความปลอดภัยโดยการใส่บิตตรวจสอบ (Parity bit) เข้าไปอีก 1 บิต ซึ่งการเพิ่มระบบแก้ไขข้อผิดพลาดเหล่านี้ทำให้หน่วยความจำมีความน่าเชื่อถือสูง แต่ก็ต้องแลกมากับความจุที่ต้องเตรียมเผื่อไว้จัดเก็บรหัส (Overhead) ตัวอย่างเช่น ในระบบ DRAM สมัยใหม่อาจต้องใช้พื้นที่สูญเปล่าเพิ่มขึ้นถึง 7% ถึง 20% เพื่อจัดเก็บรหัสป้องกันเหล่านี้

SRAM : เร็วมาก แต่แพง → Cache  
DRAM : ช้ากว่า แต่ถูกและจุเยอะ → Main Memory  
ROM : ใช้เก็บข้อมูลถาวร

# 5.3 DDR DRAM

## 1. Synchronous DRAM (SDRAM)
* **เป็นการพัฒนาจาก DRAM แบบเดิม (Asynchronous) ให้ทำงานได้มีประสิทธิภาพมากขึ้น**:
การทำงานที่สอดประสานกัน: SDRAM จะทำงานสัมพันธ์กับ สัญญาณนาฬิกา (System Clock) ทำให้การแลกเปลี่ยนข้อมูลระหว่าง Processor กับ Memory เกิดขึ้นได้อย่างแม่นยำและรวดเร็ว

* **ลดสถานะรอคอย (Wait States)**: เนื่องจาก Processor รู้ว่าข้อมูลจะพร้อมเมื่อไหร่จากจังหวะสัญญาณนาฬิกา ทำให้ไม่ต้องหยุดรอการทำงานของหน่วยความจำนานเหมือนแบบเก่า

* **Burst Mode**: ความสามารถในการส่งข้อมูลออกมาเป็นชุดๆ อย่างต่อเนื่องหลังจากระบุตำแหน่งข้อมูลตัวแรก ช่วยเพิ่มความเร็วในการอ่านข้อมูลจำนวนมาก

---

## 2. Double Data Rate SDRAM (DDR SDRAM)
เทคโนโลยีนี้คือหัวใจสำคัญของแรมในปัจจุบัน ซึ่งพัฒนาต่อยอดมาจาก SDRAM:

* ความเร็ว 2 เท่า (Double Data Rate): * โดยปกติ SDRAM จะส่งข้อมูลเฉพาะช่วง "ขาขึ้น" (Rising edge) ของสัญญาณนาฬิกา

* แต่ DDR จะส่งข้อมูลทั้งช่วง "ขาขึ้น" และ "ขาลง" (Falling edge) ของสัญญาณนาฬิกาใน 1 รอบ ทำให้รับส่งข้อมูลได้เร็วขึ้นเป็น 2 เท่าโดยไม่ต้องเพิ่มความเร็วสัญญาณนาฬิกา

วิวัฒนาการของ DDR (Prefetch Buffer):แต่ละรุ่นมีการใช้เทคนิค Prefetch หรือการดึงข้อมูลล่วงหน้ามาเตรียมไว้ในบัฟเฟอร์ก่อนส่งจริง:

* **DDR1**: Prefetch 2 บิต

* **DDR2**: Prefetch 4 บิต (เพิ่มความเร็วการรับส่งภายใน)

* **DDR3 / DDR4**: Prefetch 8 บิต (ช่วยให้ความเร็วบัสสูงขึ้นไปอีก)

การประหยัดพลังงาน: ยิ่งรุ่นใหม่ขึ้น (DDR3 -> DDR4) จะมีการใช้แรงดันไฟฟ้าที่น้อยลง เพื่อลดความร้อนและประหยัดพลังงานในภาพรวม

# 5.4 หน่วยความจำแบบแฟลช (Flash Memory)

หน่วยความจำแบบแฟลช (Flash Memory) เป็นหนึ่งในนวัตกรรมสำคัญของหน่วยความจำสารกึ่งตัวนำ (Semiconductor Memory) ที่ได้รับการพัฒนาขึ้นในช่วงกลางทศวรรษที่ 1980 โดยมีบทบาทสำคัญทั้งในฐานะหน่วยความจำภายในเครื่องและหน่วยความจำภายนอก ความโดดเด่นของเทคโนโลยีนี้คือการเป็นจุดกึ่งกลางที่ลงตัวระหว่างหน่วยความจำประเภท EPROM และ EEPROM ทั้งในด้านต้นทุนการผลิตและประสิทธิภาพการใช้งาน

### ลักษณะการทำงานเบื้องต้น
* **เทคโนโลยีการลบข้อมูล:** ใช้การลบด้วยไฟฟ้า (Electrical Erasing) เช่นเดียวกับ EEPROM แต่มีความเร็วที่เหนือกว่ามาก โดยสามารถลบข้อมูลทั้งหมดในชิปได้ภายในเวลาเพียงไม่กี่วินาที
* **การลบข้อมูลแบบ "บล็อก" (Block):** สามารถเลือกการลบข้อมูลเป็นส่วนๆ หรือ "บล็อก" แทนที่จะต้องลบข้อมูลทั้งชิปพร้อมกัน ซึ่งเป็นที่มาของชื่อ "แฟลช" (Flash) เนื่องจากลักษณะการจัดระเบียบไมโครชิปที่ยอมให้มีการลบข้อมูลเป็นส่วนๆ ได้ในการกระทำเพียงครั้งเดียว
* **ข้อจำกัดและความคุ้มค่า:** ไม่สามารถลบข้อมูลในระดับ "ไบต์" (Byte) ได้โดยอิสระ แต่ด้วยโครงสร้างที่ใช้ทรานซิสเตอร์เพียงตัวเดียวต่อหนึ่งบิต ทำให้หน่วยความจำแฟลชมีความหนาแน่นของข้อมูลสูงและมีราคาประหยัดใกล้เคียงกับ EPROM

---

### กลไกการทำงานทางเทคนิค
การทำงานของเซลล์หน่วยความจำแฟลชมีพื้นฐานมาจากหลักการของทรานซิสเตอร์ทั่วไป ซึ่งอาศัยคุณสมบัติของสารกึ่งตัวนำในการควบคุมการไหลของกระแสไฟฟ้าจาก **"ซอร์ส" (Source)** ไปยัง **"เดรน" (Drain)** โดยใช้แรงดันไฟฟ้าจำนวนเล็กน้อยที่จ่ายให้แก่ **"เกต" (Gate)** เป็นตัวควบคุม



สิ่งที่ทำให้เซลล์หน่วยความจำแฟลชแตกต่างออกไปคือ:
* **เกตลอย (Floating Gate):** มีการเพิ่มส่วนนี้เข้าไปในโครงสร้าง โดยถูกหุ้มด้วยชั้นออกไซด์บางๆ เพื่อทำหน้าที่เป็นฉนวน
* **สถานะลอจิก "1":** ในสถานะเริ่มต้นที่เกตลอยไม่มีประจุไฟฟ้า เซลล์จะทำงานเสมือนทรานซิสเตอร์ปกติ
* **สถานะลอจิก "0":** เมื่อมีการจ่ายแรงดันไฟฟ้าที่สูงเพียงพอข้ามชั้นออกไซด์ จะเกิดปรากฏการณ์ที่อิเล็กตรอนสามารถเคลื่อนที่ผ่านชั้นฉนวนเข้าไปติดอยู่ในเกตลอยได้ หรือที่เรียกว่า **"การทะลุผ่าน" (Tunneling)** อิเล็กตรอนที่ถูกกักขังอยู่นี้จะยังคงอยู่แม้จะไม่มีกระแสไฟฟ้ามาเลี้ยงวงจร
* **การตรวจสอบและคืนสถานะ:** การตรวจสอบสถานะทำได้ผ่านวงจรภายนอกที่ทดสอบการนำไฟฟ้า และเมื่อต้องการลบข้อมูลหรือคืนสถานะกลับเป็น "1" ระบบจะใช้แรงดันไฟฟ้าสูงในทิศทางตรงกันข้ามเพื่อดึงอิเล็กตรอนออกจากเกตลอย

> **คุณสมบัติการคงสภาพข้อมูล (Persistence):** หน่วยความจำแฟลชเป็นหน่วยความจำแบบถาวร (Persistent Memory) ข้อมูลจะยังคงอยู่แม้ไม่มีกระแสไฟฟ้า จึงถูกนำไปใช้งานทั้งในฐานะหน่วยความจำสำรองภายนอก (Secondary Storage) และทางเลือกทดแทน RAM ในระบบคอมพิวเตอร์บางประเภท

### ประเภทของหน่วยความจำแฟลช: NOR และ NAND
เทคโนโลยีแฟลชสามารถจำแนกออกเป็น 2 ประเภทหลักตามลักษณะโครงสร้างทางตรรกะและการเชื่อมต่อ ดังนี้:

#### 1. หน่วยความจำแฟลชแบบ NOR (NOR Flash)
* **โครงสร้าง:** เซลล์แต่ละเซลล์จะถูกเชื่อมต่อแบบขนาน (Parallel) เข้ากับสายบิต (Bit lines)
* **การทำงาน:** สามารถอ่าน เขียน หรือลบข้อมูลในระดับบิตหรือไบต์ได้อย่างอิสระ มีลักษณะการทำงานคล้ายกับลอจิกเกตแบบ NOR
* **จุดเด่นและการใช้งาน:** มีความเร็วสูงในการเข้าถึงข้อมูลเฉพาะจุด (Random Access) นิยมใช้ในระบบฝังตัว (Embedded Systems) และไมโครคอนโทรลเลอร์ เพื่อจัดเก็บโปรแกรมคำสั่งที่มีขนาดไม่ใหญ่นักและข้อมูลที่ไม่ค่อยเปลี่ยนแปลงบ่อย

#### 2. หน่วยความจำแฟลชแบบ NAND (NAND Flash)
* **โครงสร้าง:** จัดระเบียบในรูปแบบของชุดทรานซิสเตอร์ที่เชื่อมต่อกันแบบอนุกรม (Series) โดยปกติจะมีทรานซิสเตอร์ 16 หรือ 32 ตัวต่อหนึ่งชุด ทำงานคล้ายลอจิกเกตแบบ NAND
* **จุดเด่น:** มีความหนาแน่นสูง (High Density) บรรจุข้อมูลได้มากในพื้นที่ขนาดเล็ก และมีความเร็วในการเขียนข้อมูลที่เหนือกว่าแบบ NOR
* **ข้อจำกัด:** ไม่มีบัสที่อยู่ภายนอกสำหรับการเข้าถึงแบบสุ่ม การอ่านข้อมูลจึงต้องทำเป็นส่วนๆ ที่เรียกว่า "หน้า" (Page) หรือ "บล็อก" (Block)
* **การใช้งาน:** เหมาะสำหรับเป็นหน่วยความจำภายนอก เช่น แฟลชไดรฟ์ USB, เมมโมรี่การ์ด และ SSD (Solid-State Disks)

---

### การเปรียบเทียบประสิทธิภาพด้วยกราฟคิเวียต (Kiviat Graphs)
แม้สถาปัตยกรรมทั้งสองจะมีการพัฒนาสมรรถนะในทุกปี แต่ความแตกต่างเชิงสัมพัทธ์ยังคงชัดเจน ซึ่งอธิบายได้ผ่าน **กราฟคิเวียต (Kiviat Graphs)** หรือกราฟใยแมงมุม:

* **หลักการอ่าน:** ยิ่งพื้นที่ที่ล้อมรอบด้วยเส้นกราฟมีขนาดกว้างมากเท่าใด (เข้าใกล้เส้นรอบวงมากขึ้น) แสดงว่าระบบนั้นมีประสิทธิภาพในภาพรวมดีขึ้นเท่านั้น
* **NOR Flash:** โดดเด่นในด้านความเร็วการอ่านและการเข้าถึงแบบสุ่ม
* **NAND Flash:** ชนะในด้านความหนาแน่นของข้อมูลและต้นทุนต่อหน่วยความจุ

# 5.5 เทคโนโลยีหน่วยความจำโซลิดสเตตแบบไม่ลบเลือนรุ่นใหม่ 
**(Newer Nonvolatile Solid-State Memory Technologies)**

เพื่อให้เข้าใจถึงบทบาทและวิวัฒนาการของเทคโนโลยีหน่วยความจำอุบัติใหม่ จำเป็นต้องพิจารณาจากโครงสร้าง **ลำดับชั้นของหน่วยความจำ (Memory Hierarchy)** แบบดั้งเดิม ซึ่งประกอบด้วย 3 ระดับหลักที่มีคุณสมบัติแตกต่างกันดังนี้:

* **Static RAM (SRAM):** มีความเร็วในการเข้าถึงข้อมูลสูงสุด แต่มีต้นทุนการผลิตสูงและความหนาแน่นของข้อมูลต่ำ จึงนิยมใช้เป็น **หน่วยความจำแคช (Cache Memory)**
* **Dynamic RAM (DRAM):** มีราคาถูกกว่าและมีความหนาแน่นของข้อมูลสูงกว่า SRAM แต่มีความเร็วในการทำงานต่ำกว่า จึงถูกใช้เป็น **หน่วยความจำหลัก (Main Memory)**
* **Hard Disk Drive (HDD):** อุปกรณ์จัดเก็บข้อมูลแบบจานแม่เหล็กที่มีความหนาแน่นของข้อมูลสูงและต้นทุนต่อบิตต่ำที่สุด แต่มีความเร็วในการเข้าถึงข้อมูลช้าที่สุด จึงเป็นตัวเลือกหลักสำหรับ **หน่วยความจำสำรอง (External Storage)**

---

### ข้อจำกัดและการก้าวข้ามผ่านเทคโนโลยีแฟลช
ในปัจจุบัน **หน่วยความจำแฟลช (Flash Memory)** ได้เข้ามามีบทบาทสำคัญในลำดับชั้นนี้ โดยมีจุดเด่นที่เป็นหน่วยความจำแบบไม่ลบเลือน (Nonvolatile) ซึ่งแบ่งออกเป็นสองประเภทหลักคือ **NOR Flash** ที่เหมาะสำหรับการเก็บโปรแกรมในระบบฝังตัว และ **NAND Flash** ที่มีคุณสมบัติอยู่กึ่งกลางระหว่าง DRAM และฮาร์ดดิสก์

อย่างไรก็ตาม แม้เทคโนโลยีแฟลชจะมีการพัฒนาอย่างต่อเนื่อง แต่การเพิ่มความหนาแน่นและประสิทธิภาพ (Scaling) เริ่มเผชิญกับข้อจำกัดทางกายภาพของสารกึ่งตัวนำ ด้วยเหตุนี้จึงมีการคิดค้น **เทคโนโลยีหน่วยความจำเกิดใหม่ (Emerging Memories)** เพื่อทลายขีดจำกัดเดิมและเติมเต็มช่องว่างระหว่างหน่วยความจำหลักและหน่วยความจำสำรอง



---

### เทคโนโลยีหน่วยความจำเกิดใหม่ที่มีศักยภาพสูง
ปัจจุบันมีเทคโนโลยี 3 ประเภทหลักที่กำลังก้าวขึ้นมามีบทบาทสำคัญ ได้แก่:

#### 1. STT-RAM (Spin-Transfer Torque RAM)
STT-RAM คือวิวัฒนาการขั้นสูงของหน่วยความจำแม่เหล็ก (MRAM) ที่อาศัยหลักการทางแม่เหล็กและสปินของอิเล็กตรอน
* **กลไกการทำงาน:** ใช้โครงสร้าง **รอยต่ออุโมงค์แม่เหล็ก (Magnetic Tunneling Junction - MTJ)** ซึ่งประกอบด้วยชั้น "อ้างอิง" (Pinned Layer) ที่มีทิศทางแม่เหล็กคงที่ และชั้น "อิสระ" (Free Layer) ที่สามารถสลับทิศทางได้ผ่านกระแสไฟฟ้าโดยตรง
* **คุณสมบัติเด่น:** มีความเร็วในการอ่าน-เขียนสูงมาก (6-10 นาโนวินาที) ทนทานต่อการเขียนซ้ำได้มหาศาลถึง $7 \times 10^{15}$ รอบ และประหยัดพลังงานในสถานะสแตนด์บาย เหมาะสำหรับใช้เป็นหน่วยความจำแคชหรือหน่วยความจำหลัก



#### 2. PCRAM (Phase-Change RAM)
PCRAM เป็นเทคโนโลยีที่มีความพร้อมทางการตลาดสูงที่สุด โดยใช้วัสดุผสม **ชาลโคไจนายด์ (Chalcogenide Alloy)** ชนิดเดียวกับที่ใช้ในแผ่นออปติคัลดิสก์
* **กลไกการทำงาน:** ใช้การเปลี่ยนสถานะทางกายภาพของวัสดุระหว่าง **สถานะอสัณฐาน (Amorphous)** ที่มีความต้านทานสูง และ **สถานะผลึก (Crystalline)** ที่มีความต้านทานต่ำ 
* **การควบคุม:** ผ่านกระแสไฟฟ้าที่ควบคุมอุณหภูมิ (SET/RESET Operation) เพื่อเปลี่ยนโครงสร้างของโมเลกุลวัสดุ 
* **เป้าหมาย:** ถูกวางตัวเป็นคู่แข่งสำคัญที่จะเข้ามาเสริมหรือแทนที่ DRAM เพื่อยกระดับประสิทธิภาพความทนทานของระบบ



#### 3. ReRAM (Resistive RAM)
ReRAM หรือ RRAM ใช้นวัตกรรมการเปลี่ยน "ระดับความต้านทานไฟฟ้า" ภายในวัสดุไดอิเล็กทริกเป็นกลไกหลักในการบันทึกข้อมูล แทนการกักเก็บประจุไฟฟ้าแบบเดิม
* **คุณสมบัติเด่น:** ทำงานได้ภายใต้แรงดันไฟฟ้าที่ต่ำมาก มีความทนทานสูง และในทางทฤษฎีเซลล์ของ ReRAM มีขนาดเล็กกว่าเทคโนโลยีอื่นมาก ทำให้เพิ่มความจุในพื้นที่จำกัดได้อย่างดีเยี่ยม
* **บทบาท:** เป็นผู้ท้าชิงที่เหมาะสมในการทำงานร่วมกับทั้งหน่วยความจำหลักและหน่วยความจำสำรอง เพื่อลดคอขวดด้านความเร็วของระบบคอมพิวเตอร์

---

### สถานะปัจจุบันและการคาดการณ์ในอนาคต
แม้ในปัจจุบัน NAND Flash และ NOR Flash จะยังคงครองตลาดส่วนใหญ่ แต่เทคโนโลยีเกิดใหม่อย่าง STT-RAM, PCRAM และ ReRAM ได้เริ่มเข้าสู่กระบวนการผลิตเชิงพาณิชย์ในแอปพลิเคชันเฉพาะทาง (Specialty Applications) แล้ว 

ผู้เชี่ยวชาญคาดการณ์ว่าภายในอนาคตอันใกล้ เทคโนโลยีเหล่านี้จะกลายเป็น **เทคโนโลยีหลัก (Mainstream)** ที่เข้ามาเปลี่ยนแปลงโครงสร้างลำดับชั้นของหน่วยความจำให้มีความสมบูรณ์แบบ ทั้งในด้านความเร็ว ความจุ และการใช้พลังงานอย่างมีประสิทธิภาพ

---

# 5.6 คำศัพท์สำคัญ และคำถามทบทวน
**(Key Terms and Review Questions)**

### 1. คำศัพท์สำคัญ (Key Terms)
การทำความเข้าใจนิยามของคำศัพท์เหล่านี้จะช่วยให้สื่อสารและทำความเข้าใจเนื้อหาขั้นสูงได้แม่นยำขึ้น:

* **Bank Group:** กลุ่มของหน่วยความจำย่อยที่ช่วยให้สามารถรับ-ส่งข้อมูลได้พร้อมกันหลายทาง
* **Double Data Rate DRAM (DDR DRAM):** หน่วยความจำที่รับส่งข้อมูลได้ทั้งขาขึ้นและขาลงของสัญญาณนาฬิกา
* **Dynamic RAM (DRAM):** แรมที่ต้องคอยประจุไฟใหม่ตลอดเวลา (Refresh) มีความหนาแน่นสูง
* **Static RAM (SRAM):** แรมที่คงสถานะข้อมูลไว้ได้ตราบที่มีไฟเลี้ยงโดยไม่ต้อง Refresh มีความเร็วสูง
* **EEPROM:** หน่วยความจำถาวรที่ลบและเขียนซ้ำได้ด้วยสัญญาณไฟฟ้า
* **EPROM:** หน่วยความจำถาวรที่ลบข้อมูลได้ด้วยการส่องแสงอัลตราไวโอเลต (UV)
* **Error Correcting Code (ECC):** รหัสที่ใช้ตรวจจับและแก้ไขความผิดพลาดของบิตข้อมูล
* **Hamming Code:** อัลกอริทึมยอดนิยมในการระบุตำแหน่งและแก้ไขบิตที่ผิดพลาด
* **Flash Memory:** หน่วยความจำไม่ลบเลือนที่พัฒนามาจาก EEPROM ลบข้อมูลได้รวดเร็วเป็นบล็อก
* **MRAM / STT-RAM / PCRAM / ReRAM:** กลุ่มเทคโนโลยีหน่วยความจำอุบัติใหม่ที่มีประสิทธิภาพสูงและไม่ลบเลือนข้อมูล
* **Volatile Memory:** หน่วยความจำแบบลบเลือน (ข้อมูลหายเมื่อไม่มีไฟ)
* **Nonvolatile Memory:** หน่วยความจำแบบไม่ลบเลือน (ข้อมูลยังอยู่แม้ไม่มีไฟ)
* **Hard Failure:** ความเสียหายถาวรของฮาร์ดแวร์
* **Soft Error:** ความผิดพลาดชั่วคราวของข้อมูลที่เกิดจากสัญญาณรบกวน

---

### 2. แนวทางคำตอบสำหรับคำถามทบทวน (Review Questions)

**5.1 คุณสมบัติสำคัญของหน่วยความจำสารกึ่งตัวนำคืออะไร?**
* **ตอบ:** มีความสามารถในการแสดงสถานะทางไฟฟ้าที่แตกต่างกันสองสถานะ (เพื่อแทน 0 และ 1), สามารถถูกเขียนข้อมูลลงไปได้อย่างน้อยหนึ่งครั้งเพื่อกำหนดสถานะ และสามารถอ่านค่าเพื่อตรวจสอบสถานะที่บันทึกไว้ได้

**5.2 การตีความคำว่า "Random-Access Memory (RAM)" มี 2 แบบอย่างไร?**
* **ตอบ:** 1. ในเชิงกลไกการเข้าถึง: หมายถึงหน่วยความจำที่ใช้เวลาในการเข้าถึงข้อมูลทุกตำแหน่งเท่ากัน 2. ในเชิงการใช้งาน: หมายถึงหน่วยความจำหลักที่สามารถอ่านและเขียนข้อมูลใหม่ได้ตลอดเวลาในขณะที่เครื่องทำงาน

**5.3 & 5.4 ความแตกต่างระหว่าง DRAM และ SRAM (ด้านการใช้งานและคุณลักษณะ)?**
* **DRAM:** นิยมใช้เป็น **หน่วยความจำหลัก (Main Memory)** เนื่องจากมีขนาดเซลล์ที่เล็กมาก (ประหยัดพื้นที่) และราคาถูก แต่มีความเร็วต่ำกว่า
* **SRAM:** นิยมใช้เป็น **หน่วยความจำแคช (Cache)** เนื่องจากมีความเร็วในการเข้าถึงข้อมูลสูงมาก แต่มีราคาสูงและขนาดเซลล์ใหญ่กว่า

**5.5 ทำไม RAM ประเภทหนึ่งถึงถูกมองว่าเป็น Analog และอีกประเภทเป็น Digital?**
* **ตอบ:** **DRAM** ถูกมองว่าเป็นแอนาล็อกเพราะอาศัยการจัดเก็บ "ประจุไฟฟ้า" ในตัวเก็บประจุซึ่งสามารถรั่วไหลได้ตามเวลา ส่วน **SRAM** เป็นดิจิทัลเพราะใช้โครงสร้างลอจิกเกต (Flip-flop) เพื่อรักษาสถานะแรงดันไฟฟ้าให้คงที่

**5.6 ตัวอย่างการใช้งาน ROM มีอะไรบ้าง?**
* **ตอบ:** ใช้จัดเก็บโปรแกรมเริ่มต้นระบบ (Library subroutines for frequently wanted functions), ระบบควบคุมการทำงาน (System programs/BIOS) หรือตารางฟังก์ชันคงที่ต่างๆ

**5.7 ความแตกต่างระหว่าง EPROM, EEPROM และ Flash Memory?**
* **EPROM:** ลบข้อมูลด้วยแสง UV และต้องลบข้อมูลพร้อมกันทั้งชิป
* **EEPROM:** ลบด้วยไฟฟ้าและเลือกโหมดการลบในระดับไบต์ได้ แต่มีความหนาแน่นต่ำ
* **Flash Memory:** ลบด้วยไฟฟ้าได้รวดเร็ว โดยลบข้อมูลเป็นบล็อก (Block-level) ทำให้มีความจุสูงกว่า EEPROM มาก

**5.9 Parity Bit คืออะไร?**
* **ตอบ:** บิตที่ถูกเพิ่มเข้าไปในชุดข้อมูลเพื่อใช้ในการตรวจสอบความถูกต้อง โดยจะนับจำนวนบิตที่เป็น "1" ให้เป็นเลขคู่หรือเลขคี่ตามที่กำหนดไว้ เพื่อตรวจจับว่ามีบิตใดบิตหนึ่งผิดพลาดไปหรือไม่

**5.10 Syndrome ใน Hamming Code แปลความหมายอย่างไร?**
* **ตอบ:** ค่า Syndrome คือผลลัพธ์จากการตรวจสอบ Parity หากค่าเป็น 0 แสดงว่าไม่มีข้อผิดพลาด แต่หากไม่เป็น 0 ค่าที่เป็นตัวเลขนั้นจะระบุถึง "ตำแหน่งของบิต" ที่เกิดความผิดพลาดขึ้นโดยตรง

**5.11 SDRAM ต่างจาก DRAM ทั่วไปอย่างไร?**
* **ตอบ:** SDRAM จะทำงานสัมพันธ์ (Synchronize) กับสัญญาณนาฬิกาของระบบโดยตรง ทำให้การรับส่งข้อมูลระหว่าง CPU และ RAM มีความพร้อมเพรียงและรวดเร็วกว่า DRAM ทั่วไปที่ทำงานแบบอซิงโครนัส

**5.12 DDR RAM คืออะไร?**
* **ตอบ:** เป็นวิวัฒนาการของ SDRAM ที่สามารถส่งข้อมูลได้ 2 ครั้งต่อหนึ่งรอบสัญญาณนาฬิกา (ส่งทั้งช่วงขาขึ้นและขาลง) ทำให้ได้อัตราการส่งข้อมูลเพิ่มขึ้นเป็น 2 เท่า

**5.13 ความแตกต่างระหว่าง NAND และ NOR Flash?**
* **NOR Flash:** เข้าถึงข้อมูลแบบสุ่มได้เร็ว เหมาะสำหรับเก็บโปรแกรมคำสั่ง (Code)
* **NAND Flash:** ความหนาแน่นสูงและเขียนข้อมูลเร็ว เข้าถึงข้อมูลเป็นบล็อก เหมาะสำหรับเก็บข้อมูลปริมาณมาก (Data Storage) เช่น SSD

**5.14 จงระบุเทคโนโลยีหน่วยความจำไม่ลบเลือนรุ่นใหม่มา 3 ประเภท?**
* **ตอบ:** 1. **STT-RAM:** ใช้หลักการหมุนของอิเล็กตรอน 2. **PCRAM:** ใช้การเปลี่ยนสถานะวัสดุ 3. **ReRAM:** ใช้การเปลี่ยนความต้านทานไฟฟ้า

# 5.6 โจทย์ปัญหาและแบบฝึกหัด (Problems)
**(แปลและเฉลยแนวทางคำตอบ)**

### โจทย์ข้อ 5.1: การจัดองค์กรของชิป (RAM vs ROM)
**โจทย์:** จงวิเคราะห์เหตุผลว่าทำไมตามธรรมเนียมแล้ว RAM มักถูกจัดองค์กรแบบ **1 บิตต่อ 1 ชิป** ในขณะที่ ROM มักถูกจัดองค์กรแบบ **หลายบิตต่อ 1 ชิป**
* **แนวทางคำตอบ:** * **RAM:** เนื่องจาก RAM มักถูกนำมาต่อรวมกันเพื่อสร้างหน่วยความจำขนาดใหญ่ (เช่น 8GB, 16GB) การออกแบบให้ 1 ชิปส่งข้อมูลออกมาเพียง 1 บิต ช่วยให้เราสามารถนำชิปหลายตัวมาขนานกันเพื่อเพิ่มความกว้างของข้อมูล (Word size) เช่น ใช้ชิป 8 ตัวเพื่อให้ได้ข้อมูล 1 ไบต์ (8 บิต) ซึ่งช่วยลดจำนวนเข็มสัญญาณ (Pins) บนชิปแต่ละตัวให้น้อยลง ประหยัดพื้นที่และลดความซับซ้อนของแผงวงจร
    * **ROM:** มักใช้เก็บโปรแกรมหรือข้อมูลที่มีขนาดแน่นอนและมักใช้งานในอุปกรณ์ขนาดเล็ก การจัดองค์กรแบบหลายบิต (เช่น 8 บิต/1 ไบต์ต่อชิป) ช่วยให้สามารถดึงข้อมูลออกมาใช้งานได้ทันทีจากชิปเพียงตัวเดียว ซึ่งประหยัดพื้นที่ในอุปกรณ์ฝังตัว (Embedded Systems)

---

### โจทย์ข้อ 5.2: การคำนวณเวลาการ Refresh ของ DRAM
**โจทย์:** พิจารณา DRAM ที่ต้องทำการ Refresh 64 ครั้งในทุกๆ 1 ms โดยการ Refresh แต่ละครั้งใช้เวลา 150 ns และ Memory Cycle ปกติใช้เวลา 250 ns จงหาว่า "เปอร์เซ็นต์ของเวลาการทำงานทั้งหมดที่เสียไปกับการ Refresh" คือเท่าใด?

**วิธีทำ:**
1.  **หาเวลาที่ใช้ในการ Refresh ทั้งหมดต่อ 1 ms:**
    $$64 \text{ ครั้ง} \times 150 \text{ ns/ครั้ง} = 9,600 \text{ ns}$$
2.  **เทียบกับเวลาทั้งหมด (1 ms):**
    $$1 \text{ ms} = 1,000,000 \text{ ns}$$
3.  **คำนวณเป็นเปอร์เซ็นต์:**
    $$\frac{9,600 \text{ ns}}{1,000,000 \text{ ns}} \times 100 = 0.96\%$$
* **คำตอบ:** เสียเวลาไปกับการ Refresh เพียง **0.96%**

---

### โจทย์ข้อ 5.3: Timing Diagram และอัตราการส่งข้อมูล (Data Rate)
**โจทย์:** จากแผนผังเวลาการอ่านข้อมูล (Timing Diagram) ของ DRAM:
* **ก.** หาก Access time ($t1$ ถึง $t2$) คือ 60 ns และ Recharge time ($t2$ ถึง $t3$) คือ 40 ns จงหาค่า **Memory Cycle Time** และ **Maximum Data Rate** (เมื่อส่งออกครั้งละ 1 บิต)
* **ข.** หากสร้างระบบหน่วยความจำกว้าง 32 บิต จะได้อัตราการส่งข้อมูลเท่าใด?



**วิธีทำ:**
* **ข้อ ก.**
    1.  **Memory Cycle Time** = Access time + Recharge time
        $$60 \text{ ns} + 40 \text{ ns} = 100 \text{ ns}$$
    2.  **Data Rate (1-bit output)**: ใน 1 วินาที ($10^9 \text{ ns}$) จะทำงานได้กี่รอบ
        $$\frac{1}{100 \text{ ns}} = 10,000,000 \text{ รอบต่อวินาที (10 Mbps)}$$
* **ข้อ ข.**
    1.  **Data Transfer Rate (32-bit)**:
        $$10,000,000 \text{ รอบ} \times 32 \text{ บิต} = 320,000,000 \text{ bps (320 Mbps)}$$

---

### โจทย์ข้อ 5.4: การออกแบบโมดูลหน่วยความจำ (Memory Module Design)
**โจทย์:** จงวาดแผนผัง (High-level diagram) การสร้างหน่วยความจำขนาด **8 MB** โดยใช้ชิปขนาด **1 MB** จำนวน 8 ตัว (กำหนดให้ Word size คือ 1 ไบต์) พร้อมระบุการใช้เส้นสัญญาณ Address (Address lines)

**แนวทางคำอธิบายแผนผัง:**
1.  **จำนวนบิต Address:** เพื่อเข้าถึงข้อมูล 8 MB ($2^3 \times 2^{20} = 2^{23}$ ตำแหน่ง) ต้องใช้เส้นสัญญาณ Address ทั้งหมด **23 เส้น** (A0 - A22)
2.  **การแบ่งสัญญาณ:**
    * **A0 - A19 (20 เส้น):** ส่งไปยังชิป 1 MB ทุกตัวพร้อมกัน เพื่อเลือกตำแหน่งข้อมูลภายในชิปแต่ละตัว ($2^{20} = 1 \text{ MB}$)
    * **A20 - A22 (3 เส้น):** ส่งไปยังตัวถอดรหัส (**3-to-8 Decoder**) เพื่อเลือกเปิดการทำงานของชิปตัวใดตัวหนึ่ง (Chip Select - CS) จากทั้งหมด 8 ตัว
3.  **Data Lines:** เนื่องจากแต่ละชิปมีขนาด Word 1 ไบต์ (8 บิต) เส้นสัญญาณข้อมูล (D0 - D7) ของชิปทุกตัวจะถูกเชื่อมต่อขนานกันไปยัง Bus ข้อมูลหลัก

---

#### **ข้อ 5.5: การคำนวณ Access Time และ Wait State (ระบบ Intel 8086)**
**โจทย์:** ระบบทำงานที่ความถี่ 10 MHz โดย RAS ทำงานช้ากว่าสัญญาณ Address Enable 50 ns และต้องส่งข้อมูลให้ CPU ก่อนจบสถานะ T3 เป็นเวลา 60 ns
* **ก. Access Time ที่ต้องการ (แบบไม่มี Wait State):**
    1. หนึ่งรอบสัญญาณนาฬิกา (T) = $1 / 10 \text{ MHz} = 100 \text{ ns}$
    2. ช่วงเวลา T1 ถึง T3 รวมกันคือ $3 \times 100 = 300 \text{ ns}$
    3. RAS เริ่มทำงานที่กึ่งกลางของครึ่งหลังของ T1 + ดีเลย์ 50 ns: จะเริ่มที่ประมาณ $75 \text{ ns} + 50 \text{ ns} = 125 \text{ ns}$ นับจากจุดเริ่ม T1
    4. CPU อ่านข้อมูลที่ปลาย T3 แต่ข้อมูลต้องพร้อมก่อน 60 ns: คือที่เวลา $300 \text{ ns} - 60 \text{ ns} = 240 \text{ ns}$
    5. **Access Time** = เวลาที่ข้อมูลต้องพร้อม - เวลาที่เริ่ม RAS = $240 - 125 = 115 \text{ ns}$
* **ข. จำนวน Wait State (ถ้า Access Time เป็น 150 ns):**
    * หากใช้ 150 ns ข้อมูลจะพร้อมที่ $125 + 150 = 275 \text{ ns}$ ซึ่งเกินกำหนดที่ 240 ns ไป 35 ns จึงต้องเพิ่ม Wait State (รอบละ 100 ns) เข้าไป **1 รอบ (1 Wait State)**

#### **ข้อ 5.6: การ Refresh หน่วยความจำแบบคาบเวลา (Periodic Refresh)**
**โจทย์:** ใช้ DRAM ขนาด $64\text{K} \times 1$ มี 256 แถว ต้อง Refresh ทุกแถวภายใน 4 ms
* **ก. ระยะห่างระหว่างรอบ Refresh:**
    * $\frac{4 \text{ ms}}{256 \text{ แถว}} = 15.625 \mu\text{s}$ (หรือประมาณ **15.6 ไมโครวินาที**)
* **ข. ขนาดของตัวนับตำแหน่ง (Refresh Address Counter):**
    * เนื่องจากมี 256 แถว ($2^8$) จึงต้องใช้ตัวนับขนาด **8 บิต**

#### **ข้อ 5.7: การทำงานของ SRAM (Signetics 7489)**
* **หัวใจสำคัญ:** เป็นการวิเคราะห์สัญญาณ Chip Select (CS) และโหมดการทำงาน (Read/Write) ตามช่วงเวลาในแผนผัง
    * **ก. โหมด:** ขึ้นอยู่กับสัญญาณ Write Enable (WE) และ CS หาก CS ต่ำและ WE ต่ำจะเป็นโหมด "เขียน" หาก CS ต่ำและ WE สูงจะเป็นโหมด "อ่าน"
    * **ข. ข้อมูลตำแหน่ง 0-6:** ต้องดูค่าจาก Input Data ในจังหวะที่สัญญาณ Write ทำงาน
    * **ค. สถานะ Output:** ในจังหวะที่ไม่ได้อ่าน ข้อมูลขาออกมักจะเป็น High Impedance หรือสถานะพัก

#### **ข้อ 5.8: การออกแบบหน่วยความจำ 8192 บิต (16-bit word)**
**โจทย์:** ใช้ชิป SRAM $64 \times 1$ บิต สร้างหน่วยความจำรวม 8192 บิต
1. **จำนวนชิปที่ต้องใช้:** $8192 / 64 = 128 \text{ ชิป}$
2. **การจัดกลุ่ม:** เนื่องจากต้องการความกว้าง 16 บิต จึงต้องแบ่งชิปออกเป็นกลุ่มๆ ละ 16 ตัวขนานกัน (ได้ข้อมูล 16 บิตต่อชุด)
3. **ความจุรวมในเชิงตำแหน่ง:** $8192 / 16 \text{ บิต} = 512 \text{ ตำแหน่ง}$
4. **เส้นสัญญาณ:** ต้องใช้ Address 9 เส้น ($2^9 = 512$) เพื่อระบุตำแหน่ง

#### **ข้อ 5.9: การคำนวณ MTBF (Mean Time Between Failures)**
**โจทย์:** หน่วยความจำ 1 MB ($16\text{-bit processor}$) ใช้ชิป $256\text{K} \times 1$ โดยแต่ละชิปมีอัตราเสีย (FIT) 2000
1. **หาจำนวนชิปทั้งหมด:**
    * 1 MB คือ 8,388,608 บิต
    * ใช้ชิปขนาด 262,144 บิต ($256\text{K}$)
    * จำนวนชิป = $8,388,608 / 262,144 = 32 \text{ ชิป}$
2. **หาอัตราเสียรวม (Total FIT):**
    * $32 \text{ ชิป} \times 2000 \text{ FIT/ชิป} = 64,000 \text{ FIT}$ (เสีย 64,000 ครั้งต่อ $10^9$ ชั่วโมง)
3. **คำนวณค่า MTBF:**
    * $\text{MTBF} = \frac{10^9}{64,000} = 15,625 \text{ ชั่วโมง}$
* **สรุป:** โดยเฉลี่ยหน่วยความจำชุดนี้จะใช้งานได้ประมาณ **15,625 ชั่วโมง** ก่อนจะเกิดการเสียครั้งแรก

---

#### **ข้อ 5.10: เมื่อบิตตรวจสอบ (Check Bit) เกิดข้อผิดพลาดเอง**
**โจทย์:** จะเกิดอะไรขึ้นหากบิตที่ผิดพลาดคือบิตตรวจสอบ (Check Bit) ไม่ใช่บิตข้อมูล (Data Bit)?
* **คำตอบ:** อัลกอริทึมของ Hamming จะยังคงคำนวณค่า **Syndrome** ออกมาตามปกติ ซึ่งค่า Syndrome ที่ได้จะตรงกับ **เลขตำแหน่งของบิตตรวจสอบนั้นพอดี** (เช่น 1, 2, 4, 8) ระบบจะรับรู้ว่าบิตที่ตำแหน่งนั้นผิดพลาดและทำการสลับค่า (Flip) บิตนั้นกลับให้ถูกต้อง แต่เนื่องจากเป็นเพียงบิตตรวจสอบที่ผิด ข้อมูลจริง (Data Bits) จึงยังคงถูกต้องและใช้งานได้ตามเดิม

---

#### **ข้อ 5.11: การหา Check Bits สำหรับข้อมูล 8 บิต (11000010)**
**โจทย์:** จงหาบิตตรวจสอบที่จะถูกเก็บไว้พร้อมกับข้อมูล `11000010`
**วิธีทำ:**
1.  **วางตำแหน่งบิต:** ข้อมูล 8 บิตต้องการบิตตรวจสอบ 4 บิต (ตำแหน่ง 1, 2, 4, 8)
    
    * ตำแหน่ง: 1(C1), 2(C2), 3(D1), 4(C4), 5(D2), 6(D3), 7(D4), 8(C8), 9(D5), 10(D6), 11(D7), 12(D8)
    * วางข้อมูล: `_ _ 1 _ 1 0 0 _ 0 0 1 0`
2.  **คำนวณ Check Bits (ใช้ Even Parity):**
    * **C1** (ตรวจสอบตำแหน่ง 1,3,5,7,9,11): `_ 1 1 0 0 1` -> มี 1 สามตัว -> **C1 = 1**
    * **C2** (ตรวจสอบตำแหน่ง 2,3,6,7,10,11): `_ 1 0 0 0 1` -> มี 1 สองตัว -> **C2 = 0**
    * **C4** (ตรวจสอบตำแหน่ง 4,5,6,7,12): `_ 1 0 0 0` -> มี 1 หนึ่งตัว -> **C4 = 1**
    * **C8** (ตรวจสอบตำแหน่ง 8,9,10,11,12): `_ 0 0 1 0` -> มี 1 หนึ่งตัว -> **C8 = 1**
* **คำตอบ:** บิตตรวจสอบคือ **1011**

---

#### **ข้อ 5.12: การหาตำแหน่งบิตที่ผิดจากค่า Syndrome**
**โจทย์:** ข้อมูลเดิมมี Check Bits คือ `0111` แต่เมื่ออ่านออกมาคำนวณใหม่ได้ `1101` ข้อมูลที่อ่านได้คืออะไร?
**วิธีทำ:**
1.  หาค่า **Syndrome** โดยการ XOR ระหว่าง Check Bits เดิมและที่คำนวณใหม่:
    * `0111` XOR `1101` = `1010` (ฐานสอง)
2.  แปลงเป็นเลขฐานสิบ: $1010_2 = 10$
3.  **สรุป:** บิตที่ตำแหน่ง **10** เกิดข้อผิดพลาด ระบบจะทำการแก้ไขบิตที่ 10 ให้ถูกต้องก่อนส่งให้ผู้ใช้

---

#### **ข้อ 5.13: จำนวน Check Bits สำหรับข้อมูล 1024 บิต**
**โจทย์:** ต้องใช้กี่บิตเพื่อแก้ไข Single Bit Error ในข้อมูล 1024 บิต?
**วิธีทำ:** ใช้สูตร $2^K - 1 \ge M + K$ (โดย $M$ คือจำนวนบิตข้อมูล, $K$ คือบิตตรวจสอบ)
* ลองแทนค่า $K = 10$: $2^{10} - 1 = 1023$
    * $1023 \ge 1024 + 10$ (ไม่จริง: $1023 < 1034$)
* ลองแทนค่า $K = 11$: $2^{11} - 1 = 2047$
    * $2047 \ge 1024 + 11$ (จริง: $2047 \ge 1035$)
* **คำตอบ:** ต้องใช้บิตตรวจสอบ **11 บิต**

#### **ข้อ 5.14: การสร้าง SEC สำหรับข้อมูล 16 บิต**
**โจทย์:** สร้างรหัสสำหรับข้อมูล `0101000000111001` และพิสูจน์ว่าแก้ไขบิตที่ 5 ได้
**แนวคิด:**
1.  **บิตตรวจสอบ:** ข้อมูล 16 บิตต้องใช้บิตตรวจสอบ 5 บิต ($K=5$ เพราะ $2^5-1 = 31 \ge 16+5$)
2.  **การพิสูจน์:** เมื่อเกิด Error ใน Data Bit 5 (ซึ่งคือตำแหน่งที่ 9 ในแถว Hamming) ค่า Syndrome ที่คำนวณได้จากการเปรียบเทียบ Parity ของบิตตรวจสอบเดิมกับบิตตรวจสอบใหม่จะออกมาเป็น `01001` (หรือตำแหน่งที่ 9) ซึ่งจะชี้ไปยังตำแหน่งบิตที่ผิดอย่างแม่นยำ
