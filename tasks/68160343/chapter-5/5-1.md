# Semiconductor Main Memory

หน่วยความจำหลักในอดีตใช้เทคโนโลยีแกนแม่เหล็กคล้ายวงแหวนที่เรียกว่า คอร์ ทำให้อดีตคนก็เรียกหน่วยความจำว่า คอร์  
และเมื่อเทคโนโลยีไมโครอิเล็กทรอนิกส์พัฒนา จึงเปลี่ยนมาใช้ชิปสารกึ่งตัวนำ (Semiconductor) แทน เนื่องจากมีประสิทธิภาพดีกว่าและเป็นมาตรฐานในปัจจุบัน


## Organization

หน่วยความจำแบบสารกึ่งตัวนำประกอบด้วยหน่วยย่อยที่เรียกว่า memory cell โดยแต่ละเซลล์มีคุณสมบัติหลักคือ

- มีสถานะได้ 2 ค่า โดยใช้แทนข้อมูลแบบไบนารี คือ 0,1
- สามารถเขียนข้อมูลเพื่อกำหนดสถานะได้
- สามารถอ่านข้อมูลเพื่อตรวจสอบสถานะได้

การทำงานของ memory cell โดยทั่วไปจะมีขั้ว (terminal) หลัก 3 ส่วน ได้แก่

- Select: ใช้เลือกเซลล์ที่จะอ่านหรือเขียนข้อมูล
- Control: ใช้กำหนดว่าจะ “อ่าน” หรือ “เขียน”
- Data: ใช้รับข้อมูลเข้า (ตอนเขียน) หรือส่งข้อมูลออก (ตอนอ่าน)


## DRAM AND SRAM(RAM)

หน่วยความจำทุกประเภทในส่วนนี้เป็นแบบ Random Access ซึ่งสามารถเข้าถึงข้อมูลแต่ละตำแหน่งได้โดยตรงผ่านวงจรระบุตำแหน่ง (addressing) โดยไม่ต้องเรียงลำดับก่อน


## DRAM and SRAM

หน่วยความจำทั้งหมดในบทนี้เป็นแบบ Random Access คือสามารถเข้าถึงข้อมูลแต่ละตำแหน่งได้โดยตรงผ่านวงจร addressing  

RAM (Random Access Memory) เป็นหน่วยความจำที่อ่านและเขียนข้อมูลได้รวดเร็วผ่านสัญญาณไฟฟ้า และเป็นแบบ volatile (ไฟดับข้อมูลหาย) จึงใช้เป็นหน่วยความจำชั่วคราว โดยแบ่งเป็น 2 ประเภทหลักคือ DRAM และ SRAM


### DRAM (Dynamic RAM)

- เก็บข้อมูลในรูปประจุไฟฟ้าในตัวเก็บประจุ (capacitor)
- ต้องมีการ refresh เป็นระยะ เพราะประจุไฟฟ้ารั่วได้
- ขณะอ่านข้อมูลจะทำให้ประจุหาย ต้องเขียนกลับใหม่
- โครงสร้างเรียบง่าย ขนาดเล็ก → ความจุสูง ราคาถูก
- ทำงานแบบกึ่งอนาล็อก (ใช้ระดับประจุตีความเป็น 0/1)


### SRAM (Static RAM)

- เก็บข้อมูลด้วยวงจร flip-flop (logic gate)
- ไม่ต้อง refresh ข้อมูล
- คงข้อมูลได้ตราบใดที่ยังมีไฟเลี้ยง
- เร็วกว่า DRAM แต่โครงสร้างซับซ้อนและขนาดใหญ่กว่า


### เปรียบเทียบ DRAM vs SRAM

- DRAM: ราคาถูก ความจุสูง แต่ช้ากว่า และต้อง refresh
- SRAM: เร็วกว่า แต่ราคาแพงและความจุต่ำกว่า
- ทั้งคู่เป็น volatile

การใช้งาน
- SRAM → ใช้เป็น Cache Memory
- DRAM → ใช้เป็น Main Memory


## Types of ROM

ROM (Read-Only Memory) เป็นหน่วยความจำแบบ ไม่ลบเลือน (nonvolatile) ข้อมูลถูกบันทึกถาวร ไม่สามารถเขียนใหม่ได้ และไม่ต้องใช้ไฟเลี้ยงในการเก็บข้อมูล  
เหมาะสำหรับเก็บโปรแกรมหรือข้อมูลที่ต้องใช้ประจำ เช่น system program หรือ function table

ข้อเสียของ ROM
- ต้นทุนเริ่มต้นสูง
- แก้ไขข้อมูลไม่ได้ (ถ้าผิดต้องทิ้งทั้งหมด)

ประเภทของ ROM

1. PROM (Programmable ROM)
   - เขียนข้อมูลได้ ครั้งเดียว
   - เขียนด้วยไฟฟ้าหลังจากผลิตเสร็จ
   - ยืดหยุ่นกว่า ROM ปกติ

2. EPROM (Erasable PROM)
   - ลบข้อมูลได้ด้วย แสง UV
   - ต้องลบทั้งชิปก่อนเขียนใหม่
   - ใช้เวลาลบนาน (ประมาณหลายสิบนาที)
   - เขียนซ้ำได้หลายครั้ง

3. EEPROM (Electrically Erasable PROM)
   - ลบและเขียนใหม่ได้ด้วยไฟฟ้า
   - แก้ไขข้อมูลเป็น ราย byte ได้
   - ไม่ต้องลบทั้งชิป
   - ช้ากว่า RAM และราคาสูง

4. Flash Memory
   - พัฒนามาจาก EEPROM
   - ลบข้อมูลได้เป็น block (เร็วกว่า EPROM มาก)
   - ความจุสูงและราคาถูกกว่า EEPROM
   - ใช้กันแพร่หลาย (เช่น USB, SSD)

สรุปภาพรวม
- ROM → ถาวร แก้ไม่ได้
- PROM → เขียนได้ครั้งเดียว
- EPROM → ลบด้วยแสง
- EEPROM → ลบด้วยไฟฟ้า (ยืดหยุ่นสูง)
- Flash → เร็ว + ใช้งานจริงมากที่สุด


## Chip Logic

หน่วยความจำแบบสารกึ่งตัวนำถูกบรรจุในรูปแบบ ชิป (chip) โดยภายในประกอบด้วยอาร์เรย์ของ memory cell  

การออกแบบชิปต้องมีการแลกเปลี่ยนระหว่าง ความเร็ว ความจุ และต้นทุน โดยปัจจัยสำคัญคือจำนวนบิตที่สามารถอ่าน/เขียนได้ในครั้งเดียว

รูปแบบการจัดองค์กรของชิป
- แบบอ่าน/เขียนเป็น คำ (word) เช่น หลายบิตต่อครั้ง
- แบบ 1 บิตต่อชิป (อ่าน/เขียนทีละบิต)

โครงสร้างการทำงาน (ตัวอย่าง DRAM)
- เซลล์ถูกจัดเป็นตาราง แถว (row) และคอลัมน์ (column)
- เส้นแนวนอน → ใช้เลือกแถว (Select)
- เส้นแนวตั้ง → ใช้รับ/ส่งข้อมูล (Data)

การระบุตำแหน่ง (Addressing)
- ใช้ address lines เพื่อเลือกตำแหน่งข้อมูล
- แบ่งเป็นเลือกแถว (row decoder) และ เลือกคอลัมน์ (column)
- ข้อมูลจะถูกอ่าน/เขียนผ่าน data lines


## Chip Packaging

ชิปหน่วยความจำถูกบรรจุในแพ็กเกจที่มีขา (pins) สำหรับเชื่อมต่อกับภายนอก

EPROM (1M × 8) มี 32 ขา ประกอบด้วย
- address 20 ขา
- data 8 ขา
- power (Vcc) และ ground (Vss)
- chip enable (CE) ใช้เลือกชิป
- program voltage (Vpp) ใช้ตอนเขียนข้อมูล

DRAM (4M × 4)
- data เป็น input/output
- มีขา WE และ OE สำหรับกำหนดอ่าน/เขียน
- ใช้ address แบบ multiplex (row/column) จึงใช้เพียง 11 address pins
- มีขา RAS และ CAS สำหรับเลือกแถวและคอลัมน์
- มีขา NC เพื่อให้จำนวนขาเป็นเลขคู่


## Module Organization

หากชิปหน่วยความจำเก็บข้อมูลได้เพียง 1 บิตต่อคำ (word) จำเป็นต้องใช้หลายชิปตามจำนวนบิตของคำ

ตัวอย่าง:
- หน่วยความจำ 256K × 8 bits ใช้ชิป 8 ตัว (แต่ละตัวเก็บ 1 บิต)
- ใช้ 18-bit address เพื่อระบุตำแหน่งข้อมูล และส่งไปยังทุกชิป

เมื่อต้องการความจุมากขึ้น:
- ต้องจัดชิปเป็น หลายชุด (array)
- เช่น 1M × 8 bits ใช้หลายกลุ่มของชิป

ใช้ address 20 บิต โดย
- 18 บิตแรก → ใช้เลือกตำแหน่งภายในชิป
- 2 บิตสุดท้าย → ใช้เลือกกลุ่มชิปผ่าน chip enable


## Interleaved Memory

หน่วยความจำหลักประกอบด้วยชิป DRAM หลายตัว ซึ่งสามารถรวมกันเป็น memory bank  

การจัดแบบ interleaved memory ทำให้แต่ละ bank ทำงานได้อย่างอิสระ และสามารถอ่านและเขียนพร้อมกันได้หลายคำสั่งตามจำนวน Bank และช่วยเพิ่มความเร็วในการเข้าถึงข้อมูล  

หากเก็บข้อมูลต่อเนื่องไว้คนละ bank จะช่วยให้การอ่านข้อมูลเป็นชุด (block) เร็วขึ้น
