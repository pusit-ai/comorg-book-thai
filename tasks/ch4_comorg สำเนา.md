# บทที่ 4: หน่วยความจำแคช (CHAPTER 4: CACHE MEMORY)

## 4.1 ภาพรวมระบบหน่วยความจำคอมพิวเตอร์ (Computer Memory System Overview)

### คุณลักษณะของระบบหน่วยความจำ (Characteristics of Memory Systems)

หน่วยความจำคอมพิวเตอร์สามารถจำแนกตามคุณลักษณะสำคัญได้ดังนี้:

- **ตำแหน่ง (Location)**  
  ภายในตัวเครื่อง (เช่น Registers, Cache, Main Memory) หรือภายนอก (เช่น Disk, Tape)

- **ความจุ (Capacity)**  
  ระบุเป็นจำนวนไบต์ (Bytes) หรือเวิร์ด (Words)

- **วิธีการเข้าถึง (Access Method)**
  - **Sequential access**: เข้าถึงตามลำดับ  
  - **Direct access**: เข้าถึงโดยตรงไปยังบล็อก  
  - **Random access**: เข้าถึงตำแหน่งใดก็ได้ด้วยเวลาเท่ากัน (เช่น RAM)  
  - **Associative access**: ค้นหาจากเนื้อหาข้อมูล

---

## ลำดับชั้นของหน่วยความจำ (The Memory Hierarchy)

เนื่องจากมีความแตกต่างกันระหว่าง **ความเร็ว ความจุ และราคา** จึงต้องมีการจัดลำดับชั้นของหน่วยความจำ

- ราคาต่อบิตลดลงเมื่อลงไปสู่ชั้นล่าง
- ความจุเพิ่มขึ้นเมื่อลงไปสู่ชั้นล่าง
- เวลาในการเข้าถึงนานขึ้น (ช้าลง) เมื่อลงไปสู่ชั้นล่าง
- ความถี่ในการเข้าถึงโดยหน่วยประมวลผล (CPU) ลดลงเมื่อลงไปสู่ชั้นล่าง

---

# 4.2 หลักการของหน่วยความจำแคช (Cache Memory Principles)

แคชถูกออกแบบมาเพื่อแก้ปัญหาความเร็วที่แตกต่างกันระหว่าง **CPU** และ **หน่วยความจำหลัก (Main Memory)**

การทำงาน:

- เมื่อ **CPU ต้องการข้อมูล** จะตรวจสอบในแคชก่อน

- **Cache Hit**  
  พบข้อมูลในแคช CPU สามารถดึงไปใช้งานได้ทันที

- **Cache Miss**  
  ไม่พบข้อมูลในแคช ต้องดึงข้อมูลจากหน่วยความจำหลักมาเก็บไว้ในแคชหนึ่งบล็อก (Block) แล้วจึงส่งให้ CPU

---

# 4.3 องค์ประกอบของการออกแบบแคช (Elements of Cache Design)

## 1. ฟังก์ชันการจับคู่ (Mapping Function)

เนื่องจากแคชมีขนาดเล็กกว่าหน่วยความจำหลัก จึงต้องมีวิธีการกำหนดว่าข้อมูลจากหน่วยความจำหลักจะไปอยู่ตำแหน่งใดในแคช

- **Direct Mapping**  
  หนึ่งบล็อกในหน่วยความจำหลักวางได้เพียงตำแหน่งเดียวที่กำหนดไว้ในแคช

- **Associative Mapping**  
  หนึ่งบล็อกสามารถวางที่ตำแหน่งใดก็ได้ในแคช

- **Set-Associative Mapping**  
  แคชถูกแบ่งเป็นเซต และบล็อกจะถูกวางในตำแหน่งใดก็ได้ภายในเซตที่กำหนด

---

## 2. อัลกอริทึมการแทนที่ (Replacement Algorithms)

เมื่อแคชเต็มและต้องนำข้อมูลใหม่เข้ามา ต้องเลือกข้อมูลเดิมออก

- **Least Recently Used (LRU)**  
  นำข้อมูลที่ไม่ได้ถูกใช้นานที่สุดออก

- **First-In-First-Out (FIFO)**  
  นำข้อมูลที่เข้ามานานที่สุดออก

---

## 3. นโยบายการเขียน (Write Policy)

- **Write Through**  
  เขียนข้อมูลลงทั้งแคชและหน่วยความจำหลักพร้อมกัน

- **Write Back**  
  เขียนข้อมูลลงแคชก่อน และจะเขียนลงหน่วยความจำหลักเมื่อบล็อกนั้นถูกดึงออกจากแคช

---

# 4.4 การจัดระเบียบแคชของ Pentium 4 (Pentium 4 Cache Organization)

ตัวอย่างการใช้งานจริงในตัวประมวลผล

- ใช้แคชหลายระดับ (**Multi-level Cache**) เช่น  
  - **L1 Cache**  
  - **L2 Cache**

เพื่อเพิ่มความเร็วในการประมวลผลสูงสุด

---

# 4.5 สรุปและคำศัพท์สำคัญ (Key Terms)

- **Hit Ratio**  
  อัตราส่วนการพบข้อมูลในแคช

- **Locality of Reference**  
  หลักการที่ว่าข้อมูลที่ถูกอ้างอิงมักจะเกาะกลุ่มกัน ทั้งในด้านเวลาและตำแหน่ง ซึ่งเป็นปัจจัยสำคัญที่ทำให้ระบบแคชทำงานได้อย่างมีประสิทธิภาพ
