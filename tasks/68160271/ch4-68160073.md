\# บทที่ 4 - หน่วยความจำแคช (Cache Memory)



\## บทนำ



หน่วยความจำของคอมพิวเตอร์แม้จะดูเหมือนเป็นองค์ประกอบที่มีแนวคิดพื้นฐานไม่ซับซ้อน แต่ในความเป็นจริงแล้วถือเป็นองค์ประกอบที่มีความหลากหลายมากที่สุดในระบบคอมพิวเตอร์ ทั้งในด้านชนิดของหน่วยความจำ เทคโนโลยีที่ใช้ โครงสร้างการจัดระเบียบ ประสิทธิภาพการทำงาน และต้นทุนในการผลิต



ไม่มีเทคโนโลยีหน่วยความจำชนิดใดชนิดหนึ่งที่สามารถตอบสนองความต้องการทั้งหมดของระบบคอมพิวเตอร์ได้อย่างเหมาะสม ดังนั้นระบบคอมพิวเตอร์โดยทั่วไปจึงใช้แนวคิดของ \*\*ลำดับชั้นของหน่วยความจำ (Memory Hierarchy)\*\* ซึ่งประกอบด้วยหน่วยความจำหลายระดับที่ทำงานร่วมกัน



\- หน่วยความจำบางส่วนอยู่ภายในระบบและสามารถเข้าถึงได้โดยตรงจากหน่วยประมวลผลกลาง (CPU)

\- หน่วยความจำบางประเภทอยู่ภายนอกและต้องเข้าถึงผ่านโมดูลอินพุต/เอาต์พุต



บทนี้จะเน้นศึกษาหน่วยความจำภายในระบบ โดยเฉพาะองค์ประกอบสำคัญของระบบคอมพิวเตอร์สมัยใหม่ ได้แก่ \*\*หน่วยความจำแคช (Cache Memory)\*\*



\---



\# วัตถุประสงค์การเรียนรู้ของบทนี้



หลังจากศึกษาบทนี้แล้ว ผู้อ่านควรสามารถ



\- อธิบายลักษณะสำคัญของระบบหน่วยความจำคอมพิวเตอร์ได้

\- อธิบายแนวคิดของลำดับชั้นหน่วยความจำ

\- อธิบายหลักการพื้นฐานของหน่วยความจำแคช

\- อธิบายองค์ประกอบสำคัญในการออกแบบแคช

\- แยกความแตกต่างระหว่าง Direct Mapping, Associative Mapping และ Set-Associative Mapping

\- อธิบายเหตุผลในการใช้แคชหลายระดับ

\- วิเคราะห์ผลกระทบด้านประสิทธิภาพของระบบหน่วยความจำหลายระดับ

\# 4.1 ภาพรวมของระบบหน่วยความจำคอมพิวเตอร์ (Computer Memory System Overview)



\## ลักษณะของระบบหน่วยความจำ



เพื่อให้เข้าใจระบบหน่วยความจำได้ง่ายขึ้น เราสามารถจำแนกหน่วยความจำตามลักษณะสำคัญหลายประการ ได้แก่



\- ตำแหน่งของหน่วยความจำ

\- ความจุของหน่วยความจำ

\- วิธีการเข้าถึงข้อมูล

\- ประสิทธิภาพในการทำงาน

\- ลักษณะทางกายภาพของหน่วยความจำ

\- การจัดระเบียบของข้อมูล



การจำแนกหน่วยความจำตามลักษณะเหล่านี้ช่วยให้สามารถวิเคราะห์และออกแบบระบบหน่วยความจำที่เหมาะสมกับความต้องการของระบบคอมพิวเตอร์ได้



\---



\## การจำแนกหน่วยความจำตามตำแหน่ง (Location)



หน่วยความจำสามารถแบ่งตามตำแหน่งได้เป็นสองประเภทหลัก



\- หน่วยความจำภายใน (Internal Memory)

\- หน่วยความจำภายนอก (External Memory)



\### หน่วยความจำภายใน



หน่วยความจำภายในหมายถึงหน่วยความจำที่อยู่ภายในระบบคอมพิวเตอร์และสามารถเข้าถึงได้โดยตรงจาก CPU ตัวอย่างเช่น



\- รีจิสเตอร์ (Registers)

\- หน่วยความจำแคช (Cache)

\- หน่วยความจำหลัก (Main Memory)



หน่วยความจำประเภทนี้มีความเร็วสูง แต่มีความจุจำกัด



\### หน่วยความจำภายนอก



หน่วยความจำภายนอกหมายถึงอุปกรณ์จัดเก็บข้อมูลที่ใช้สำหรับเก็บข้อมูลระยะยาว เช่น



\- ฮาร์ดดิสก์

\- SSD

\- อุปกรณ์จัดเก็บข้อมูลแบบแม่เหล็ก

\- อุปกรณ์จัดเก็บข้อมูลแบบออปติคัล



หน่วยความจำประเภทนี้มีความจุสูงแต่ความเร็วต่ำกว่า



\---

\## การจำแนกหน่วยความจำตามความจุ (Capacity)



\### Overview

\- ความจุของหน่วยความจำ (Capacity) คือปริมาณข้อมูลสูงสุดที่สามารถเก็บได้

\- เป็นปัจจัยสำคัญที่มีผลต่อประสิทธิภาพของระบบ

\- หน่วยความจำที่มีความจุสูงสามารถรองรับโปรแกรมและข้อมูลได้มากขึ้น



\---



\### หน่วยวัดความจุ

\- Bit

\- Byte

\- Kilobyte (KB)

\- Megabyte (MB)

\- Gigabyte (GB)

\- Terabyte (TB)



\---



\### ความสัมพันธ์ของหน่วย

\- 1 Byte = 8 Bits

\- 1 KB = 1024 Bytes

\- 1 MB = 1024 KB

\- 1 GB = 1024 MB

\- 1 TB = 1024 GB



\---



\### อธิบายแต่ละหน่วย

\- Bit

\- หน่วยเล็กที่สุด (0 หรือ 1)



\- Byte

\- ใช้เก็บตัวอักษร 1 ตัว



\- KB

\- ใช้เก็บเอกสารขนาดเล็ก



\- MB

\- ใช้เก็บรูปภาพ เพลง หรือโปรแกรมขนาดเล็ก



\- GB

\- ใช้เก็บวิดีโอ เกม และโปรแกรมขนาดใหญ่



\- TB

\- ใช้ในระบบเก็บข้อมูลขนาดใหญ่ เช่น Server



\---



\### ความสำคัญของ Capacity

\- ส่งผลต่อจำนวนโปรแกรมที่เปิดได้พร้อมกัน

\- ส่งผลต่อความเร็วโดยรวมของระบบ (ถ้า RAM ไม่พอจะช้า)

\- รองรับข้อมูลขนาดใหญ่ เช่น เกม หรือวิดีโอ



\---



\### ตัวอย่างการใช้งาน

\- RAM 8GB - ใช้งานทั่วไป เล่นเกมระดับกลาง

\- RAM 16GB - ทำงานหนัก ตัดต่อ เล่นเกม

\- Storage 1TB - เก็บข้อมูลจำนวนมาก

\# วิธีการเข้าถึงข้อมูล (Access Method)



หน่วยความจำสามารถจำแนกตามวิธีการเข้าถึงข้อมูลได้ดังนี้



\- Sequential Access

\- Direct Access

\- Random Access

\- Associative Access



\## Sequential Access

\- การเข้าถึงข้อมูลแบบลำดับ

\- ต้องอ่านข้อมูลจากต้นไปปลาย

\- ไม่สามารถข้ามไปตำแหน่งที่ต้องการได้ทันที



\### How it works

\- ข้อมูลเรียงเป็นลำดับ

\- เริ่มอ่านจากตำแหน่งแรก

\- ต้องผ่านข้อมูลก่อนหน้าก่อนถึงจุดที่ต้องการ



\### Examples

\- Magnetic Tape

\- Log Files

\- Streaming Data



\### Advantages

\- โครงสร้างง่าย

\- ใช้ทรัพยากรน้อย

\- เหมาะกับงานอ่านต่อเนื่อง



\### Disadvantages

\- ค้นหาช้า

\- ไม่ยืดหยุ่น



\### Use Cases

\- Backup

\- Batch Processing



\## Direct Access

\- การเข้าถึงข้อมูลแบบระบุตำแหน่ง

\- สามารถเข้าถึง block ที่ต้องการได้



\### How it works

\- ข้อมูลแบ่งเป็น block

\- แต่ละ block มี address

\- สามารถเลือกอ่าน block ได้



\### Examples

\- HDD

\- SSD

\- CD DVD



\### Advantages

\- เร็วกว่า Sequential

\- เลือกตำแหน่งได้



\### Disadvantages

\- ต้องรู้ตำแหน่งข้อมูล

\- มี overhead



\### Use Cases

\- File System

\- Database



\## Random Access

\- การเข้าถึงข้อมูลได้ทันทีทุกตำแหน่ง

\- ใช้ address ในการเข้าถึง



\### How it works

\- ใช้ memory address

\- CPU เข้าถึงข้อมูลโดยตรง

\- ไม่ต้องเรียงลำดับ



\### Examples

\- RAM

\- Cache



\### Advantages

\- เร็วมาก

\- เข้าถึงทันที



\### Disadvantages

\- ราคาแพง

\- ข้อมูลหายเมื่อปิดเครื่อง



\### Use Cases

\- โปรแกรมที่กำลังทำงาน

\- ระบบปฏิบัติการ

\- เกม



\## Associative Access

\- การเข้าถึงข้อมูลโดยใช้เนื้อหา

\- ไม่ต้องใช้ address



\### How it works

\- ใช้ CAM

\- ค้นหาแบบขนาน

\- เปรียบเทียบหลายตำแหน่งพร้อมกัน



\### Examples

\- CAM

\- Cache Lookup

\- Search Engine



\### Advantages

\- ค้นหาเร็วมาก

\- ไม่ต้องรู้ตำแหน่ง



\### Disadvantages

\- ซับซ้อน

\- ราคาแพง

\- ใช้พลังงานสูง



\### Use Cases

\- Network

\- AI

\- Database Search



\# ประสิทธิภาพของหน่วยความจำ (Performance)



\- ประสิทธิภาพของหน่วยความจำ คือความสามารถในการเข้าถึงและส่งข้อมูลได้รวดเร็วและมีประสิทธิภาพ

\- เป็นปัจจัยสำคัญที่มีผลต่อความเร็วของคอมพิวเตอร์

\- วัดได้จากหลายค่า เช่น เวลาเข้าถึง และอัตราการส่งข้อมูล



\## Key Metrics

\- Access Time

\- Memory Cycle Time

\- Transfer Rate



\## Access Time

\- เวลาที่ใช้ในการเข้าถึงข้อมูลในหน่วยความจำ

\- นับตั้งแต่เริ่มร้องขอข้อมูลจนได้รับข้อมูล

\- ยิ่งน้อยยิ่งดี



\## Memory Cycle Time

\- เวลาที่หน่วยความจำใช้ในการทำงาน 1 รอบ

\- รวมเวลาอ่านและเตรียมพร้อมสำหรับรอบถัดไป

\- มักมากกว่า Access Time



\## Transfer Rate

\- อัตราการส่งข้อมูลระหว่างหน่วยความจำกับ CPU

\- วัดเป็น byte ต่อวินาที

\- ยิ่งสูงยิ่งดี

\# ลำดับชั้นของหน่วยความจำ (Memory Hierarchy)



\- Memory Hierarchy คือการจัดลำดับหน่วยความจำตามความเร็ว ราคา และความจุ

\- ถูกออกแบบมาเพื่อให้ระบบทำงานได้เร็วและคุ้มค่า

\- หน่วยความจำที่เร็วจะมีขนาดเล็กและราคาแพง

\- หน่วยความจำที่ช้าจะมีขนาดใหญ่และราคาถูก



\## Levels of Memory

\- Register

\- Cache Memory

\- Main Memory (RAM)

\- Secondary Storage

\- Tertiary Storage



\## Register

\- อยู่ใน CPU

\- เร็วที่สุด

\- ความจุน้อยมาก

\- ใช้เก็บข้อมูลชั่วคราวที่ CPU กำลังใช้งาน



\## Cache Memory

\- อยู่ใกล้ CPU

\- เร็วมาก

\- เก็บข้อมูลที่ใช้งานบ่อย

\- ช่วยลดเวลาในการเข้าถึง RAM



\## Main Memory

\- RAM

\- ใช้เก็บโปรแกรมและข้อมูลที่กำลังทำงาน

\- ความเร็วปานกลาง



\## Secondary Storage

\- HDD

\- SSD

\- ความจุสูง

\- ใช้เก็บข้อมูลถาวร



\## Tertiary Storage

\- Tape

\- Cloud Storage

\- ใช้สำหรับสำรองข้อมูลระยะยาว

\- เข้าถึงช้ามาก



\## Characteristics

\- ความเร็วลดลงตามลำดับชั้น

\- ความจุเพิ่มขึ้นตามลำดับชั้น

\- ราคาต่อหน่วยลดลงตามลำดับชั้น



\## Why Important

\- ช่วยให้คอมพิวเตอร์ทำงานเร็วขึ้น

\- ลดต้นทุน

\- เพิ่มประสิทธิภาพโดยรวมของระบบ



\## Summary

\- บนสุด เร็วสุด แพงสุด ความจุน้อย

\- ล่างสุด ช้าสุด ถูกสุด ความจุเยอะ

\- ใช้ร่วมกันเพื่อให้ระบบสมดุล

\# 4.2 หลักการของหน่วยความจำแคช (Cache Memory Principles)



หน่วยความจำแคชถูกออกแบบมาเพื่อแก้ปัญหาความแตกต่างของความเร็วระหว่าง CPU และหน่วยความจำหลัก



แม้ว่า CPU จะสามารถประมวลผลคำสั่งได้อย่างรวดเร็ว แต่การเข้าถึงข้อมูลจากหน่วยความจำหลักมักใช้เวลานานกว่า ดังนั้นจึงมีการเพิ่มหน่วยความจำขนาดเล็กที่มีความเร็วสูงระหว่าง CPU และหน่วยความจำหลัก



หน่วยความจำนี้เรียกว่า \*\*Cache Memory\*\*



แคชจะเก็บข้อมูลที่ถูกใช้งานบ่อยไว้ใกล้กับ CPU เพื่อให้สามารถเข้าถึงได้รวดเร็วขึ้น



\---



\# แนวคิด Locality of Reference



\- Locality of Reference คือหลักการสำคัญที่อธิบายพฤติกรรมการเข้าถึงข้อมูลของโปรแกรม

\- ใช้ในการออกแบบ Cache Memory ให้ทำงานได้มีประสิทธิภาพ

\- แนวคิดคือ โปรแกรมมักจะใช้ข้อมูลซ้ำ ๆ หรือใช้ข้อมูลที่อยู่ใกล้กัน



\---



\## ประเภทของ Locality

\- Temporal Locality

\- Spatial Locality



\---



\## Temporal Locality

\- หากข้อมูลหนึ่งถูกใช้งานแล้ว

\- มีโอกาสสูงที่จะถูกใช้งานอีกครั้งในเวลาอันใกล้



\### หลักการ

\- ข้อมูลที่ถูกใช้งานล่าสุดมีความสำคัญ

\- ควรเก็บไว้ใน Cache เพื่อใช้งานซ้ำ



\### ตัวอย่าง

\- ตัวแปรในลูป

\- คำสั่งที่ถูกเรียกซ้ำในฟังก์ชัน

\- การใช้งานตัวแปรเดิมหลายครั้ง



\### ประโยชน์

\- ลดเวลาเข้าถึงข้อมูล

\- เพิ่มประสิทธิภาพของ Cache



\---



\## Spatial Locality

\- หากมีการเข้าถึงข้อมูลตำแหน่งหนึ่ง

\- มีแนวโน้มว่าจะเข้าถึงตำแหน่งใกล้เคียง



\### หลักการ

\- ข้อมูลที่อยู่ติดกันมักถูกใช้งานต่อเนื่อง

\- Cache จะโหลดข้อมูลเป็น block



\### ตัวอย่าง

\- การอ่านคำสั่งโปรแกรมต่อเนื่อง

\- การอ่าน array

\- การวนลูปผ่านข้อมูลที่เรียงกัน



\### ประโยชน์

\- ลดจำนวนครั้งในการเข้าถึงหน่วยความจำหลัก

\- เพิ่มความเร็วในการทำงาน



\---



\## ความสัมพันธ์กับ Cache

\- Cache จะเก็บข้อมูลที่ใช้บ่อย (Temporal)

\- Cache จะโหลดข้อมูลเป็น block (Spatial)

\- ช่วยลด Cache Miss



\---



\# Cache Hit และ Cache Miss



\- เมื่อ CPU ต้องการข้อมูล จะตรวจสอบที่ Cache ก่อน

\- จะเกิดได้ 2 กรณี

\- Cache Hit

\- Cache Miss

\- ทั้งสองกรณีมีผลต่อความเร็วของระบบโดยตรง



\---



\## Cache Hit

\- เกิดขึ้นเมื่อข้อมูลที่ CPU ต้องการอยู่ใน Cache แล้ว

\- CPU สามารถเข้าถึงข้อมูลได้ทันที



\### ขั้นตอน

\- CPU ขอข้อมูล

\- Cache พบข้อมูล

\- ส่งข้อมูลให้ CPU ทันที



\### ข้อดี

\- เร็วมาก

\- ลดเวลาในการเข้าถึงข้อมูล

\- เพิ่มประสิทธิภาพของระบบ



\### ตัวอย่าง

\- การใช้ตัวแปรซ้ำในลูป

\- การเรียกฟังก์ชันเดิมหลายครั้ง



\---



\## Cache Miss

\- เกิดขึ้นเมื่อข้อมูลที่ต้องการไม่อยู่ใน Cache



\### ขั้นตอน

\- CPU ขอข้อมูล

\- Cache ไม่พบข้อมูล

\- ไปค้นหาที่ Main Memory (RAM)

\- นำข้อมูลเข้าสู่ Cache

\- ส่งข้อมูลให้ CPU



\### ข้อเสีย

\- ใช้เวลานาน

\- ทำให้ระบบช้าลง



\---



\## ประเภทของ Cache Miss

\- Compulsory Miss

\- Capacity Miss

\- Conflict Miss



\---



\## Compulsory Miss

\- เกิดขึ้นครั้งแรกที่มีการเข้าถึงข้อมูล

\- ยังไม่เคยโหลดเข้ามาใน Cache



\---



\## Capacity Miss

\- Cache มีขนาดไม่พอ

\- ข้อมูลบางส่วนถูกแทนที่ออกไป



\---



\## Conflict Miss

\- เกิดจากการแมปข้อมูลชนกัน

\- พบมากใน Direct Mapping



\---



\## Hit Rate และ Miss Rate

\- Hit Rate คือเปอร์เซ็นต์ที่เกิด Cache Hit

\- Miss Rate คือเปอร์เซ็นต์ที่เกิด Cache Miss



\### สูตร

\- Hit Rate = จำนวน Hit / จำนวนการเข้าถึงทั้งหมด

\- Miss Rate = จำนวน Miss / จำนวนการเข้าถึงทั้งหมด



\---



\## ความสำคัญ

\- Hit Rate สูง = ระบบเร็ว

\- Miss Rate สูง = ระบบช้า

\- เป้าหมายคือเพิ่ม Hit และลด Miss



\---



\# 4.3 เทคนิคการแมปข้อมูลในหน่วยความจำแคช (Cache Mapping Techniques)



\## Overview

\- เมื่อมีการนำข้อมูลจากหน่วยความจำหลักเข้าสู่แคช ต้องมีการกำหนดตำแหน่งจัดเก็บ

\- วิธีนี้เรียกว่า Mapping Technique

\- มี 3 ประเภทหลัก

\- Direct Mapping

\- Associative Mapping

\- Set-Associative Mapping



\---



\## Direct Mapping

\- เป็นวิธีที่ง่ายที่สุด



\### หลักการ

\- Main Memory Block → Cache Line เดียว



\### สูตร

\- Cache Line = (Main Memory Block Number) mod (Number of Cache Lines)



\### ข้อดี

\- ฮาร์ดแวร์เรียบง่าย

\- ค่าใช้จ่ายต่ำ



\### ข้อเสีย

\- มีโอกาสเกิด Conflict Miss สูง



\---



\## Associative Mapping

\- บล็อกใดก็ได้สามารถไปอยู่ตำแหน่งใดก็ได้ในแคช



\### ข้อดี

\- ลดปัญหา Conflict Miss



\### ข้อเสีย

\- ฮาร์ดแวร์ซับซ้อน

\- ต้นทุนสูง



\---



\## Set-Associative Mapping

\- ผสมระหว่าง Direct และ Associative



\### หลักการ

\- แบ่งแคชออกเป็นหลาย Set

\- แต่ละ Set มีหลายบล็อก



\### ข้อดี

\- ลด Conflict Miss

\- ประสิทธิภาพดีกว่า Direct Mapping



\### ข้อเสีย

\- ซับซ้อนกว่า Direct Mapping



\---



\# 4.4 อัลกอริทึมการแทนที่ข้อมูลในแคช



\## Overview

\- เมื่อแคชเต็ม ต้องเลือกข้อมูลบางส่วนออก

\- ใช้อัลกอริทึมในการตัดสินใจ



\## ประเภท

\- Least Recently Used (LRU)

\- First In First Out (FIFO)

\- Least Frequently Used (LFU)

\- Random Replacement



\---



\## LRU

\- ลบข้อมูลที่ไม่ได้ใช้นานที่สุด



\## FIFO

\- ลบข้อมูลที่เข้ามาก่อน



\## LFU

\- ลบข้อมูลที่ถูกใช้น้อยที่สุด



\## Random

\- สุ่มลบข้อมูล



\---



\# 4.5 นโยบายการเขียนข้อมูลในแคช



\## Overview

\- กำหนดวิธีเขียนข้อมูลลงแคชและหน่วยความจำหลัก



\## ประเภท

\- Write-Through

\- Write-Back



\---



\## Write-Through

\- เขียนลงแคชและหน่วยความจำหลักพร้อมกัน



\### ข้อดี

\- ข้อมูลตรงกันเสมอ



\### ข้อเสีย

\- ช้ากว่า



\---



\## Write-Back

\- เขียนเฉพาะในแคชก่อน



\### หลักการ

\- อัปเดตหน่วยความจำหลักเมื่อมีการแทนที่ข้อมูล



\### ข้อดี

\- เร็วกว่า



\### ข้อเสีย

\- ข้อมูลอาจไม่ตรงกันทันที



\---



\# 4.6 แคชหลายระดับ (Multilevel Cache)



\## Overview

\- ใช้แคชหลายระดับเพื่อเพิ่มประสิทธิภาพ



\## ระดับ

\- L1 Cache

\- L2 Cache

\- L3 Cache



\---



\## L1 Cache

\- อยู่ใกล้ CPU มากที่สุด (มักอยู่ในตัว CPU เลย)

\- เร็วที่สุดในบรรดาแคชทั้งหมด

\- ขนาดเล็กมาก (เช่น 32KB - 128KB ต่อคอร์)

\- แบ่งเป็น 2 ส่วน

\- Instruction Cache (เก็บคำสั่ง)

\- Data Cache (เก็บข้อมูล)

\- ใช้เก็บข้อมูลที่ CPU ใช้งานทันที

\- มี latency ต่ำมาก (เข้าถึงแทบจะทันที)

\- ถ้าเกิด miss จะไปค้นหาใน L2 ต่อ



\---



\## L2 Cache

\- อยู่ถัดจาก L1 (อาจอยู่ใน CPU หรือใกล้ CPU)

\- ขนาดใหญ่กว่า L1 (เช่น 256KB - 1MB ต่อคอร์)

\- ความเร็วช้ากว่า L1 เล็กน้อย

\- ทำหน้าที่เป็นตัวสำรองของ L1

\- เก็บข้อมูลที่อาจถูกใช้งานต่อ

\- ลดจำนวนครั้งที่ต้องไป RAM

\- latency ปานกลาง

\- ถ้า L2 miss จะไป L3



\---



\## L3 Cache

\- ขนาดใหญ่ที่สุดในระบบแคช (เช่น 2MB - 64MB)

\- ใช้ร่วมกันระหว่างหลายคอร์ (Shared Cache)

\- ความเร็วช้ากว่า L2 แต่ยังเร็วกว่า RAM มาก

\- ช่วยลดการเข้าถึงหน่วยความจำหลัก (RAM)

\- เหมาะกับงาน multi-core เช่น เกม หรือโปรแกรมหนัก

\- latency สูงกว่า L1 และ L2 แต่ยังถือว่าเร็ว

\- ถ้า L3 miss จะไป RAM
