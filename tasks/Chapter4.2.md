## 4.2 หลักการของหน่วยความจำแคช (Cache Memory Principles)

Cache Memory ถูกออกแบบมาเพื่อรวมข้อดีของ:

* หน่วยความจำที่เร็ว (แต่แพง)
* หน่วยความจำที่ใหญ่ (แต่ช้า)

### หลักการทำงานของ Cache

* Cache จะเก็บ “สำเนา” ของข้อมูลบางส่วนจาก Main Memory
* เมื่อ CPU ต้องการข้อมูล:

  * ถ้าพบใน Cache → เรียกว่า **Cache Hit** → ดึงข้อมูลได้เร็ว
  * ถ้าไม่พบ → เรียกว่า **Cache Miss** → ต้องไปดึงจาก Main Memory แล้วนำมาเก็บใน Cache

### แนวคิดสำคัญ

* ใช้หลัก **Locality of Reference**
* เมื่อโหลดข้อมูล 1 บล็อกมา มักจะถูกใช้งานซ้ำในอนาคต

### โครงสร้าง Cache

* แบ่งข้อมูลเป็น “บล็อก” (Block)
* Cache จะเก็บเป็น “ไลน์” (Line)
* แต่ละไลน์จะมี:

  * ข้อมูล (Data)
  * Tag (ใช้ระบุว่ามาจากตำแหน่งไหนใน Main Memory)
  * Control bits

### หลายระดับของ Cache

* L1 Cache → เร็วที่สุด ขนาดเล็ก
* L2 Cache → ช้าลง แต่ใหญ่ขึ้น
* L3 Cache → ช้าที่สุดในกลุ่ม Cache แต่ใหญ่ที่สุด

### ประโยชน์ของ Cache

* ลดเวลาในการเข้าถึงหน่วยความจำ
* เพิ่มประสิทธิภาพของ CPU
* ทำให้ระบบทำงานเร็วขึ้นโดยรวม
