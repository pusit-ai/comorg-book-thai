# **บทที่ 7: อินพุต/เอาต์พุต (Input/Output)** #
นอกเหนือจากตัวประมวลผลและหน่วยความจำ องค์ประกอบหลักที่สามของระบบคอมพิวเตอร์คือชุดของโมดูลอินพุต/เอาต์พุต (I/O)
 ซึ่งทำหน้าที่เป็นตัวกลางในการสื่อสารระหว่างคอมพิวเตอร์กับโลกภายนอก โดยจัดการความแตกต่างของรูปแบบข้อมูลและความเร็วระหว่างอุปกรณ์ต่อพ่วงและระบบบัสภายใน

สาเหตุที่เราไม่เชื่อมต่ออุปกรณ์ต่อพ่วงเข้ากับระบบบัสโดยตรงมีดังนี้:
*   อุปกรณ์ต่อพ่วงมีรูปแบบข้อมูลและความยาวของคำที่แตกต่างจากคอมพิวเตอร์ที่เชื่อมต่ออยู่
*   ดังนั้นจึงจำเป็นต้องมีโมดูล I/O ซึ่งมีหน้าที่หลักสองประการ ได้แก่:
    *   เชื่อมต่อกับตัวประมวลผลและหน่วยความจำผ่านระบบบัสหรือสวิตช์ส่วนกลาง
    *   เชื่อมต่อกับอุปกรณ์ต่อพ่วงอย่างน้อยหนึ่งตัวด้วยลิงก์ข้อมูลที่ปรับแต่งมาโดยเฉพาะ

---

### **7.1 อุปกรณ์ภายนอก (External Devices)**
การทำงานของ I/O สำเร็จได้ผ่านอุปกรณ์ภายนอกหลากหลายประเภทที่ใช้ในการแลกเปลี่ยนข้อมูลระหว่างสภาพแวดล้อมภายนอกและคอมพิวเตอร์ อุปกรณ์ภายนอกจะเชื่อมต่อกับคอมพิวเตอร์ผ่านลิงก์ไปยังโมดูล I/O ลิงก์นี้ใช้เพื่อแลกเปลี่ยนสัญญาณควบคุม สถานะ และข้อมูลระหว่างโมดูล I/O และอุปกรณ์ภายนอก อุปกรณ์ภายนอกที่เชื่อมต่อกับโมดูล I/O มักถูกเรียกว่า **อุปกรณ์ต่อพ่วง (Peripheral device)**

เราสามารถแบ่งประเภทอุปกรณ์ภายนอกออกเป็น 3 ประเภทใหญ่ๆ ได้แก่:
*   **มนุษย์อ่านได้ (Human readable):** เหมาะสำหรับการสื่อสารกับผู้ใช้คอมพิวเตอร์ เช่น หน้าจอแสดงผล (VDTs) และเครื่องพิมพ์
*   **เครื่องจักรอ่านได้ (Machine readable):** เหมาะสำหรับการสื่อสารกับอุปกรณ์ เช่น ระบบจานแม่เหล็กและเทปแม่เหล็ก รวมถึงเซนเซอร์และตัวกระตุ้น (Actuators) ที่ใช้ในงานหุ่นยนต์
*   **การสื่อสาร (Communication):** เหมาะสำหรับการสื่อสารกับอุปกรณ์ระยะไกล ซึ่งอาจเป็นอุปกรณ์ที่มนุษย์อ่านได้ อุปกรณ์ที่เครื่องจักรอ่านได้ หรือแม้แต่คอมพิวเตอร์เครื่องอื่น

อินเทอร์เฟซไปยังโมดูล I/O จะอยู่ในรูปแบบของสัญญาณควบคุม ข้อมูล และสถานะ สัญญาณควบคุมจะกำหนดหน้าที่ที่อุปกรณ์จะทำ เช่น ส่งข้อมูลไปยังโมดูล I/O (INPUT หรือ READ) หรือรับข้อมูลจากโมดูล I/O (OUTPUT หรือ WRITE) ข้อมูลจะอยู่ในรูปแบบของชุดบิตที่จะส่งหรือรับ ส่วนสัญญาณสถานะจะบ่งบอกสภาวะของอุปกรณ์ เช่น READY/NOT-READY เพื่อแสดงว่าอุปกรณ์พร้อมสำหรับการถ่ายโอนข้อมูลหรือไม่

---

### **7.2 โมดูล I/O (I/O Modules)**
#### **หน้าที่ของโมดูล (Module Function)**
หน้าที่หลักหรือข้อกำหนดสำหรับโมดูล I/O แบ่งออกเป็นหมวดหมู่ดังนี้:
*   **การควบคุมและจังหวะเวลา (Control and timing):** เพื่อประสานการไหลของทราฟฟิกระหว่างทรัพยากรภายในและอุปกรณ์ภายนอก
*   **การสื่อสารกับตัวประมวลผล (Processor communication):** รวมถึงการถอดรหัสคำสั่ง (Command decoding), การแลกเปลี่ยนข้อมูล (Data exchange), การรายงานสถานะ (Status reporting) และการรับรู้อยู่ (Address recognition)
*   **การสื่อสารกับอุปกรณ์ (Device communication):** การแลกเปลี่ยนคำสั่ง ข้อมูลสถานะ และข้อมูลกับอุปกรณ์ต่อพ่วง
*   **การบัฟเฟอร์ข้อมูล (Data buffering):** เนื่องจากอัตราการถ่ายโอนข้อมูลของหน่วยความจำหลักหรือตัวประมวลผลมักจะสูงกว่าอัตราของอุปกรณ์ต่อพ่วงอย่างมาก ข้อมูลจึงต้องถูกพักไว้ในโมดูล I/O
*   **การตรวจจับข้อผิดพลาด (Error detection):** รับผิดชอบในการตรวจจับและรายงานข้อผิดพลาดไปยังตัวประมวลผล ทั้งข้อผิดพลาดทางกลไกและไฟฟ้า (เช่น กระดาษติด) หรือการเปลี่ยนแปลงของรูปแบบบิตระหว่างการส่ง

#### **โครงสร้างของโมดูล I/O (I/O Module Structure)**
โมดูล I/O เชื่อมต่อกับคอมพิวเตอร์ผ่านชุดของสายสัญญาณ (เช่น ระบบบัส) ข้อมูลที่ถ่ายโอนจะถูกบัฟเฟอร์ใน **เรจิสเตอร์ข้อมูล (Data registers)** อย่างน้อยหนึ่งตัว นอกจากนี้ยังอาจมี **เรจิสเตอร์สถานะ (Status registers)** เพื่อให้ข้อมูลสถานะปัจจุบัน และอาจทำหน้าที่เป็น **เรจิสเตอร์ควบคุม (Control register)** เพื่อรับข้อมูลการควบคุมโดยละเอียดจากตัวประมวลผล

---

### **7.3 โปรแกรม I/O (Programmed I/O)**
เมื่อตัวประมวลผลกำลังรันโปรแกรมและพบคำสั่งที่เกี่ยวข้องกับ I/O มันจะดำเนินการคำสั่งนั้นโดยส่งคำสั่งไปยังโมดูล I/O ที่เหมาะสม ในกรณีของโปรแกรม I/O โมดูล I/O จะดำเนินการตามที่ร้องขอและตั้งค่าบิตที่เหมาะสมในเรจิสเตอร์สถานะ I/O แต่จะไม่ดำเนินการใดๆ เพิ่มเติมเพื่อแจ้งเตือนตัวประมวลผล โดยเฉพาะอย่างยิ่ง มันจะไม่ขัดจังหวะ (Interrupt) ตัวประมวลผล

ดังนั้นจึงเป็นหน้าที่ของตัวประมวลผลที่จะต้องตรวจสอบสถานะของโมดูล I/O เป็นระยะๆ จนกว่าจะพบว่าการดำเนินการเสร็จสิ้น ข้อเสียหลักของเทคนิคนี้คือเป็นกระบวนการที่ใช้เวลานานและทำให้ตัวประมวลผลต้องยุ่งอยู่กับการตรวจสอบสถานะโดยไม่จำเป็น

#### **คำสั่ง I/O (I/O Instructions)**
รูปแบบของคำสั่งขึ้นอยู่กับวิธีระบุที่อยู่ของอุปกรณ์ภายนอก:
*   **Memory-mapped I/O:** อุปกรณ์ I/O และหน่วยความจำใช้พื้นที่ที่อยู่เดียวกัน ตัวประมวลผลมองเรจิสเตอร์สถานะและข้อมูลของโมดูล I/O เป็นตำแหน่งหน่วยความจำ และใช้คำสั่งเข้าถึงหน่วยความจำปกติในการทำ I/O
*   **Isolated I/O:** พอร์ต I/O ถูกแยกออกจากพื้นที่ที่อยู่หน่วยความจำ จำเป็นต้องมีคำสั่งเฉพาะสำหรับ I/O (เช่น IN, OUT)

---

### **7.4 อินเทอร์รัปต์ I/O (Interrupt-Driven I/O)**
ปัญหาของโปรแกรม I/O คือตัวประมวลผลต้องรอนานเพื่อให้โมดูล I/O พร้อม ทางเลือกอื่นคือตัวประมวลผลส่งคำสั่ง I/O ไปยังโมดูลแล้วไปทำงานอื่นที่มีประโยชน์ต่อ โมดูล I/O จะทำการขัดจังหวะ (Interrupt) ตัวประมวลผลเพื่อขอรับบริการเมื่อพร้อมแลกเปลี่ยนข้อมูล

#### **การประมวลผลอินเทอร์รัปต์ (Interrupt Processing)**
เมื่ออุปกรณ์ I/O ทำการดำเนินการเสร็จสิ้น จะมีลำดับเหตุการณ์ดังนี้:
1.  อุปกรณ์ส่งสัญญาณอินเทอร์รัปต์ไปยังตัวประมวลผล
2.  ตัวประมวลผลทำงานปัจจุบันให้เสร็จสิ้นก่อนตอบสนอง
3.  ตัวประมวลผลตรวจสอบอินเทอร์รัปต์และส่งสัญญาณตอบรับ (Acknowledgment)
4.  ตัวประมวลผลบันทึกข้อมูลที่จำเป็นเพื่อกลับมาทำงานเดิมได้ (เช่น Program Counter และ PSW) ลงในสแตก
5.  ตัวประมวลผลโหลดค่า Program Counter ใหม่ตามที่อยู่อินเทอร์รัปต์ เพื่อเริ่มทำงานในโปรแกรมจัดการอินเทอร์รัปต์ (Interrupt handler)
6.  โปรแกรมจัดการอินเทอร์รัปต์ประมวลผล I/O
7.  เมื่อเสร็จสิ้น ค่าเรจิสเตอร์ที่บันทึกไว้จะถูกดึงกลับมา และตัวประมวลผลจะกลับไปทำงานเดิมที่ถูกขัดจังหวะไว้

---

### **7.5 การเข้าถึงหน่วยความจำโดยตรง (Direct Memory Access - DMA)**
#### **หน้าที่ของ DMA (DMA Function)**
DMA เกี่ยวข้องกับโมดูลเพิ่มเติมบนระบบบัส โมดูล DMA สามารถเลียนแบบตัวประมวลผลและเข้าควบคุมระบบเพื่อถ่ายโอนข้อมูลระหว่างหน่วยความจำและอุปกรณ์ I/O โดยตรง เทคนิคนี้มักเรียกว่า **การขโมยไซเคิล (Cycle stealing)** เนื่องจากโมดูล DMA จะบังคับให้ตัวประมวลผลหยุดทำงานชั่วคราวเพื่อใช้บัส

เมื่อตัวประมวลผลต้องการอ่านหรือเขียนบล็อกข้อมูล มันจะส่งข้อมูลต่อไปนี้ไปยังโมดูล DMA:
*   คำสั่งอ่านหรือเขียน
*   ที่อยู่ของอุปกรณ์ I/O ที่เกี่ยวข้อง
*   ตำแหน่งเริ่มต้นในหน่วยความจำที่ต้องการอ่านหรือเขียน
*   จำนวนคำที่ต้องการถ่ายโอน

หลังจากนั้นตัวประมวลผลจะไปทำงานอื่นต่อ โดยปล่อยให้โมดูล DMA จัดการการถ่ายโอนข้อมูลทั้งหมดทีละคำเข้าสู่หน่วยความจำโดยตรง เมื่อเสร็จสิ้น โมดูล DMA จะส่งสัญญาณอินเทอร์รัปต์ไปยังตัวประมวลผล

---

### **7.6 การเข้าถึงแคชโดยตรง (Direct Cache Access - DCA)**
แม้ว่า DMA จะมีประสิทธิภาพ แต่สำหรับอัตราข้อมูลที่เพิ่มขึ้นอย่างรวดเร็วของเครือข่าย (เช่น 10-Gbps หรือ 100-Gbps Ethernet) DMA ไม่สามารถขยายขนาดเพื่อรองรับความต้องการได้

#### **Intel Direct Data I/O (DDIO)**
Intel ได้นำ DCA มาใช้ในรูปแบบของ DDIO ซึ่งมีอยู่ในตระกูลตัวประมวลผล Xeon E5 ในการทำงานปกติของ DMA ข้อมูลที่รับเข้ามาจะถูกเขียนลงในหน่วยความจำหลัก ซึ่งอาจทำให้เกิดคอขวด แต่ด้วย DDIO ข้อมูลที่รับเข้าจากโมดูลควบคุม I/O จะสามารถถูกนำไปวางไว้ในแคชระดับสุดท้าย (Last-level cache หรือ L3) ได้โดยตรง สิ่งนี้ทำให้ตัวประมวลผลสามารถอ่านข้อมูลจากแคชได้ทันทีโดยไม่ต้องรออ่านจากหน่วยความจำหลัก ซึ่งช่วยเพิ่มประสิทธิภาพอย่างมากทั้งข้อมูลขาเข้าและขาออก

---

### **7.7 ช่องสัญญาณและตัวประมวลผล I/O (I/O Channels and Processors)**
วิวัฒนาการของหน้าที่ I/O สามารถสรุปได้ดังนี้:
1.  CPU ควบคุมอุปกรณ์ต่อพ่วงโดยตรง
2.  เพิ่มโมดูล I/O หรือตัวควบคุม โดย CPU ใช้โปรแกรม I/O แบบไม่มีอินเทอร์รัปต์
3.  ใช้การกำหนดค่าเดิมแต่เพิ่มอินเทอร์รัปต์เพื่อเพิ่มประสิทธิภาพ
4.  โมดูล I/O ได้รับการเข้าถึงหน่วยความจำโดยตรงผ่าน DMA
5.  โมดูล I/O ถูกปรับปรุงให้เป็นตัวประมวลผลในตัวมันเอง พร้อมชุดคำสั่งเฉพาะสำหรับ I/O เรียกว่า **ช่องสัญญาณ I/O (I/O Channel)**
6.  โมดูล I/O มีหน่วยความจำในตัว และทำหน้าที่เสมือนคอมพิวเตอร์แยกส่วนเพื่อจัดการอุปกรณ์จำนวนมาก

---

### **7.8 มาตรฐานการเชื่อมต่อภายนอก (External Interconnection Standards)**
มาตรฐานที่ใช้กันอย่างแพร่หลาย ได้แก่:
*   **USB (Universal Serial Bus):** อินเทอร์เฟซเริ่มต้นสำหรับอุปกรณ์ความเร็วต่ำและยังใช้กับ I/O ความเร็วสูง เช่น เครื่องพิมพ์และดิสก์ไดรฟ์
*   **Ethernet:** มาตรฐานสำหรับระบบเครือข่ายท้องถิ่น (LAN) ปัจจุบันมีความเร็วสูงสุดถึง 100 Gbps
*   **Wi-Fi:** เทคโนโลยีการเข้าถึงอินเทอร์เน็ตไร้สายที่สำคัญทั้งในบ้านและองค์กร

---

### **7.9 โครงสร้าง I/O ของ IBM zEnterprise EC12**
ระบบเมนเฟรม zEnterprise EC12 มีระบบย่อย I/O เฉพาะที่จัดการการดำเนินการ I/O ทั้งหมด เพื่อลดภาระการประมวลผลและหน่วยความจำจากตัวประมวลผลหลัก
*   **System Assist Processor (SAP):** เป็นตัวประมวลผลหลักที่กำหนดค่าสำหรับการทำงาน I/O เพื่อจัดการคิวและช่องสัญญาณ
*   **Subchannels (ช่องสัญญาณย่อย):** ปรากฏต่อโปรแกรมเป็นอุปกรณ์เชิงตรรกะที่ประกอบด้วยข้อมูลที่จำเป็นสำหรับการดำเนินการ I/O

---

### **7.10 คำสำคัญ, คำถามทบทวน และโจทย์ปัญหา**
ส่วนนี้ประกอบด้วยคำสำคัญที่สำคัญ เช่น **Direct Memory Access (DMA), Interrupt, I/O module, Programmed I/O** และคำถามเพื่อทดสอบความเข้าใจ เช่น ความแตกต่างระหว่าง Memory-mapped I/O และ Isolated I/O รวมถึงวิธีการที่ตัวประมวลผลระบุอุปกรณ์ที่ขัดจังหวะ
