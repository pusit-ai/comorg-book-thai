## วัตถุประสงค์การเรียนรู้

หลังจากศึกษาบทนี้แล้ว ผู้เรียนควรสามารถ:

- นำเสนอภาพรวมของคุณลักษณะหลักของระบบหน่วยความจำคอมพิวเตอร์ และการใช้ลำดับชั้นของหน่วยความจำได้
- อธิบายแนวคิดพื้นฐานและจุดมุ่งหมายของหน่วยความจำแคชได้
- อภิปรายองค์ประกอบสำคัญของการออกแบบแคชได้
- แยกแยะความแตกต่างระหว่างการแมปปิ้งแบบตรง (direct mapping) การแมปปิ้งแบบเชื่อมโยง (associative mapping) และการแมปปิ้งแบบเซตเชื่อมโยง (set-associative mapping) ได้
- อธิบายเหตุผลในการใช้แคชหลายระดับได้
- เข้าใจผลกระทบต่อประสิทธิภาพของหน่วยความจำหลายระดับได้

## 4.1 ภาพรวมของระบบหน่วยความจำคอมพิวเตอร์

แม้ว่าจะดูเรียบง่ายในแนวคิด แต่หน่วยความจำคอมพิวเตอร์กลับมีความหลากหลายในด้านประเภท เทคโนโลยี การจัดระเบียบ ประสิทธิภาพ และต้นทุนมากที่สุดเมื่อเทียบกับคุณลักษณะอื่น ๆ ของระบบคอมพิวเตอร์ ไม่มีเทคโนโลยีเดียวที่ตอบสนองความต้องการหน่วยความจำของระบบคอมพิวเตอร์ได้อย่างเหมาะสมที่สุด ดังนั้น ระบบคอมพิวเตอร์ทั่วไปจึงมีลำดับชั้นของระบบย่อยหน่วยความจำ บางส่วนอยู่ภายในระบบ (เข้าถึงได้โดยตรงจากโปรเซสเซอร์) และบางส่วนอยู่ภายนอก (เข้าถึงได้ผ่านโมดูล I/O)

บทนี้และบทถัดไปมุ่งเน้นที่องค์ประกอบหน่วยความจำภายใน ในขณะที่บทต่อไปจะกล่าวถึงหน่วยความจำภายนอก ในการเริ่มต้น ส่วนแรกจะตรวจสอบคุณลักษณะสำคัญของหน่วยความจำคอมพิวเตอร์ ส่วนที่เหลือของบทจะตรวจสอบองค์ประกอบสำคัญของระบบคอมพิวเตอร์สมัยใหม่ทุกระบบ นั่นคือ หน่วยความจำแคช

### คุณลักษณะของระบบหน่วยความจำ

เรื่องที่ซับซ้อนของหน่วยความจำคอมพิวเตอร์สามารถจัดการได้ดีขึ้น หากจำแนกระบบหน่วยความจำตามคุณลักษณะสำคัญ คุณลักษณะที่สำคัญที่สุดมีดังแสดงใน **ตารางที่ 4.1**

**ตารางที่ 4.1: คุณลักษณะสำคัญของระบบหน่วยความจำคอมพิวเตอร์**

| หมวดหมู่ | รายละเอียด |
|---------|----------|
| **ตำแหน่ง (Location)** | ภายใน (เช่น รีจิสเตอร์ของโปรเซสเซอร์, แคช, หน่วยความจำหลัก) / ภายนอก (เช่น ออปติคัลดิสก์, แม่เหล็กดิสก์, เทป) |
| **ความจุ (Capacity)** | จำนวนคำ (Number of words) / จำนวนไบต์ (Number of bytes) |
| **หน่วยการถ่ายโอน (Unit of Transfer)** | คำ (Word) / บล็อก (Block) |
| **วิธีการเข้าถึง (Access Method)** | แบบลำดับ (Sequential) / แบบตรง (Direct) / แบบสุ่ม (Random) / แบบเชื่อมโยง (Associative) |
| **ประสิทธิภาพ (Performance)** | เวลาการเข้าถึง (Access time) / เวลาวงจรหน่วยความจำ (Cycle time) / อัตราการถ่ายโอน (Transfer rate) |
| **ประเภททางกายภาพ (Physical Type)** | สารกึ่งตัวนำ (Semiconductor) / แม่เหล็ก (Magnetic) / ออปติคัล (Optical) / แม่เหล็ก-ออปติคัล (Magneto-optical) |
| **คุณลักษณะทางกายภาพ (Physical Characteristics)** | ระเหย/ไม่ระเหย (Volatile/Nonvolatile) / ลบได้/ลบไม่ได้ (Erasable/Nonerasable) |
| **การจัดระเบียบ (Organization)** | โมดูลหน่วยความจำ (Memory modules) |

คำว่า **ตำแหน่ง** (location) หมายถึงว่าหน่วยความจำอยู่ภายในหรือภายนอกคอมพิวเตอร์ หน่วยความจำภายในมักเทียบเท่ากับหน่วยความจำหลัก แต่ก็มีรูปแบบอื่น ๆ ของหน่วยความจำภายใน โปรเซสเซอร์ต้องการหน่วยความจำท้องถิ่นของตัวเองในรูปแบบของรีจิสเตอร์ นอกจากนี้ ส่วนควบคุมของโปรเซสเซอร์อาจต้องการหน่วยความจำภายในของตัวเองด้วย แคชเป็นอีกรูปแบบหนึ่งของหน่วยความจำภายใน หน่วยความจำภายนอกประกอบด้วยอุปกรณ์จัดเก็บข้อมูลต่อพ่วง เช่น ดิสก์และเทป ที่โปรเซสเซอร์เข้าถึงได้ผ่านตัวควบคุม I/O

คุณลักษณะที่ชัดเจนของหน่วยความจำคือ **ความจุ** (capacity) สำหรับหน่วยความจำภายใน มักแสดงเป็นไบต์ (1 ไบต์ = 8 บิต) หรือคำ ความยาวคำทั่วไปคือ 8, 16 และ 32 บิต ความจุหน่วยความจำภายนอกมักแสดงเป็นไบต์

แนวคิดที่เกี่ยวข้องคือ **หน่วยการถ่ายโอน** (unit of transfer) สำหรับหน่วยความจำภายใน หน่วยการถ่ายโอนเท่ากับจำนวนสายไฟฟ้าเข้าและออกจากโมดูลหน่วยความจำ ซึ่งอาจเท่ากับความยาวคำ แต่มักใหญ่กว่า เช่น 64, 128 หรือ 256 บิต มีแนวคิดสามประการที่เกี่ยวข้องสำหรับหน่วยความจำภายใน:

- **คำ (Word):** หน่วยการจัดระเบียบ "ธรรมชาติ" ของหน่วยความจำ ขนาดของคำโดยทั่วไปเท่ากับจำนวนบิตที่ใช้แทนจำนวนเต็มและความยาวคำสั่ง มีข้อยกเว้นมากมาย เช่น CRAY C90 มีความยาวคำ 64 บิตแต่ใช้การแทนจำนวนเต็ม 46 บิต สถาปัตยกรรม Intel x86 มีความยาวคำสั่งหลากหลายและขนาดคำ 32 บิต

- **หน่วยที่ระบุตำแหน่งได้ (Addressable units):** ในบางระบบ หน่วยที่ระบุตำแหน่งได้คือคำ แต่หลายระบบอนุญาตให้ระบุตำแหน่งระดับไบต์ ความสัมพันธ์ระหว่างความยาวที่อยู่ A บิตกับจำนวน N ของหน่วยที่ระบุตำแหน่งได้คือ 2^A = N

- **หน่วยการถ่ายโอน (Unit of transfer):** สำหรับหน่วยความจำหลัก คือจำนวนบิตที่อ่านออกหรือเขียนเข้าหน่วยความจำในแต่ละครั้ง ไม่จำเป็นต้องเท่ากับคำหรือหน่วยที่ระบุตำแหน่งได้ สำหรับหน่วยความจำภายนอก ข้อมูลมักถูกถ่ายโอนในหน่วยที่ใหญ่กว่าคำมาก เรียกว่า บล็อก

ความแตกต่างอีกประการหนึ่งคือ **วิธีการเข้าถึง** (method of accessing) ซึ่งมีดังนี้:

- **การเข้าถึงแบบลำดับ (Sequential access):** หน่วยความจำถูกจัดระเบียบเป็นหน่วยข้อมูลที่เรียกว่าเรคคอร์ด การเข้าถึงต้องทำในลำดับเชิงเส้นที่กำหนดไว้ เวลาในการเข้าถึงเรคคอร์ดใด ๆ จึงแปรผัน ตัวอย่างคือหน่วยเทป

- **การเข้าถึงแบบตรง (Direct access):** แต่ละบล็อกหรือเรคคอร์ดมีที่อยู่เฉพาะตามตำแหน่งทางกายภาพ การเข้าถึงทำได้โดยการเข้าถึงโดยตรงไปยังบริเวณทั่วไปบวกกับการค้นหาแบบลำดับ เวลาการเข้าถึงยังคงแปรผัน ตัวอย่างคือหน่วยดิสก์

- **การเข้าถึงแบบสุ่ม (Random access):** แต่ละตำแหน่งที่ระบุตำแหน่งได้ในหน่วยความจำมีกลไกการระบุตำแหน่งทางกายภาพที่เดินสาย (wired-in) เฉพาะ เวลาในการเข้าถึงตำแหน่งที่กำหนดไม่ขึ้นกับลำดับของการเข้าถึงก่อนหน้าและเป็นค่าคงที่ หน่วยความจำหลักและระบบแคชบางระบบเป็นแบบเข้าถึงแบบสุ่ม

- **การเข้าถึงแบบเชื่อมโยง (Associative):** เป็นหน่วยความจำแบบเข้าถึงสุ่มชนิดหนึ่งที่ช่วยให้เปรียบเทียบตำแหน่งบิตที่ต้องการภายในคำเพื่อจับคู่ที่ระบุและทำสิ่งนี้สำหรับคำทั้งหมดพร้อมกัน คำถูกเรียกคืนตามเนื้อหาส่วนหนึ่งของตัวเอง ไม่ใช่ที่อยู่ หน่วยความจำแคชอาจใช้การเข้าถึงแบบเชื่อมโยง

จากมุมมองของผู้ใช้ คุณลักษณะที่สำคัญที่สุดสองประการของหน่วยความจำคือ ความจุและ **ประสิทธิภาพ** (performance) มีพารามิเตอร์ประสิทธิภาพสามประการ:

- **เวลาการเข้าถึง (Access time/latency):** สำหรับหน่วยความจำแบบเข้าถึงสุ่ม คือเวลาที่ใช้ในการดำเนินการอ่านหรือเขียน นั่นคือเวลาตั้งแต่ส่งที่อยู่ไปยังหน่วยความจำจนกว่าข้อมูลจะถูกจัดเก็บหรือพร้อมใช้งาน

- **เวลาวงจรหน่วยความจำ (Memory cycle time):** แนวคิดนี้ใช้กับหน่วยความจำแบบเข้าถึงสุ่มเป็นหลัก และประกอบด้วยเวลาการเข้าถึงบวกกับเวลาเพิ่มเติมที่จำเป็นก่อนการเข้าถึงครั้งที่สองจะเริ่มได้

- **อัตราการถ่ายโอน (Transfer rate):** อัตราที่ข้อมูลสามารถถ่ายโอนเข้าหรือออกจากหน่วยหน่วยความจำ สำหรับหน่วยความจำแบบสุ่ม เท่ากับ 1/(เวลาวงจร) สำหรับหน่วยความจำแบบไม่สุ่ม:

```
Tn = TA + n/R
```

โดย:
- `Tn` = เวลาเฉลี่ยในการอ่านหรือเขียน n บิต
- `TA` = เวลาการเข้าถึงเฉลี่ย
- `n` = จำนวนบิต
- `R` = อัตราการถ่ายโอน (บิตต่อวินาที)

**คุณลักษณะทางกายภาพ:** ในหน่วยความจำแบบระเหย (volatile) ข้อมูลจะสูญหายเมื่อตัดไฟ ในหน่วยความจำแบบไม่ระเหย (nonvolatile) ข้อมูลที่บันทึกไว้จะคงอยู่โดยไม่เสื่อมสภาพจนกว่าจะเปลี่ยนโดยตั้งใจ ไม่ต้องการไฟฟ้าเพื่อเก็บรักษาข้อมูล หน่วยความจำแบบ nonerasable ไม่สามารถเปลี่ยนแปลงได้ ยกเว้นโดยการทำลายหน่วยจัดเก็บ หน่วยความจำเซมิคอนดักเตอร์ประเภทนี้รู้จักกันในชื่อ **หน่วยความจำอ่านอย่างเดียว (ROM)**

### ลำดับชั้นของหน่วยความจำ

ข้อจำกัดในการออกแบบหน่วยความจำของคอมพิวเตอร์สรุปได้จากคำถามสามข้อ: ต้องการเท่าไหร่? เร็วแค่ไหน? แพงแค่ไหน?

คำถามว่าต้องการเท่าไหร่นั้นค่อนข้างปลายเปิด หากความจุมีพร้อม แอปพลิเคชันก็มักจะถูกพัฒนาขึ้นเพื่อใช้ประโยชน์จากมัน สำหรับคำถามว่าเร็วแค่ไหนนั้น ตอบได้ง่ายกว่าในบางแง่มุม เพื่อให้ได้ประสิทธิภาพสูงสุด หน่วยความจำต้องสามารถตามทันโปรเซสเซอร์ได้ คำถามสุดท้ายเรื่องต้นทุนก็ต้องพิจารณาด้วย สำหรับระบบจริง ต้นทุนหน่วยความจำต้องสมเหตุสมผลเมื่อเทียบกับส่วนประกอบอื่น ๆ

มีการแลกเปลี่ยนระหว่างคุณลักษณะสำคัญสามประการของหน่วยความจำ: ความจุ เวลาการเข้าถึง และต้นทุน ความสัมพันธ์ที่ครอบคลุมเทคโนโลยีต่าง ๆ ได้แก่:

- เวลาการเข้าถึงเร็วกว่า → ต้นทุนต่อบิตสูงกว่า
- ความจุมากกว่า → ต้นทุนต่อบิตต่ำกว่า
- ความจุมากกว่า → เวลาการเข้าถึงช้ากว่า

ทางออกของปัญหานี้ไม่ได้พึ่งพาส่วนประกอบหน่วยความจำหรือเทคโนโลยีเดียว แต่ใช้ **ลำดับชั้นของหน่วยความจำ** (memory hierarchy) เมื่อเลื่อนลงในลำดับชั้น จะเกิดสิ่งต่อไปนี้:

- (a) ต้นทุนต่อบิตลดลง
- (b) ความจุเพิ่มขึ้น
- (c) เวลาการเข้าถึงเพิ่มขึ้น
- (d) ความถี่ในการเข้าถึงหน่วยความจำโดยโปรเซสเซอร์ลดลง

ดังนั้น หน่วยความจำขนาดเล็ก ราคาแพง และเร็วกว่าจะถูกเสริมด้วยหน่วยความจำขนาดใหญ่ ราคาถูก และช้ากว่า ปัจจัยสำคัญต่อความสำเร็จของการจัดระเบียบนี้คือข้อ (d): ความถี่ในการเข้าถึงที่ลดลง

พื้นฐานสำหรับความถูกต้องของข้อ (d) คือหลักการที่รู้จักกันในชื่อ **การอ้างอิงเฉพาะที่ (locality of reference)** ในระหว่างการทำงานของโปรแกรม การอ้างอิงหน่วยความจำโดยโปรเซสเซอร์ ทั้งสำหรับคำสั่งและข้อมูล มีแนวโน้มที่จะรวมกลุ่ม โปรแกรมมักมีลูปซ้ำและซับรูทีน เมื่อเข้าสู่ลูปหรือซับรูทีน จะมีการอ้างอิงซ้ำ ๆ ไปยังชุดคำสั่งขนาดเล็ก ในทำนองเดียวกัน การดำเนินการกับตารางและอาร์เรย์เกี่ยวข้องกับการเข้าถึงชุดคำข้อมูลที่รวมกลุ่ม

**ตัวอย่างที่ 4.1:** สมมติว่าโปรเซสเซอร์เข้าถึงหน่วยความจำสองระดับ ระดับ 1 มี 1,000 คำและเวลาการเข้าถึง 0.01 ms; ระดับ 2 มี 100,000 คำและเวลาการเข้าถึง 0.1 ms สมมติว่าถ้าคำที่ต้องการเข้าถึงอยู่ในระดับ 1 โปรเซสเซอร์จะเข้าถึงโดยตรง ถ้าอยู่ในระดับ 2 คำนั้นจะถูกถ่ายโอนไปยังระดับ 1 ก่อนแล้วจึงเข้าถึงโดยโปรเซสเซอร์

สมมติว่า 95% ของการเข้าถึงหน่วยความจำพบในระดับ 1 เวลาเฉลี่ยในการเข้าถึงคำ:

```
(0.95)(0.01 ms) + (0.05)(0.01 ms + 0.1 ms) = 0.0095 + 0.0055 = 0.015 ms
```

เวลาการเข้าถึงเฉลี่ยใกล้เคียงกับ 0.01 ms มากกว่า 0.1 ms ตามที่ต้องการ

**โครงสร้างลำดับชั้นหน่วยความจำทั่วไป:**

```
[รีจิสเตอร์]          ← เร็วที่สุด, เล็กที่สุด, แพงที่สุด
[แคช]
[หน่วยความจำหลัก]     ← หน่วยความจำภายใน (Inboard)
[แมกเนติกดิสก์]
[CD-ROM / DVD / Blu-ray]  ← หน่วยเก็บข้อมูลภายนอก (Outboard)
[เทปแม่เหล็ก]         ← ช้าที่สุด, ใหญ่ที่สุด, ถูกที่สุด (Off-line)
```

หน่วยความจำหลักเป็นระบบหน่วยความจำภายในหลักของคอมพิวเตอร์ แต่ละตำแหน่งในหน่วยความจำหลักมีที่อยู่เฉพาะ หน่วยความจำหลักมักถูกขยายด้วยแคชที่เล็กกว่าและเร็วกว่า แคชมักไม่ปรากฏแก่โปรแกรมเมอร์หรือแม้แต่โปรเซสเซอร์ มันเป็นอุปกรณ์สำหรับจัดเตรียมการเคลื่อนย้ายข้อมูลระหว่างหน่วยความจำหลักและรีจิสเตอร์ของโปรเซสเซอร์เพื่อปรับปรุงประสิทธิภาพ

## 4.2 หลักการของหน่วยความจำแคช

หน่วยความจำแคชได้รับการออกแบบเพื่อรวมเวลาการเข้าถึงหน่วยความจำของหน่วยความจำราคาแพงความเร็วสูงเข้ากับขนาดหน่วยความจำขนาดใหญ่ของหน่วยความจำราคาถูกความเร็วต่ำ มีหน่วยความจำหลักขนาดใหญ่และช้าค่อนข้างมาก พร้อมด้วยหน่วยความจำแคชขนาดเล็กและเร็วกว่า แคชมีสำเนาของส่วนหนึ่งของหน่วยความจำหลัก

เมื่อโปรเซสเซอร์พยายามอ่านคำจากหน่วยความจำ จะมีการตรวจสอบว่าคำนั้นอยู่ในแคชหรือไม่ ถ้าอยู่ คำนั้นจะถูกส่งไปยังโปรเซสเซอร์ ถ้าไม่อยู่ บล็อกหน่วยความจำหลักที่มีจำนวนคำที่กำหนดไว้จะถูกอ่านเข้าไปในแคช แล้วคำนั้นจะถูกส่งไปยังโปรเซสเซอร์ เนื่องจากปรากฏการณ์การอ้างอิงเฉพาะที่ เมื่อบล็อกข้อมูลถูกนำเข้าแคชเพื่อตอบสนองการอ้างอิงหน่วยความจำครั้งเดียว มีแนวโน้มสูงที่จะมีการอ้างอิงในอนาคตไปยังตำแหน่งหน่วยความจำเดียวกันหรือคำอื่น ๆ ในบล็อก

**โครงสร้างแคช/หน่วยความจำหลัก:**

หน่วยความจำหลักประกอบด้วยคำที่ระบุตำแหน่งได้สูงสุด 2^n คำ โดยแต่ละคำมีที่อยู่ n บิตเฉพาะ สำหรับวัตถุประสงค์การแมปปิ้ง หน่วยความจำนี้ถือว่าประกอบด้วยบล็อกความยาวคงที่ของ K คำแต่ละบล็อก นั่นคือมี M = 2^n/K บล็อกในหน่วยความจำหลัก แคชประกอบด้วย m บล็อกที่เรียกว่า **บรรทัด (lines)** แต่ละบรรทัดมี K คำบวกแท็กสองสามบิต

> **หมายเหตุ:** คำว่า "บรรทัด" (line) ใช้แทนคำว่า "บล็อก" เพื่อหลีกเลี่ยงความสับสนกับบล็อกหน่วยความจำหลัก และเพราะบรรทัดแคชประกอบด้วยไม่เพียงแค่ K คำข้อมูลเหมือนบล็อกหน่วยความจำหลัก แต่ยังมีบิตแท็กและบิตควบคุมด้วย

**การทำงานของแคชหลายระดับ:**

```
[CPU] ←→ [L1 Cache] ←→ [L2 Cache] ←→ [L3 Cache] ←→ [หน่วยความจำหลัก]
         (เร็วที่สุด)   (เร็ว)       (ปานกลาง)    (ช้า)
```

**กระบวนการอ่านข้อมูล (Cache Read Operation):**

```
เริ่มต้น (START)
    ↓
รับที่อยู่ RA จาก CPU
    ↓
บล็อกที่มี RA อยู่ในแคชหรือไม่?
    ↓ ใช่                    ↓ ไม่ใช่
ดึงคำ RA และส่งไป CPU    เข้าถึงหน่วยความจำหลักเพื่อบล็อกที่มี RA
    ↓                         ↓
เสร็จสิ้น (DONE)         จัดสรรบรรทัดแคชสำหรับบล็อกหน่วยความจำหลัก
                              ↓
                     โหลดบล็อกหน่วยความจำหลักเข้าบรรทัดแคช
                              ↓
                         ส่งคำ RA ไป CPU
                              ↓
                         เสร็จสิ้น (DONE)
```

## 4.3 องค์ประกอบของการออกแบบแคช

ส่วนนี้ให้ภาพรวมของพารามิเตอร์การออกแบบแคชและรายงานผลลัพธ์ทั่วไปบางอย่าง มีการกล่าวถึงการใช้แคชในการประมวลผลสมรรถนะสูง (High-Performance Computing: HPC) ซึ่งเกี่ยวข้องกับซูเปอร์คอมพิวเตอร์และซอฟต์แวร์ของพวกมัน โดยเฉพาะสำหรับแอปพลิเคชันทางวิทยาศาสตร์ที่เกี่ยวข้องกับข้อมูลจำนวนมาก การคำนวณเวกเตอร์และเมทริกซ์ และการใช้อัลกอริทึมแบบขนาน

**ตารางที่ 4.2: องค์ประกอบของการออกแบบแคช**

| หมวดหมู่ | ตัวเลือก |
|---------|---------|
| **ที่อยู่แคช** | ลอจิคัล / ฟิสิคัล |
| **ขนาดแคช** | - |
| **ฟังก์ชันการแมปปิ้ง** | ตรง (Direct) / เชื่อมโยง (Associative) / เซตเชื่อมโยง (Set associative) |
| **อัลกอริทึมการแทนที่** | LRU / FIFO / LFU / สุ่ม (Random) |
| **นโยบายการเขียน** | Write through / Write back |
| **ขนาดบรรทัด** | - |
| **จำนวนแคช** | เดี่ยวหรือสองระดับ / รวม (Unified) หรือแยก (Split) |

### ที่อยู่แคช

โปรเซสเซอร์ที่ไม่ฝังตัวเกือบทั้งหมด และโปรเซสเซอร์ฝังตัวจำนวนมาก สนับสนุน **หน่วยความจำเสมือน (virtual memory)** ซึ่งเป็นคุณสมบัติที่ช่วยให้โปรแกรมระบุตำแหน่งหน่วยความจำจากมุมมองเชิงตรรกะ โดยไม่คำนึงถึงจำนวนหน่วยความจำหลักที่มีจริง เมื่อใช้หน่วยความจำเสมือน ช่องที่อยู่ของคำสั่งเครื่องจะมีที่อยู่เสมือน สำหรับการอ่านและเขียนในหน่วยความจำหลัก หน่วยจัดการหน่วยความจำฮาร์ดแวร์ (Memory Management Unit: MMU) จะแปลงที่อยู่เสมือนแต่ละรายการเป็นที่อยู่ทางกายภาพในหน่วยความจำหลัก

เมื่อใช้ที่อยู่เสมือน นักออกแบบระบบอาจเลือกวางแคชระหว่างโปรเซสเซอร์กับ MMU หรือระหว่าง MMU กับหน่วยความจำหลัก:

- **แคชลอจิคัล (Logical cache / Virtual cache):** เก็บข้อมูลโดยใช้ที่อยู่เสมือน โปรเซสเซอร์เข้าถึงแคชโดยตรงโดยไม่ผ่าน MMU ข้อดีคือเข้าถึงแคชเร็วกว่า เพราะแคชตอบสนองได้ก่อนที่ MMU จะแปลที่อยู่

- **แคชฟิสิคัล (Physical cache):** เก็บข้อมูลโดยใช้ที่อยู่ทางกายภาพของหน่วยความจำหลัก

ข้อเสียของแคชลอจิคัลคือระบบหน่วยความจำเสมือนส่วนใหญ่ให้พื้นที่ที่อยู่เสมือนเดียวกันแก่แต่ละแอปพลิเคชัน ดังนั้นที่อยู่เสมือนเดียวกันในสองแอปพลิเคชันต่างกันจึงอ้างถึงที่อยู่ทางกายภาพที่ต่างกัน แคชจึงต้องถูกล้างทั้งหมดในทุกครั้งที่มีการสลับบริบทแอปพลิเคชัน

### ฟังก์ชันการแมปปิ้ง

เนื่องจากมีบรรทัดแคชน้อยกว่าบล็อกหน่วยความจำหลัก จึงต้องมีอัลกอริทึมสำหรับการแมปบล็อกหน่วยความจำหลักเข้าสู่บรรทัดแคช มีเทคนิคสามแบบ: แบบตรง (direct) แบบเชื่อมโยง (associative) และแบบเซตเชื่อมโยง (set-associative)

**ตัวอย่างที่ 4.2:** สำหรับทุกกรณี ตัวอย่างมีองค์ประกอบดังนี้:
- แคชสามารถเก็บ 64 kB
- ข้อมูลถูกถ่ายโอนระหว่างหน่วยความจำหลักและแคชในบล็อก 4 ไบต์ ดังนั้นแคชถูกจัดระเบียบเป็น 16K = 2^14 บรรทัดของ 4 ไบต์แต่ละบรรทัด
- หน่วยความจำหลักมี 16 MB โดยแต่ละไบต์ระบุตำแหน่งได้ด้วยที่อยู่ 24 บิต (2^24 = 16M) ดังนั้นสำหรับวัตถุประสงค์การแมปปิ้ง หน่วยความจำหลักประกอบด้วย 4M บล็อกของ 4 ไบต์

#### การแมปปิ้งแบบตรง (Direct Mapping)

เทคนิคที่ง่ายที่สุด เรียกว่า **การแมปปิ้งแบบตรง** แมปแต่ละบล็อกของหน่วยความจำหลักไปยังบรรทัดแคชเดียวที่เป็นไปได้ การแมปปิ้งแสดงเป็น:

```
i = j mod m
```

โดย:
- `i` = หมายเลขบรรทัดแคช
- `j` = หมายเลขบล็อกหน่วยความจำหลัก
- `m` = จำนวนบรรทัดในแคช

**สรุปพารามิเตอร์การแมปปิ้งแบบตรง:**

| พารามิเตอร์ | สูตร |
|------------|------|
| ความยาวที่อยู่ | (s + w) บิต |
| จำนวนหน่วยที่ระบุตำแหน่งได้ | 2^(s+w) คำหรือไบต์ |
| ขนาดบล็อก = ขนาดบรรทัด | 2^w คำหรือไบต์ |
| จำนวนบล็อกในหน่วยความจำหลัก | 2^s |
| จำนวนบรรทัดในแคช | m = 2^r |
| ขนาดแคช | 2^(r+w) คำหรือไบต์ |
| ขนาดแท็ก | (s - r) บิต |

**โครงสร้างที่อยู่:**
```
┌──────────┬──────────┬──────────┐
│   แท็ก   │  บรรทัด  │   คำ    │
│  (s-r บิต)│  (r บิต) │  (w บิต)│
└──────────┴──────────┴──────────┘
```

**ตัวอย่างที่ 4.2a:** ในตัวอย่าง m = 16K = 2^14 และ i = j mod 2^14 การแมปปิ้งจะเป็น:

| บรรทัดแคช | ที่อยู่หน่วยความจำเริ่มต้นของบล็อก |
|---------|--------------------------------|
| 0 | 000000, 010000, …, FF0000 |
| 1 | 000004, 010004, …, FF0004 |
| ... | ... |
| 2^14 - 1 | 00FFFC, 01FFFC, …, FFFFFC |

การอ่านทำงานดังนี้: แคชได้รับที่อยู่ 24 บิต หมายเลขบรรทัด 14 บิตถูกใช้เป็นดัชนีเข้าสู่แคช ถ้าหมายเลขแท็ก 8 บิตตรงกับแท็กที่เก็บในบรรทัดนั้น หมายเลขคำ 2 บิตจะถูกใช้เพื่อเลือกหนึ่งใน 4 ไบต์ในบรรทัดนั้น

**ข้อดีและข้อเสียของการแมปปิ้งแบบตรง:**
- ✅ เรียบง่ายและราคาถูกในการนำไปใช้
- ❌ มีตำแหน่งแคชที่ตายตัวสำหรับบล็อกที่กำหนด ถ้าโปรแกรมอ้างอิงคำจากสองบล็อกต่างกันซ้ำ ๆ ที่แมปไปยังบรรทัดเดียวกัน บล็อกจะถูกสลับกันอยู่ตลอดเวลา (เรียกว่า **การกระชาก/thrashing**) และอัตราการโดนจะต่ำ

#### การแมปปิ้งแบบเชื่อมโยง (Associative Mapping)

**การแมปปิ้งแบบเชื่อมโยง** เอาชนะข้อเสียของการแมปปิ้งแบบตรงโดยอนุญาตให้แต่ละบล็อกหน่วยความจำหลักโหลดเข้าสู่บรรทัดแคชใด ๆ ก็ได้ ลอจิกควบคุมแคชตีความที่อยู่หน่วยความจำเป็นเพียงฟิลด์แท็กและฟิลด์คำ ฟิลด์แท็กระบุบล็อกหน่วยความจำหลักเฉพาะ ในการตรวจสอบว่าบล็อกอยู่ในแคชหรือไม่ ลอจิกควบคุมแคชต้องตรวจสอบแท็กของทุกบรรทัดพร้อมกัน

**สรุปพารามิเตอร์การแมปปิ้งแบบเชื่อมโยง:**

| พารามิเตอร์ | สูตร |
|------------|------|
| ความยาวที่อยู่ | (s + w) บิต |
| จำนวนหน่วยที่ระบุตำแหน่งได้ | 2^(s+w) คำหรือไบต์ |
| ขนาดบล็อก = ขนาดบรรทัด | 2^w คำหรือไบต์ |
| จำนวนบล็อกในหน่วยความจำหลัก | 2^s |
| จำนวนบรรทัดในแคช | ไม่ได้กำหนด |
| ขนาดแท็ก | s บิต |

**โครงสร้างที่อยู่:**
```
┌──────────────────────┬──────────┐
│         แท็ก         │    คำ    │
│       (s บิต)        │  (w บิต) │
└──────────────────────┴──────────┘
```

**ตัวอย่างที่ 4.2b:** ที่อยู่หน่วยความจำหลักประกอบด้วยแท็ก 22 บิตและหมายเลขไบต์ 2 บิต แท็ก 22 บิตต้องถูกเก็บพร้อมกับบล็อกข้อมูล 32 บิตสำหรับแต่ละบรรทัดในแคช

**ข้อดีและข้อเสียของการแมปปิ้งแบบเชื่อมโยง:**
- ✅ มีความยืดหยุ่นในการเลือกบล็อกที่จะแทนที่ ช่วยให้อัตราการโดนสูงขึ้น
- ❌ ต้องการวงจรที่ซับซ้อนในการตรวจสอบแท็กของทุกบรรทัดแคชพร้อมกัน

#### การแมปปิ้งแบบเซตเชื่อมโยง (Set-Associative Mapping)

**การแมปปิ้งแบบเซตเชื่อมโยง** เป็นการประนีประนอมที่แสดงจุดแข็งของทั้งแนวทางแบบตรงและแบบเชื่อมโยงพร้อมลดข้อเสียของพวกมัน

แคชประกอบด้วยจำนวนเซต แต่ละเซตประกอบด้วยจำนวนบรรทัด ความสัมพันธ์คือ:

```
m = v × k
i = j mod v
```

โดย:
- `i` = หมายเลขเซตแคช
- `j` = หมายเลขบล็อกหน่วยความจำหลัก
- `m` = จำนวนบรรทัดในแคช
- `v` = จำนวนเซต
- `k` = จำนวนบรรทัดในแต่ละเซต

เรียกว่า **การแมปปิ้งแบบเซตเชื่อมโยง k ทาง** ด้วยการแมปปิ้งแบบเซตเชื่อมโยง บล็อก B_j สามารถแมปไปยังบรรทัดใด ๆ ของเซต j

**สรุปพารามิเตอร์การแมปปิ้งแบบเซตเชื่อมโยง:**

| พารามิเตอร์ | สูตร |
|------------|------|
| ความยาวที่อยู่ | (s + w) บิต |
| จำนวนหน่วยที่ระบุตำแหน่งได้ | 2^(s+w) คำหรือไบต์ |
| ขนาดบล็อก = ขนาดบรรทัด | 2^w คำหรือไบต์ |
| จำนวนบล็อกในหน่วยความจำหลัก | 2^s |
| จำนวนบรรทัดในเซต | k |
| จำนวนเซต | v = 2^d |
| จำนวนบรรทัดในแคช | m = kv = k × 2^d |
| ขนาดแคช | k × 2^(d+w) คำหรือไบต์ |
| ขนาดแท็ก | (s - d) บิต |

**โครงสร้างที่อยู่:**
```
┌──────────┬──────────┬──────────┐
│   แท็ก   │   เซต   │   คำ    │
│ (s-d บิต)│  (d บิต) │  (w บิต)│
└──────────┴──────────┴──────────┘
```

**ตัวอย่างที่ 4.2c:** ตัวอย่างใช้การแมปปิ้งแบบเซตเชื่อมโยงสองทาง (two-way set-associative) พร้อมสองบรรทัดในแต่ละเซต หมายเลขเซต 13 บิตระบุเซตเฉพาะของสองบรรทัดภายในแคช บล็อก 000000, 008000, ..., FF8000 แมปไปยังเซตแคช 0 บล็อกใด ๆ เหล่านั้นสามารถโหลดเข้าสู่หนึ่งในสองบรรทัดในเซต

**การเปรียบเทียบสามวิธีการแมปปิ้ง:**

| คุณลักษณะ | แบบตรง | แบบเชื่อมโยง | แบบเซตเชื่อมโยง |
|----------|--------|-------------|----------------|
| ความซับซ้อน | ต่ำ | สูง | ปานกลาง |
| ความยืดหยุ่น | ต่ำ | สูง | ปานกลาง |
| ต้นทุนฮาร์ดแวร์ | ต่ำ | สูง | ปานกลาง |
| อัตราการโดน | ต่ำ (อาจเกิด thrashing) | สูง | ดี |

ในกรณีสุดขั้วของ v = m, k = 1 การแมปปิ้งแบบเซตเชื่อมโยงลดลงเป็นการแมปปิ้งแบบตรง และสำหรับ v = 1, k = m จะลดลงเป็นการแมปปิ้งแบบเชื่อมโยง การใช้สองบรรทัดต่อเซต (v = m/2, k = 2) เป็นการจัดระเบียบแบบเซตเชื่อมโยงที่พบมากที่สุด ซึ่งช่วยเพิ่มอัตราการโดนอย่างมีนัยสำคัญเมื่อเทียบกับการแมปปิ้งแบบตรง

### อัลกอริทึมการแทนที่

เมื่อแคชถูกเติมเต็มแล้ว เมื่อนำบล็อกใหม่เข้าสู่แคช หนึ่งในบล็อกที่มีอยู่จะต้องถูกแทนที่ สำหรับการแมปปิ้งแบบตรงไม่มีทางเลือก สำหรับเทคนิคแบบเชื่อมโยงและเซตเชื่อมโยง จำเป็นต้องมีอัลกอริทึมการแทนที่ อัลกอริทึมที่พบมากที่สุดสี่แบบ:

1. **LRU (Least Recently Used - ใช้งานน้อยที่สุดล่าสุด):** แทนที่บล็อกในเซตที่อยู่ในแคชนานที่สุดโดยไม่มีการอ้างอิงถึง สำหรับแบบเซตเชื่อมโยงสองทาง นำไปใช้ได้ง่าย แต่ละบรรทัดมีบิต USE เมื่อบรรทัดถูกอ้างอิง บิต USE ของมันจะตั้งเป็น 1 และบิต USE ของบรรทัดอื่นในเซตนั้นจะตั้งเป็น 0 เพราะเป็นที่ยอมรับว่าตำแหน่งหน่วยความจำที่ใช้งานล่าสุดมีแนวโน้มถูกอ้างอิงมากกว่า LRU จึงควรให้อัตราการโดนที่ดีที่สุด **LRU เป็นอัลกอริทึมการแทนที่ที่นิยมมากที่สุด**

2. **FIFO (First In First Out - เข้าก่อนออกก่อน):** แทนที่บล็อกในเซตที่อยู่ในแคชนานที่สุด นำไปใช้ได้ง่ายเป็นเทคนิค round-robin หรือ circular buffer

3. **LFU (Least Frequently Used - ใช้บ่อยน้อยที่สุด):** แทนที่บล็อกในเซตที่มีการอ้างอิงน้อยที่สุด นำไปใช้ได้โดยเชื่อมโยงตัวนับกับแต่ละบรรทัด

4. **สุ่ม (Random):** เลือกบรรทัดแบบสุ่มจากบรรทัดผู้สมัคร การศึกษาการจำลองแสดงให้เห็นว่าการแทนที่แบบสุ่มให้ประสิทธิภาพที่ด้อยกว่าอัลกอริทึมที่ใช้ข้อมูลการใช้งานเพียงเล็กน้อย

### นโยบายการเขียน

เมื่อบล็อกที่อยู่ในแคชจะถูกแทนที่ มีสองกรณีที่ต้องพิจารณา ถ้าบล็อกเดิมในแคชไม่ถูกเปลี่ยนแปลง อาจเขียนทับด้วยบล็อกใหม่โดยไม่ต้องเขียนออกบล็อกเดิมก่อน ถ้ามีการดำเนินการเขียนอย่างน้อยหนึ่งครั้งกับคำในบรรทัดนั้นของแคช หน่วยความจำหลักต้องอัพเดตโดยเขียนบรรทัดแคชออกไปยังบล็อกหน่วยความจำก่อนนำบล็อกใหม่เข้า

มีสองปัญหาที่ต้องพิจารณา:

1. มากกว่าหนึ่งอุปกรณ์อาจเข้าถึงหน่วยความจำหลัก (เช่น โมดูล I/O) ถ้าคำถูกเปลี่ยนแปลงเฉพาะในแคช คำหน่วยความจำที่สอดคล้องกันจะไม่ถูกต้อง
2. ถ้าโปรเซสเซอร์หลายตัวแนบกับบัสเดียวกัน แต่ละตัวมีแคชของตัวเอง ถ้าคำถูกเปลี่ยนในแคชหนึ่ง อาจทำให้คำในแคชอื่น ๆ ไม่ถูกต้อง

**นโยบายหลักสองแบบ:**

**1. Write Through (เขียนผ่าน):**
- การดำเนินการเขียนทั้งหมดทำกับหน่วยความจำหลักและแคชพร้อมกัน
- ✅ หน่วยความจำหลักถูกต้องเสมอ โมดูลโปรเซสเซอร์-แคชอื่น ๆ สามารถตรวจสอบการรับส่งข้อมูลในหน่วยความจำหลักเพื่อรักษาความสอดคล้อง
- ❌ สร้างการรับส่งข้อมูลหน่วยความจำจำนวนมากและอาจสร้างคอขวด

**2. Write Back (เขียนกลับ):**
- การอัพเดตทำในแคชเท่านั้น เมื่อเกิดการอัพเดต **บิตสกปรก (dirty bit)** ที่เกี่ยวข้องกับบรรทัดจะถูกตั้งค่า เมื่อบล็อกถูกแทนที่ จะถูกเขียนกลับไปยังหน่วยความจำหลักก็ต่อเมื่อบิตสกปรกถูกตั้งค่าเท่านั้น
- ✅ ลดการเขียนหน่วยความจำให้น้อยที่สุด
- ❌ ส่วนหนึ่งของหน่วยความจำหลักไม่ถูกต้อง การเข้าถึงโดยโมดูล I/O สามารถอนุญาตได้ผ่านแคชเท่านั้น ทำให้วงจรซับซ้อนและอาจเกิดคอขวด

**ตัวอย่างที่ 4.3:** พิจารณาแคชที่มีขนาดบรรทัด 32 ไบต์ และหน่วยความจำหลักที่ต้องการ 30 ns ในการถ่ายโอนคำ 4 ไบต์

สำหรับ write-back: บรรทัดสกปรกแต่ละบรรทัดถูกเขียนกลับครั้งเดียว = 8 × 30 = 240 ns
สำหรับ write-through: การอัพเดตแต่ละครั้งต้องการ 30 ns

ดังนั้น ถ้าบรรทัดเฉลี่ยที่ถูกเขียนอย่างน้อยหนึ่งครั้งถูกเขียนมากกว่า 8 ครั้งก่อนสลับออก write-back จะมีประสิทธิภาพมากกว่า

**ความสอดคล้องของแคช (Cache Coherency):**

ในระบบที่มีหลายโปรเซสเซอร์ ถ้าข้อมูลในแคชหนึ่งถูกเปลี่ยนแปลง สิ่งนี้ทำให้ไม่เพียงคำที่สอดคล้องกันในหน่วยความจำหลักไม่ถูกต้อง แต่ยังรวมถึงคำเดียวกันในแคชอื่น ๆ ด้วย ระบบที่ป้องกันปัญหานี้เรียกว่า **รักษาความสอดคล้องของแคช** แนวทางที่เป็นไปได้ ได้แก่:

- **การเฝ้าดูบัสกับ write-through:** ตัวควบคุมแคชแต่ละตัวตรวจสอบสายที่อยู่เพื่อตรวจจับการดำเนินการเขียนไปยังหน่วยความจำโดยมาสเตอร์บัสอื่น ๆ ถ้ามาสเตอร์อื่นเขียนไปยังตำแหน่งในหน่วยความจำที่ใช้ร่วมกันที่อยู่ในแคชด้วย ตัวควบคุมแคชจะทำให้รายการแคชนั้นไม่ถูกต้อง

- **ความโปร่งใสของฮาร์ดแวร์:** ใช้ฮาร์ดแวร์เพิ่มเติมเพื่อให้แน่ใจว่าการอัพเดตหน่วยความจำหลักผ่านแคชสะท้อนในแคชทั้งหมด

- **หน่วยความจำที่ไม่สามารถแคชได้:** เฉพาะส่วนหนึ่งของหน่วยความจำหลักที่ใช้ร่วมกันโดยมากกว่าหนึ่งโปรเซสเซอร์ถูกกำหนดว่าไม่สามารถแคชได้

### ขนาดบรรทัด

เมื่อบล็อกข้อมูลถูกดึงมาและวางในแคช ไม่เพียงแค่คำที่ต้องการเท่านั้น แต่ยังมีคำที่อยู่ใกล้เคียงจำนวนหนึ่งด้วยที่ถูกดึงมา เมื่อขนาดบล็อกเพิ่มขึ้นจากขนาดเล็กมากไปสู่ขนาดใหญ่ขึ้น อัตราการโดนจะเพิ่มขึ้นในตอนแรกเนื่องจากหลักการเฉพาะที่ แต่อัตราการโดนจะเริ่มลดลงเมื่อบล็อกใหญ่ขึ้นมาก เพราะความน่าจะเป็นในการใช้ข้อมูลที่ดึงมาใหม่มีน้อยกว่าความน่าจะเป็นในการใช้ข้อมูลที่ต้องถูกแทนที่

สองผลกระทบเฉพาะคือ:
1. บล็อกขนาดใหญ่ลดจำนวนบล็อกที่สามารถใส่ในแคช เนื่องจากการดึงบล็อกแต่ละครั้งเขียนทับเนื้อหาแคชเก่า
2. เมื่อบล็อกใหญ่ขึ้น คำเพิ่มเติมแต่ละคำอยู่ห่างจากคำที่ร้องขอมากขึ้นและมีโอกาสน้อยลงที่จะต้องการในอนาคตอันใกล้

ขนาด 8 ถึง 64 ไบต์ดูเหมาะสมใกล้เคียงกับค่าที่ดีที่สุด สำหรับระบบ HPC ขนาดบรรทัดแคช 64 และ 128 ไบต์ถูกใช้บ่อยที่สุด

### จำนวนแคช

#### แคชหลายระดับ

เมื่อความหนาแน่นของลอจิกเพิ่มขึ้น จึงเป็นไปได้ที่จะมีแคชบนชิปเดียวกับโปรเซสเซอร์: **แคชบนชิป (on-chip cache)** เมื่อเทียบกับแคชที่เข้าถึงได้ผ่านบัสภายนอก แคชบนชิปลดกิจกรรมบัสภายนอกของโปรเซสเซอร์ จึงเร่งเวลาการทำงานและเพิ่มประสิทธิภาพระบบโดยรวม

การออกแบบที่ง่ายที่สุดรู้จักกันในชื่อ **แคชสองระดับ** โดย L1 อยู่ภายในและ L2 ภายนอก เหตุผลในการรวม L2 คือ: ถ้าไม่มี L2 และโปรเซสเซอร์ร้องขอตำแหน่งหน่วยความจำที่ไม่อยู่ใน L1 โปรเซสเซอร์ต้องเข้าถึง DRAM หรือ ROM ผ่านบัส ซึ่งส่งผลให้ประสิทธิภาพต่ำ ในทางกลับกัน ถ้าใช้ SRAM ที่เป็น L2 ข้อมูลที่ขาดหายบ่อยครั้งสามารถดึงมาได้อย่างรวดเร็ว

ด้วยพื้นที่บนชิปที่เพิ่มขึ้น ไมโครโปรเซสเซอร์ส่วนใหญ่ในปัจจุบันได้ย้าย L2 ไปยังชิปโปรเซสเซอร์และเพิ่ม L3 cache นอกจากนี้ระบบขนาดใหญ่ เช่น IBM mainframe zEnterprise systems ยังรวม 3 ระดับแคชบนชิปและระดับที่สี่ที่ใช้ร่วมกันระหว่างหลายชิป

#### แคชรวม vs แคชแยก

เมื่อแคชบนชิปปรากฏตัวครั้งแรก การออกแบบส่วนใหญ่ประกอบด้วยแคชเดียวที่ใช้เก็บทั้งการอ้างอิงข้อมูลและคำสั่ง ล่าสุด เป็นเรื่องปกติที่จะแยกแคชออกเป็นสอง: หนึ่งสำหรับคำสั่งและหนึ่งสำหรับข้อมูล

**ข้อดีของแคชรวม (Unified Cache):**
1. สำหรับขนาดแคชที่กำหนด แคชรวมมีอัตราการโดนสูงกว่าแคชแยก เพราะมันสมดุลโหลดระหว่างการดึงคำสั่งและข้อมูลโดยอัตโนมัติ
2. ต้องออกแบบและนำไปใช้เพียงแคชเดียว

**ข้อดีของแคชแยก (Split Cache):**
- ข้อได้เปรียบสำคัญคือขจัดการแย่งชิงแคชระหว่างหน่วยดึง/ถอดรหัสคำสั่งและหน่วยประมวลผล สิ่งนี้สำคัญในการออกแบบที่ใช้การ pipelining ของคำสั่ง

แนวโน้มคือแคชแยกที่ระดับ L1 และแคชรวมสำหรับระดับสูงกว่า โดยเฉพาะสำหรับเครื่อง superscalar ที่เน้นการทำงานคำสั่งแบบขนานและการ prefetch คำสั่งที่คาดการณ์ล่วงหน้า

**องค์ประกอบหลักของ Pentium 4:**

```
[System Bus]
     │
 [L3 Cache (1 MB)] ←→ [L2 Cache (512 kB)]
                              │
              ┌───────────────┘
              │
    [Instruction Fetch/Decode Unit]
              │
    [L1 Instruction Cache (12K µops)]
              │
    [Out-of-Order Execution Logic]
              │
    [Integer Register File] ←→ [FP Register File]
              │
    [Load/Store Units] [ALUs] [FP Units]
              │
    [L1 Data Cache (16 kB)]
```

โปรเซสเซอร์ประกอบด้วยสี่ส่วนหลัก:
1. **หน่วยดึง/ถอดรหัส (Fetch/decode unit):** ดึงคำสั่งโปรแกรมตามลำดับจากแคช L2 ถอดรหัสเป็นชุด micro-operation และเก็บผลลัพธ์ในแคชคำสั่ง L1
2. **ลอจิกการทำงานนอกลำดับ (Out-of-order execution logic):** กำหนดตารางการทำงานของ micro-operation ตามความพร้อมของข้อมูลและทรัพยากร
3. **หน่วยประมวลผล (Execution units):** ทำงาน micro-operation ดึงข้อมูลที่ต้องการจากแคชข้อมูล L1 และเก็บผลลัพธ์ชั่วคราวในรีจิสเตอร์
4. **ระบบย่อยหน่วยความจำ (Memory subsystem):** รวม L2, L3 cache และบัสระบบ ใช้เพื่อเข้าถึงหน่วยความจำหลักเมื่อ L1 และ L2 มี cache miss

### โจทย์ปัญหา

**4.1** แคชเซตเชื่อมโยงประกอบด้วย 64 บรรทัด หรือสล็อต แบ่งเป็นเซตสี่บรรทัด หน่วยความจำหลักมี 4K บล็อกของ 128 คำแต่ละบล็อก แสดงรูปแบบที่อยู่หน่วยความจำหลัก

## ภาคผนวก 4A: คุณลักษณะประสิทธิภาพของหน่วยความจำสองระดับ

ในบทนี้มีการกล่าวถึงแคชที่ทำหน้าที่เป็นบัฟเฟอร์ระหว่างหน่วยความจำหลักและโปรเซสเซอร์ สร้างหน่วยความจำภายในสองระดับ สถาปัตยกรรมสองระดับนี้ใช้ประโยชน์จากคุณสมบัติที่รู้จักกันในชื่อ **ความเฉพาะที่ (locality)** เพื่อให้ประสิทธิภาพดีกว่าหน่วยความจำระดับเดียวที่เปรียบได้

กลไกแคชหน่วยความจำหลักเป็นส่วนหนึ่งของสถาปัตยกรรมคอมพิวเตอร์ที่นำไปใช้ในฮาร์ดแวร์และมักไม่ปรากฏแก่ระบบปฏิบัติการ ยังมีอีกสองกรณีของการใช้หน่วยความจำสองระดับที่ใช้ประโยชน์จากความเฉพาะที่ด้วย: หน่วยความจำเสมือน (virtual memory) และแคชดิสก์ (disk cache)

### ความเฉพาะที่ (Locality)

พื้นฐานสำหรับข้อได้เปรียบด้านประสิทธิภาพของหน่วยความจำสองระดับคือหลักการที่รู้จักกันในชื่อ **การอ้างอิงเฉพาะที่ (locality of reference)** หลักการนี้ระบุว่าการอ้างอิงหน่วยความจำมีแนวโน้มที่จะรวมกลุ่ม

เหตุผลที่หลักการความเฉพาะที่สมเหตุสมผล:

1. ยกเว้นคำสั่งแตกสาขาและเรียกใช้งาน ซึ่งประกอบด้วยเพียงเศษส่วนเล็กน้อยของคำสั่งโปรแกรมทั้งหมด การทำงานของโปรแกรมเป็นแบบลำดับ ดังนั้นในกรณีส่วนใหญ่ คำสั่งถัดไปที่จะดึงมาตามคำสั่งล่าสุดทันที

2. เป็นเรื่องหายากที่จะมีลำดับการเรียกใช้งานที่ยาวนานโดยไม่ขาดตอน โปรแกรมจะอยู่ในหน้าต่างความลึกของการเรียกใช้งานขั้นตอนที่ค่อนข้างแคบ ดังนั้นในช่วงเวลาสั้น ๆ การอ้างอิงคำสั่งมีแนวโน้มที่จะเฉพาะที่กับขั้นตอนไม่กี่อย่าง

3. โครงสร้างวนซ้ำส่วนใหญ่ประกอบด้วยคำสั่งจำนวนค่อนข้างน้อยที่ทำซ้ำหลายครั้ง ตลอดระยะเวลาของการวนซ้ำ การคำนวณจึงถูกจำกัดอยู่ในส่วนเล็ก ๆ ต่อเนื่องของโปรแกรม

4. ในโปรแกรมจำนวนมาก การคำนวณส่วนใหญ่เกี่ยวข้องกับการประมวลผลโครงสร้างข้อมูล เช่น อาร์เรย์หรือลำดับเรคคอร์ด ในหลายกรณี การอ้างอิงต่อเนื่องไปยังโครงสร้างข้อมูลเหล่านี้จะเป็นไปยังรายการข้อมูลที่อยู่ใกล้ชิด

**ความแตกต่างระหว่างความเฉพาะที่สองประเภท:**

- **ความเฉพาะที่เชิงพื้นที่ (Spatial locality):** หมายถึงแนวโน้มในการทำงานที่เกี่ยวข้องกับตำแหน่งหน่วยความจำจำนวนหนึ่งที่รวมกลุ่ม สิ่งนี้สะท้อนแนวโน้มของโปรเซสเซอร์ในการเข้าถึงคำสั่งตามลำดับ โดยทั่วไปใช้ประโยชน์โดยใช้บล็อกแคชที่ใหญ่กว่าและโดยการรวมกลไกการ prefetch เข้ากับลอจิกควบคุมแคช

- **ความเฉพาะที่เชิงเวลา (Temporal locality):** หมายถึงแนวโน้มของโปรเซสเซอร์ในการเข้าถึงตำแหน่งหน่วยความจำที่เพิ่งถูกใช้ไป เช่น เมื่อลูปวนซ้ำถูกทำงาน โดยทั่วไปใช้ประโยชน์โดยเก็บค่าคำสั่งและข้อมูลที่ใช้ล่าสุดในแคชและโดยการใช้ประโยชน์จากลำดับชั้นแคช

### การทำงานของหน่วยความจำสองระดับ

คุณสมบัติความเฉพาะที่สามารถใช้ประโยชน์ในการสร้างหน่วยความจำสองระดับ หน่วยความจำระดับบน (M1) มีขนาดเล็กกว่า เร็วกว่า และแพงกว่า (ต่อบิต) มากกว่าหน่วยความจำระดับล่าง (M2) M1 ใช้เป็นที่เก็บชั่วคราวสำหรับส่วนหนึ่งของเนื้อหาของ M2 ขนาดใหญ่กว่า

ในการแสดงเวลาเฉลี่ยในการเข้าถึงรายการ:

```
Ts = H × T1 + (1 - H) × (T1 + T2)
   = T1 + (1 - H) × T2                    ...(4.2)
```

โดย:
- `Ts` = เวลาการเข้าถึงเฉลี่ย (ระบบ)
- `T1` = เวลาการเข้าถึงของ M1 (แคช)
- `T2` = เวลาการเข้าถึงของ M2 (หน่วยความจำหลัก)
- `H` = อัตราการโดน (สัดส่วนของเวลาที่พบการอ้างอิงใน M1)

### ประสิทธิภาพ (Performance)

**การวิเคราะห์ต้นทุน:**

```
Cs = (C1×S1 + C2×S2) / (S1 + S2)           ...(4.3)
```

โดย:
- `Cs` = ต้นทุนเฉลี่ยต่อบิตสำหรับหน่วยความจำรวมสองระดับ
- `C1` = ต้นทุนเฉลี่ยต่อบิตของหน่วยความจำระดับบน M1
- `C2` = ต้นทุนเฉลี่ยต่อบิตของหน่วยความจำระดับล่าง M2
- `S1` = ขนาดของ M1
- `S2` = ขนาดของ M2

เราต้องการ Cs ≈ C2 กำหนดว่า C1 >> C2 จำเป็นต้องมี S1 < S2

**การวิเคราะห์ประสิทธิภาพการเข้าถึง (Access Efficiency):**

ปริมาณ T1/Ts เรียกว่า **ประสิทธิภาพการเข้าถึง** วัดว่าเวลาการเข้าถึงเฉลี่ย (Ts) ใกล้เคียงกับเวลาการเข้าถึง M1 (T1) เพียงใด:

```
T1/Ts = 1 / [1 + (1 - H) × (T2/T1)]       ...(4.4)
```

โดยทั่วไป:
- เวลาการเข้าถึงแคชบนชิปเร็วกว่าเวลาการเข้าถึงหน่วยความจำหลักประมาณ 25 ถึง 50 เท่า (T2/T1 = 25 ถึง 50)
- เวลาการเข้าถึงแคชนอกชิปเร็วกว่าเวลาการเข้าถึงหน่วยความจำหลักประมาณ 5 ถึง 15 เท่า (T2/T1 = 5 ถึง 15)
- เวลาการเข้าถึงหน่วยความจำหลักเร็วกว่าเวลาการเข้าถึงดิสก์ประมาณ 1,000 เท่า (T2/T1 = 1,000)

ดังนั้น อัตราการโดนในช่วง 0.9 ขึ้นไปดูเหมือนจะจำเป็นเพื่อตอบสนองข้อกำหนดด้านประสิทธิภาพ

**ผลกระทบของความเฉพาะที่ต่ออัตราการโดน:**

- ถ้าไม่มีความเฉพาะที่: อัตราการโดนเป็นฟังก์ชันเชิงเส้นตรงของขนาดหน่วยความจำสัมพัทธ์
- ถ้ามีความเฉพาะที่ระดับปานกลาง: อัตราการโดนสูงกว่าสัดส่วนเชิงเส้นตรง
- ถ้ามีความเฉพาะที่ที่แข็งแกร่ง: สามารถบรรลุอัตราการโดนสูงได้แม้กับหน่วยความจำระดับบนที่ค่อนข้างเล็ก

การศึกษาจำนวนมากแสดงให้เห็นว่าขนาดแคชค่อนข้างเล็กจะให้อัตราการโดนสูงกว่า 0.75 โดยไม่คำนึงถึงขนาดหน่วยความจำหลัก แคชในช่วง 1K ถึง 128K คำมักเพียงพอ ในขณะที่หน่วยความจำหลักในปัจจุบันมักอยู่ในช่วงกิกะไบต์
