# 4.3 Mapping Functions

เนื่องจากจำนวน Line ใน Cache มีน้อยกว่า Block ใน Main Memory จึงต้องมีฟังก์ชันการจับคู่ตำแหน่ง

## 1. Direct Mapping
* **หลักการ:** แต่ละ Block ใน Main Memory จะถูกกำหนดให้ลงได้เพียง Line เดียวที่ระบุไว้เท่านั้น ($i = j \text{ modulo } m$)
* **ข้อดี:** ง่ายและต้นทุนต่ำ
* **ข้อเสีย:** หากโปรแกรมเรียกใช้ข้อมูลสอง Block ที่ต้องลง Line เดียวกันสลับไปมา จะเกิดการลบและเขียนทับกันตลอดเวลา (Thrashing)

## 2. Associative Mapping
* **หลักการ:** ทุก Block ใน Main Memory สามารถวางใน Line ใดก็ได้ของ Cache อย่างอิสระ
* **ข้อดี:** ยืดหยุ่นสูงสุด และช่วยเพิ่ม Hit Ratio ได้ดี
* **ข้อเสีย:** ต้องมีวงจรซับซ้อนเพื่อเปรียบเทียบ Tag ของทุก Line ใน Cache พร้อมกัน

## 3. Set-Associative Mapping
* **หลักการ:** เป็นการผสมผสาน โดยแบ่ง Cache ออกเป็นชุด (Sets) แต่ละชุดมีจำนวน Line คงที่ (เช่น k-way) 
* Block หนึ่งจะถูกระบุให้ลงได้เฉพาะใน Set ที่กำหนด แต่ภายใน Set นั้นจะวางลง Line ใดก็ได้
* เป็นวิธีที่นิยมที่สุดเพราะให้ความสมดุลระหว่างความเร็วและความซับซ้อนของวงจร
