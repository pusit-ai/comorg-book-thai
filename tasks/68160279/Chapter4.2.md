# 4.2 Cache Memory Principles

Cache Memory ออกแบบมาเพื่อรวมข้อดีของหน่วยความจำความเร็วสูงราคาแพง เข้ากับหน่วยความจำความจุสูงราคาถูก

## กลไกพื้นฐาน
* Cache จะเก็บสำเนาของข้อมูลบางส่วนจาก Main Memory
* เมื่อ Processor ต้องการอ่านข้อมูล ระบบจะตรวจสอบใน Cache ก่อน
* **Cache Hit:** หากพบข้อมูล ระบบจะส่งให้ Processor ทันที
* **Cache Miss:** หากไม่พบ ข้อมูลหนึ่งบล็อก (Block) จะถูกดึงจาก Main Memory มาเก็บใน Cache แล้วจึงส่งให้ Processor

## หลักการ Locality of Reference
ประสิทธิภาพของ Cache ขึ้นอยู่กับปรากฏการณ์ที่ว่า การอ้างอิงหน่วยความจำของโปรแกรมมักจะมีลักษณะเป็นกลุ่ม (Clustered) ได้แก่:
* **Temporal Locality:** ข้อมูลที่เพิ่งถูกใช้งานมีแนวโน้มจะถูกใช้งานซ้ำอีกในอนาคตอันใกล้
* **Spatial Locality:** ข้อมูลที่อยู่ใกล้กับตำแหน่งที่เพิ่งถูกเรียกใช้งาน มีแนวโน้มจะถูกเรียกใช้งานตามมา

## องค์ประกอบการออกแบบ (Elements of Cache Design)
* **Cache Size:** ขนาดที่เหมาะสมเพื่อให้ได้ความเร็วใกล้เคียง Cache และราคาใกล้เคียง Main Memory
* **Mapping Function:** วิธีระบุว่า Block ใดใน Memory จะไปวางที่ Line ใดใน Cache
* **Replacement Algorithm:** วิธีเลือก Block เก่าออกเมื่อต้องนำข้อมูลใหม่เข้า
* **Write Policy:** วิธีกำหนดว่าเมื่อมีการเขียนข้อมูลใน Cache จะอัปเดต Main Memory อย่างไร
