# 4.5 คำศัพท์สำคัญ คำถามทบทวน และโจทย์ปัญหา (Key Terms, Review Questions, and Problems)

## บทนำ

หัวข้อ 4.5 เป็นส่วนสรุปเชิงวิชาการของบทว่าด้วยหน่วยความจำแคช (Cache Memory) โดยมีบทบาทสำคัญ 3 ประการ ได้แก่ การรวบรวมคำศัพท์หลักที่ใช้ตลอดบท การตั้งคำถามทบทวนเพื่อประเมินความเข้าใจเชิงแนวคิด และการเสนอโจทย์ปัญหาเพื่อฝึกการวิเคราะห์เชิงสถาปัตยกรรมคอมพิวเตอร์ (Computer Architecture) ในระดับที่ลึกขึ้น

**สาระสำคัญของหัวข้อนี้มิได้มีเพียงการรวบรวมรายการคำหรือชุดคำถามเท่านั้น แต่ยังสะท้อนให้เห็นกรอบความคิดหลักของทั้งบท ได้แก่ ความสัมพันธ์ระหว่างโครงสร้างหน่วยความจำ (Memory Organization) การแมปแคช (Cache Mapping) นโยบายการแทนที่ข้อมูล (Replacement Policy) นโยบายการเขียน (Write Policy) และผลกระทบต่อสมรรถนะของระบบโดยรวม**

ในที่นี้ได้แปลและเรียบเรียงใหม่เป็นภาษาไทยเชิงวิชาการ โดยคงลำดับและรายละเอียดของต้นฉบับไว้อย่างครบถ้วน พร้อมขยายความศัพท์และเจตนาของคำถามแต่ละส่วนให้ชัดเจนยิ่งขึ้น

## 1. คำศัพท์สำคัญ (Key Terms)

คำศัพท์ต่อไปนี้เป็นแกนกลางของการทำความเข้าใจเรื่องหน่วยความจำแคช (Cache Memory) และลำดับชั้นของหน่วยความจำ (Memory Hierarchy) ในระบบคอมพิวเตอร์

| คำศัพท์ภาษาอังกฤษ | คำเรียกภาษาไทย | คำอธิบายเชิงวิชาการ |
|---|---|---|
| Access Time | เวลาเข้าถึง (Access Time) | ระยะเวลาที่ระบบใช้ในการเข้าถึงข้อมูลหรือคำสั่งจากหน่วยความจำหนึ่งระดับใดระดับหนึ่ง ซึ่งเป็นตัวชี้วัดพื้นฐานของความเร็วในการทำงานของหน่วยความจำนั้น |
| Associative Mapping | การแมปแบบแอสโซซิเอทีฟ (Associative Mapping) | วิธีการแมประหว่างบล็อกจากหน่วยความจำหลัก (Main Memory) กับแคช (Cache) ที่เปิดโอกาสให้บล็อกใด ๆ สามารถถูกเก็บในบรรทัดใดก็ได้ของแคช ทำให้ยืดหยุ่นสูง แต่ต้องแลกกับฮาร์ดแวร์เปรียบเทียบแท็ก (Tag Comparison) ที่ซับซ้อนขึ้น |
| Cache Hit | การพบข้อมูลในแคช (Cache Hit) | สถานะที่ข้อมูลหรือคำสั่งที่หน่วยประมวลผลร้องขอมีอยู่แล้วในแคช ทำให้สามารถตอบสนองได้รวดเร็วโดยไม่ต้องอ้างอิงไปยังหน่วยความจำหลัก |
| Cache Line | บรรทัดแคช (Cache Line) | หน่วยจัดเก็บพื้นฐานของแคช ซึ่งมักบรรจุข้อมูลเป็นบล็อก (Block) จากหน่วยความจำหลักพร้อมแท็กและบิตควบคุมต่าง ๆ |
| Cache Memory | หน่วยความจำแคช (Cache Memory) | หน่วยความจำความเร็วสูงที่วางอยู่ระหว่างหน่วยประมวลผลกลาง (Central Processing Unit: CPU) กับหน่วยความจำหลัก เพื่อเก็บข้อมูลที่มีแนวโน้มถูกใช้งานบ่อยหรือใกล้เคียงกันตามหลักการเฉพาะที่ (Locality) |
| Cache Miss | การพลาดแคช (Cache Miss) | สถานะที่ข้อมูลที่ร้องขอไม่ปรากฏอยู่ในแคช จึงต้องไปดึงข้อมูลจากหน่วยความจำระดับล่าง ซึ่งก่อให้เกิดค่าเสียเวลาเพิ่มเติม (Miss Penalty) |
| Cache Set | ชุดของแคช (Cache Set) | กลุ่มของบรรทัดแคชที่ใช้ในการจัดวางข้อมูลภายใต้การแมปแบบเซต-แอสโซซิเอทีฟ (Set-Associative Mapping) โดยหนึ่งบล็อกจากหน่วยความจำหลักจะถูกแมปเข้าสู่ชุดใดชุดหนึ่ง |
| Data Cache | แคชข้อมูล (Data Cache) | แคชที่ออกแบบมาเพื่อเก็บข้อมูลที่โปรแกรมใช้งาน เช่น ตัวแปร อาร์เรย์ หรือโครงสร้างข้อมูล โดยแยกจากแคชคำสั่งในบางสถาปัตยกรรม |
| Direct Access | การเข้าถึงโดยตรง (Direct Access) | วิธีเข้าถึงข้อมูลที่แม้จะไม่สุ่มโดยสมบูรณ์ แต่สามารถเคลื่อนไปยังตำแหน่งเป้าหมายได้ค่อนข้างตรง เช่น การเข้าถึงข้อมูลบนดิสก์ |
| Direct Mapping | การแมปโดยตรง (Direct Mapping) | วิธีการแมปที่บล็อกหนึ่งจากหน่วยความจำหลักสามารถอยู่ได้เพียงตำแหน่งเดียวในแคช ทำให้วงจรง่ายและเข้าถึงเร็ว แต่มีโอกาสเกิดความขัดแย้งของการแมป (Conflict Miss) สูง |
| High-Performance Computing (HPC) | การประมวลผลสมรรถนะสูง (High-Performance Computing: HPC) | แนวทางการประมวลผลที่มุ่งเน้นพลังคำนวณสูงมาก ซึ่งการออกแบบระบบหน่วยความจำและแคชมีผลอย่างยิ่งต่อสมรรถนะโดยรวม |
| Hit | การฮิต (Hit) | คำเรียกโดยย่อของสถานะที่คำร้องขอข้อมูลประสบความสำเร็จในระดับหน่วยความจำที่กำลังตรวจสอบอยู่ โดยในบริบทนี้มักหมายถึงการพบข้อมูลในแคช |
| Hit Ratio | อัตราการฮิต (Hit Ratio) | สัดส่วนของจำนวนครั้งที่ข้อมูลถูกพบในแคชเมื่อเทียบกับจำนวนครั้งของการอ้างอิงหน่วยความจำทั้งหมด เป็นตัวชี้วัดสำคัญของประสิทธิภาพแคช |
| Instruction Cache | แคชคำสั่ง (Instruction Cache) | แคชที่ใช้เก็บคำสั่งของโปรแกรมโดยเฉพาะ เพื่อให้หน่วยประมวลผลสามารถดึงคำสั่งได้รวดเร็วขึ้น |
| L1 Cache | แคชระดับที่ 1 (Level 1 Cache: L1 Cache) | แคชที่อยู่ใกล้แกนประมวลผลมากที่สุด มีขนาดเล็กแต่เข้าถึงได้เร็วที่สุด |
| L2 Cache | แคชระดับที่ 2 (Level 2 Cache: L2 Cache) | แคชระดับถัดจาก L1 ซึ่งมักมีขนาดใหญ่ขึ้นแต่ช้าลงเล็กน้อย ใช้ลดภาระของการเข้าถึงหน่วยความจำหลัก |
| L3 Cache | แคชระดับที่ 3 (Level 3 Cache: L3 Cache) | แคชขนาดใหญ่กว่าระดับก่อนหน้า มักถูกใช้ร่วมกันระหว่างหลายแกนประมวลผลในระบบสมัยใหม่ |
| Line | บรรทัด (Line) | คำเรียกย่อของบรรทัดแคช (Cache Line) ซึ่งเป็นช่องจัดเก็บข้อมูลหนึ่งหน่วยในแคช |
| Locality | หลักการเฉพาะที่ (Locality) | หลักการที่อธิบายว่าโปรแกรมมักเข้าถึงข้อมูลหรือคำสั่งที่อยู่ใกล้เคียงกันในเชิงเวลาและเชิงตำแหน่ง จึงเป็นพื้นฐานสำคัญที่ทำให้แคชมีประสิทธิผล |
| Logical Cache | แคชเชิงตรรกะ (Logical Cache) | แนวคิดของแคชที่อ้างอิงตามที่อยู่ตรรกะหรือที่อยู่เสมือน (Virtual/Logical Address) แทนที่จะอ้างอิงตามที่อยู่จริงโดยตรง |
| Memory Hierarchy | ลำดับชั้นหน่วยความจำ (Memory Hierarchy) | โครงสร้างการจัดวางหน่วยความจำหลายระดับตั้งแต่รีจิสเตอร์ แคช หน่วยความจำหลัก ไปจนถึงหน่วยเก็บข้อมูลรอง เพื่อสร้างสมดุลระหว่างความเร็ว ความจุ และต้นทุน |
| Miss | การพลาด (Miss) | สถานะตรงข้ามกับฮิต หมายถึงการไม่พบข้อมูลในระดับหน่วยความจำที่กำลังตรวจสอบ |
| Multilevel Cache | แคชหลายระดับ (Multilevel Cache) | การใช้แคชมากกว่าหนึ่งระดับ เช่น L1, L2 และ L3 เพื่อผสานข้อดีของแคชขนาดเล็กที่เร็วกับแคชขนาดใหญ่ที่เก็บข้อมูลได้มาก |
| Physical Address | ที่อยู่จริง (Physical Address) | ที่อยู่ของข้อมูลในหน่วยความจำหลักจริงหลังผ่านกระบวนการแปลที่อยู่แล้ว |
| Physical Cache | แคชเชิงกายภาพ (Physical Cache) | แคชที่ทำงานโดยอิงกับที่อยู่จริง (Physical Address) ของหน่วยความจำ |
| Random Access | การเข้าถึงแบบสุ่ม (Random Access) | วิธีเข้าถึงข้อมูลที่สามารถอ้างถึงตำแหน่งใดก็ได้โดยใช้เวลาใกล้เคียงกัน เช่น หน่วยความจำหลักแบบแรม (RAM) |
| Replacement Algorithm | ขั้นตอนวิธีการแทนที่ข้อมูล (Replacement Algorithm) | ชุดกฎที่ใช้ตัดสินใจว่าควรนำบรรทัดแคชใดออก เมื่อแคชเต็มและจำเป็นต้องนำบล็อกใหม่เข้ามา |
| Secondary Memory | หน่วยความจำสำรอง (Secondary Memory) | สื่อเก็บข้อมูลที่มีความจุมากแต่ช้ากว่า เช่น ฮาร์ดดิสก์หรือโซลิดสเตตไดรฟ์ ซึ่งใช้เก็บข้อมูลระยะยาว |
| Sequential Access | การเข้าถึงแบบลำดับ (Sequential Access) | วิธีเข้าถึงข้อมูลที่ต้องไล่ผ่านรายการข้อมูลตามลำดับ เช่น เทปแม่เหล็ก |
| Set-Associative Mapping | การแมปแบบเซต-แอสโซซิเอทีฟ (Set-Associative Mapping) | วิธีการแมปที่เป็นจุดสมดุลระหว่างการแมปโดยตรงกับการแมปแบบแอสโซซิเอทีฟ กล่าวคือ บล็อกหนึ่งจะถูกจำกัดให้อยู่ได้เฉพาะภายในชุดหนึ่ง แต่เลือกได้หลายบรรทัดภายในชุดนั้น |
| Spatial Locality | หลักการเฉพาะที่เชิงพื้นที่ (Spatial Locality) | แนวโน้มที่โปรแกรมจะเข้าถึงข้อมูลหรือคำสั่งที่มีตำแหน่งใกล้เคียงกับตำแหน่งที่เพิ่งเข้าถึง |
| Split Cache | แคชแยก (Split Cache) | สถาปัตยกรรมแคชที่แยกส่วนเก็บคำสั่ง (Instruction Cache) และส่วนเก็บข้อมูล (Data Cache) ออกจากกัน |
| Tag | แท็ก (Tag) | ส่วนของที่อยู่ซึ่งถูกเก็บไว้ในแคชเพื่อใช้ตรวจสอบว่าข้อมูลในบรรทัดแคชตรงกับที่อยู่ที่ร้องขอหรือไม่ |
| Temporal Locality | หลักการเฉพาะที่เชิงเวลา (Temporal Locality) | แนวโน้มที่ข้อมูลหรือคำสั่งที่เพิ่งถูกใช้งานจะมีโอกาสถูกใช้อีกในเวลาอันใกล้ |
| Unified Cache | แคชรวม (Unified Cache) | แคชที่ใช้เก็บทั้งคำสั่งและข้อมูลในพื้นที่เดียวกัน ไม่ได้แยกออกเป็นสองแคชเฉพาะทาง |
| Virtual Address | ที่อยู่เสมือน (Virtual Address) | ที่อยู่ที่โปรแกรมหรือกระบวนการใช้อ้างอิงก่อนจะถูกแปลไปเป็นที่อยู่จริง |
| Virtual Cache | แคชเชิงเสมือน (Virtual Cache) | แคชที่ใช้ที่อยู่เสมือนเป็นฐานในการอ้างอิงและค้นหา |
| Write Back | การเขียนกลับ (Write Back) | นโยบายการเขียนข้อมูลที่บันทึกการเปลี่ยนแปลงไว้ในแคชก่อน แล้วจึงเขียนกลับสู่หน่วยความจำหลักเมื่อมีการแทนที่หรือเมื่อถึงเงื่อนไขที่กำหนด |
| Write Through | การเขียนผ่าน (Write Through) | นโยบายการเขียนที่เขียนข้อมูลลงแคชและหน่วยความจำหลักพร้อมกัน เพื่อคงความสอดคล้องของข้อมูล แต่เพิ่มทราฟฟิกการเขียน |

### ข้อสังเกตเชิงวิชาการ

- คำศัพท์ในหมวดนี้ไม่ได้แยกจากกันโดยเด็ดขาด แต่เชื่อมโยงกันเป็นเครือข่ายแนวคิดเดียวกันของสถาปัตยกรรมหน่วยความจำ
- กลุ่มคำอย่าง `hit`, `miss`, `hit ratio`, `replacement algorithm`, และ `write policy` เป็นกลุ่มที่ใช้วัดและควบคุมสมรรถนะของแคชโดยตรง
- กลุ่มคำอย่าง `direct mapping`, `associative mapping`, และ `set-associative mapping` เป็นแกนกลางของการออกแบบเชิงโครงสร้าง
- กลุ่มคำอย่าง `spatial locality` และ `temporal locality` เป็นหลักการเชิงพฤติกรรมของโปรแกรม ซึ่งอธิบายว่าทำไมแคชจึงช่วยเพิ่มประสิทธิภาพได้จริง

## 2. คำถามทบทวน (Review Questions)

คำถามทบทวนในส่วนนี้มุ่งตรวจสอบความเข้าใจเชิงแนวคิด ไม่ได้เน้นการคำนวณเชิงตัวเลขเพียงอย่างเดียว แต่ต้องอาศัยการเชื่อมโยงหลักการจากทั้งบท

### 2.1 คำถามที่ 4.1

**การเข้าถึงแบบลำดับ (Sequential Access), การเข้าถึงโดยตรง (Direct Access) และการเข้าถึงแบบสุ่ม (Random Access) แตกต่างกันอย่างไร**

คำถามข้อนี้ต้องการให้ผู้เรียนแยกความแตกต่างของวิธีเข้าถึงข้อมูลในหน่วยความจำหรือสื่อบันทึกแต่ละชนิด โดยพิจารณาทั้งลักษณะการค้นหาตำแหน่งข้อมูล เวลาเข้าถึง และผลต่อการออกแบบระบบ

### 2.2 คำถามที่ 4.2

**โดยทั่วไปแล้ว เวลาเข้าถึง (Access Time), ต้นทุนของหน่วยความจำ (Memory Cost) และความจุ (Capacity) มีความสัมพันธ์กันอย่างไร**

สาระของคำถามข้อนี้คือการทำความเข้าใจข้อแลกเปลี่ยนพื้นฐานของลำดับชั้นหน่วยความจำ กล่าวคือ หน่วยความจำที่เร็วมากมักมีราคาสูงและความจุต่ำ ขณะที่หน่วยความจำที่มีความจุสูงมักช้ากว่าและมีต้นทุนต่อบิตต่ำกว่า

### 2.3 คำถามที่ 4.3

**หลักการเฉพาะที่ (Principle of Locality) สัมพันธ์กับการใช้หน่วยความจำหลายระดับ (Multiple Memory Levels) อย่างไร**

คำถามนี้ชี้ให้ผู้เรียนเชื่อมโยงพฤติกรรมการอ้างอิงหน่วยความจำของโปรแกรมกับเหตุผลที่ระบบคอมพิวเตอร์นิยมสร้างลำดับชั้นหน่วยความจำหลายระดับ เช่น แคชหลายชั้นและหน่วยความจำหลัก

### 2.4 คำถามที่ 4.4

**การแมปโดยตรง (Direct Mapping), การแมปแบบแอสโซซิเอทีฟ (Associative Mapping) และการแมปแบบเซต-แอสโซซิเอทีฟ (Set-Associative Mapping) แตกต่างกันอย่างไร**

ผู้ตอบควรอธิบายทั้งในมิติของความยืดหยุ่นในการจัดวางบล็อก ความซับซ้อนของฮาร์ดแวร์ เวลาเข้าถึง และโอกาสเกิดความขัดแย้งในการแมป

### 2.5 คำถามที่ 4.5

**สำหรับแคชแบบแมปโดยตรง (Direct-Mapped Cache) ที่อยู่ของหน่วยความจำหลัก (Main Memory Address) สามารถมองได้ว่าแบ่งออกเป็น 3 ฟิลด์ (Fields) อะไรบ้าง จงระบุและให้ความหมายของแต่ละฟิลด์**

คำถามนี้ตรวจว่าผู้เรียนเข้าใจโครงสร้างที่อยู่ในระบบแคชแบบแมปโดยตรง โดยทั่วไปมักเกี่ยวข้องกับแท็ก (Tag), หมายเลขบรรทัด (Line) และตำแหน่งคำหรือไบต์ภายในบล็อก (Word/Byte Offset)

### 2.6 คำถามที่ 4.6

**สำหรับแคชแบบแอสโซซิเอทีฟ (Associative Cache) ที่อยู่ของหน่วยความจำหลักประกอบด้วย 2 ฟิลด์ใดบ้าง จงระบุและอธิบาย**

คำถามนี้เน้นการเปรียบเทียบว่าเมื่อไม่มีข้อจำกัดด้านตำแหน่งตายตัวเหมือน Direct Mapping โครงสร้างฟิลด์ของที่อยู่จะเปลี่ยนไปอย่างไร

### 2.7 คำถามที่ 4.7

**สำหรับแคชแบบเซต-แอสโซซิเอทีฟ (Set-Associative Cache) ที่อยู่ของหน่วยความจำหลักประกอบด้วย 3 ฟิลด์ใดบ้าง จงระบุและอธิบาย**

จุดสำคัญของคำถามนี้อยู่ที่การแยกความเข้าใจระหว่างส่วน `tag`, `set` และ `word/byte offset` ซึ่งเป็นพื้นฐานของการวิเคราะห์การแมปในระบบแคชแบบหลายทาง (n-way)

### 2.8 คำถามที่ 4.8

**หลักการเฉพาะที่เชิงพื้นที่ (Spatial Locality) และหลักการเฉพาะที่เชิงเวลา (Temporal Locality) แตกต่างกันอย่างไร**

ผู้เรียนต้องอธิบายความแตกต่างระหว่างการกลับไปใช้ข้อมูลเดิมซ้ำในช่วงเวลาใกล้กัน กับการเข้าถึงข้อมูลตำแหน่งที่อยู่ใกล้กันในหน่วยความจำ

### 2.9 คำถามที่ 4.9

**โดยทั่วไปแล้ว มีกลยุทธ์ใดบ้างที่ใช้ในการใช้ประโยชน์จากหลักการเฉพาะที่เชิงพื้นที่ (Spatial Locality) และหลักการเฉพาะที่เชิงเวลา (Temporal Locality)**

คำถามนี้มีนัยให้ตอบถึงกลยุทธ์เชิงสถาปัตยกรรม เช่น การใช้บล็อกขนาดเหมาะสม การพรีเฟตช์ (Prefetching) การเก็บข้อมูลที่ใช้ล่าสุดไว้ในแคช และการใช้แคชหลายระดับ

## 3. โจทย์ปัญหา (Problems)

โจทย์ในส่วนนี้พัฒนาไปไกลกว่าคำถามทบทวน เพราะต้องอาศัยทั้งการตีความโจทย์ การออกแบบโครงสร้างบิตของที่อยู่ การคำนวณจำนวนบล็อกหรือจำนวนเซต ตลอดจนการวิเคราะห์ผลกระทบต่อสมรรถนะของระบบ

### 3.1 โจทย์ 4.1

แคชแบบเซต-แอสโซซิเอทีฟ (Set-Associative Cache) หนึ่งระบบประกอบด้วย 64 บรรทัด (Lines หรือ Slots) โดยแบ่งออกเป็นชุด (Sets) ชุดละ 4 บรรทัด หน่วยความจำหลักประกอบด้วย 4K บล็อก (Blocks) และแต่ละบล็อกมีขนาด 128 คำ (Words)

**จงแสดงรูปแบบของที่อยู่หน่วยความจำหลัก (Main Memory Address Format)**

### 3.2 โจทย์ 4.2

แคชแบบเซต-แอสโซซิเอทีฟชนิดสองทาง (Two-Way Set-Associative Cache) มีขนาดบรรทัดละ 16 ไบต์ และมีขนาดรวมทั้งหมด 8 กิโลไบต์ (8 kB) ขณะที่หน่วยความจำหลักมีขนาด 64 เมกะไบต์ (64 MB) และเป็นหน่วยความจำแบบระบุตำแหน่งได้ถึงระดับไบต์ (Byte Addressable)

**จงแสดงรูปแบบของที่อยู่หน่วยความจำหลัก**

### 3.3 โจทย์ 4.3

สำหรับที่อยู่หน่วยความจำหลักในเลขฐานสิบหก (Hexadecimal) ต่อไปนี้ `111111`, `666666`, และ `BBBBBB`

**จงหาข้อมูลต่อไปนี้โดยตอบในรูปเลขฐานสิบหก**

- สำหรับแคชแบบแมปโดยตรง (Direct-Mapped Cache) โดยใช้รูปแบบเดียวกับ Figure 4.10:
  `Tag`, `Line`, และ `Word`
- สำหรับแคชแบบแอสโซซิเอทีฟ (Associative Cache) โดยใช้รูปแบบเดียวกับ Figure 4.12:
  `Tag` และ `Word`
- สำหรับแคชแบบเซต-แอสโซซิเอทีฟสองทาง (Two-Way Set-Associative Cache) โดยใช้รูปแบบเดียวกับ Figure 4.15:
  `Tag`, `Set`, และ `Word`

### 3.4 โจทย์ 4.4

**จงระบุค่าต่อไปนี้**

- สำหรับตัวอย่างแคชแบบแมปโดยตรงใน Figure 4.10:
  ความยาวที่อยู่ (Address Length), จำนวนหน่วยที่อ้างตำแหน่งได้ (Number of Addressable Units), ขนาดบล็อก (Block Size), จำนวนบล็อกในหน่วยความจำหลัก (Number of Blocks in Main Memory), จำนวนบรรทัดในแคช (Number of Lines in Cache), และขนาดของแท็ก (Tag Size)
- สำหรับตัวอย่างแคชแบบแอสโซซิเอทีฟใน Figure 4.12:
  ความยาวที่อยู่, จำนวนหน่วยที่อ้างตำแหน่งได้, ขนาดบล็อก, จำนวนบล็อกในหน่วยความจำหลัก, จำนวนบรรทัดในแคช, และขนาดของแท็ก
- สำหรับตัวอย่างแคชแบบเซต-แอสโซซิเอทีฟสองทางใน Figure 4.15:
  ความยาวที่อยู่, จำนวนหน่วยที่อ้างตำแหน่งได้, ขนาดบล็อก, จำนวนบล็อกในหน่วยความจำหลัก, จำนวนบรรทัดต่อชุด (Lines in Set), จำนวนชุด (Number of Sets), จำนวนบรรทัดทั้งหมดในแคช, และขนาดของแท็ก

### 3.5 โจทย์ 4.5

พิจารณาไมโครโปรเซสเซอร์ (Microprocessor) แบบ 32 บิต ที่มีแคชบนชิป (On-Chip Cache) ขนาด 16 กิโลไบต์ ซึ่งเป็นแคชแบบเซต-แอสโซซิเอทีฟ 4 ทาง (Four-Way Set-Associative Cache) สมมติว่าแคชนี้มีขนาดบรรทัดเท่ากับ 4 คำ โดยแต่ละคำมีขนาด 32 บิต

**จงวาดแผนภาพบล็อก (Block Diagram) ของแคชนี้ แสดงการจัดองค์กรภายใน และอธิบายว่าฟิลด์ต่าง ๆ ของที่อยู่ถูกนำมาใช้เพื่อตัดสินว่าเกิดแคชฮิตหรือแคชมิสอย่างไร**

นอกจากนี้

**คำจากตำแหน่งหน่วยความจำ `ABCDE8F8` จะถูกแมปไปเก็บไว้ตรงส่วนใดของแคช**

### 3.6 โจทย์ 4.6

กำหนดคุณลักษณะของหน่วยความจำแคชภายนอก (External Cache Memory) ดังนี้

- เป็นแคชแบบเซต-แอสโซซิเอทีฟ 4 ทาง (Four-Way Set Associative)
- มีขนาดบรรทัดเท่ากับ 2 คำ โดยแต่ละคำมีขนาด 16 บิต
- สามารถรองรับข้อมูลจากหน่วยความจำหลักรวมได้ 4K คำ โดยอ้างอิงเป็นคำขนาด 32 บิต
- ใช้ร่วมกับโปรเซสเซอร์ขนาด 16 บิต ที่สร้างที่อยู่ขนาด 24 บิต

**จงออกแบบโครงสร้างของแคชโดยให้มีข้อมูลสำคัญครบถ้วน และแสดงว่าระบบตีความที่อยู่จากโปรเซสเซอร์อย่างไร**

### 3.7 โจทย์ 4.7

หน่วยประมวลผล Intel 80486 มีแคชบนชิปแบบรวม (Unified On-Chip Cache) ขนาด 8 kB มีโครงสร้างแบบเซต-แอสโซซิเอทีฟ 4 ทาง และมีความยาวบล็อกเท่ากับ 4 คำ โดยแต่ละคำเป็น 32 บิต แคชนี้ถูกจัดเป็น 128 ชุด และในแต่ละบรรทัดมีบิตตรวจสอบความถูกต้องของบรรทัด (Line Valid Bit) 1 บิต พร้อมบิต `B0`, `B1`, และ `B2` ซึ่งใช้กับกลไก LRU

เมื่อเกิดแคชมิส Intel 80486 จะอ่านบรรทัดข้อมูลขนาด 16 ไบต์จากหน่วยความจำหลักด้วยการอ่านแบบเบิร์สต์ (Bus Memory Read Burst)

**จงวาดแผนภาพอย่างย่อของแคชนี้ และแสดงว่าฟิลด์ต่าง ๆ ของที่อยู่ถูกตีความอย่างไร**

### 3.8 โจทย์ 4.8

พิจารณาเครื่องคอมพิวเตอร์ที่มีหน่วยความจำหลักแบบระบุตำแหน่งได้ถึงระดับไบต์ (Byte Addressable Main Memory) ขนาด `2^16` ไบต์ และมีขนาดบล็อกเท่ากับ 8 ไบต์ สมมติว่าใช้แคชแบบแมปโดยตรง (Direct Mapped Cache) ซึ่งประกอบด้วย 32 บรรทัด

**จงตอบคำถามต่อไปนี้**

- ที่อยู่หน่วยความจำขนาด 16 บิตนี้ถูกแบ่งออกเป็น `tag`, `line number` และ `byte number` อย่างไร
- ไบต์ที่มีที่อยู่ต่อไปนี้จะถูกเก็บลงในบรรทัดใดของแคช
  `0001 0001 0001 1011`
  `1100 0011 0011 0100`
  `1101 0000 0001 1101`
  `1010 1010 1010 1010`
- สมมติว่าไบต์ที่มีที่อยู่ `0001 1010 0001 1010` ถูกเก็บอยู่ในแคชแล้ว ไบต์อื่น ๆ ที่ถูกเก็บอยู่ในบล็อกเดียวกันมีที่อยู่ใดบ้าง
- โดยรวมแล้ว แคชนี้สามารถเก็บข้อมูลได้กี่ไบต์
- เพราะเหตุใดจึงต้องเก็บแท็ก (Tag) ไว้ในแคชด้วย

### 3.9 โจทย์ 4.9

สำหรับแคชบนชิปของ Intel 80486 มีการใช้ขั้นตอนวิธีการแทนที่ข้อมูลแบบที่เรียกว่า **Pseudo Least Recently Used (Pseudo-LRU)** โดยในแต่ละหนึ่งใน 128 ชุดของแคช ซึ่งแต่ละชุดมี 4 บรรทัด (กำกับเป็น `L0`, `L1`, `L2`, `L3`) จะมีบิต `B0`, `B1`, และ `B2` ประกอบอยู่

กลไกการแทนที่ทำงานดังนี้

- เมื่อจำเป็นต้องแทนที่บรรทัดหนึ่ง ระบบจะตรวจสอบก่อนว่าการใช้งานล่าสุดอยู่ในกลุ่ม `L0/L1` หรือ `L2/L3`
- จากนั้นจึงพิจารณาต่อไปว่าภายในคู่นั้น บล็อกใดเป็นบล็อกที่ถูกใช้งานล่าสุดน้อยที่สุด
- ตรรกะดังกล่าวแสดงไว้ใน Figure 4.19

**จงตอบคำถามต่อไปนี้**

- อธิบายว่าบิต `B0`, `B1`, และ `B2` ถูกกำหนดค่าอย่างไร และใช้งานอย่างไรในขั้นตอนวิธีการแทนที่ตาม Figure 4.19
- แสดงให้เห็นว่าขั้นตอนวิธีของ 80486 เป็นเพียงการประมาณค่าอัลกอริทึม LRU ที่แท้จริง โดยคำใบ้คือให้พิจารณากรณีที่ลำดับการใช้งานล่าสุดคือ `L0`, `L2`, `L3`, `L1`
- พิสูจน์ว่า หากต้องการใช้อัลกอริทึม LRU ที่แท้จริง จะต้องใช้บิตถึง 6 บิตต่อหนึ่งชุด

### 3.10 โจทย์ 4.10

แคชแบบเซต-แอสโซซิเอทีฟหนึ่งระบบมีขนาดบล็อกเท่ากับ 4 คำ โดยแต่ละคำมีขนาด 16 บิต และมีขนาดชุด (Set Size) เท่ากับ 2 ทั้งระบบแคชสามารถรองรับข้อมูลรวมได้ 4096 คำ ส่วนขนาดของหน่วยความจำหลักที่สามารถนำมาแคชได้คือ `64K × 32 bits`

**จงออกแบบโครงสร้างของแคช และแสดงว่าแอดเดรสของโปรเซสเซอร์ถูกตีความอย่างไร**

### 3.11 โจทย์ 4.11

พิจารณาระบบหน่วยความจำที่ใช้แอดเดรสขนาด 32 บิตในระดับไบต์ (Byte-Level Addressing) และมีแคชที่ใช้ขนาดบรรทัดเท่ากับ 64 ไบต์

**จงวิเคราะห์ในกรณีต่อไปนี้**

- กรณีแคชแบบแมปโดยตรง (Direct Mapped Cache) ที่ฟิลด์แท็กในที่อยู่มีขนาด 20 บิต:
  แสดงรูปแบบที่อยู่ และหาค่าต่อไปนี้ ได้แก่ จำนวนหน่วยที่อ้างตำแหน่งได้ จำนวนบล็อกในหน่วยความจำหลัก จำนวนบรรทัดในแคช และขนาดของแท็ก
- กรณีแคชแบบแอสโซซิเอทีฟ (Associative Cache):
  แสดงรูปแบบที่อยู่ และหาจำนวนหน่วยที่อ้างตำแหน่งได้ จำนวนบล็อกในหน่วยความจำหลัก จำนวนบรรทัดในแคช และขนาดของแท็ก
- กรณีแคชแบบเซต-แอสโซซิเอทีฟ 4 ทาง (Four-Way Set-Associative Cache) ที่ฟิลด์แท็กมีขนาด 9 บิต:
  แสดงรูปแบบที่อยู่ และหาจำนวนหน่วยที่อ้างตำแหน่งได้ จำนวนบล็อกในหน่วยความจำหลัก จำนวนบรรทัดต่อชุด จำนวนชุดในแคช จำนวนบรรทัดทั้งหมดในแคช และขนาดของแท็ก

- ขนาดคำ (Word Size) เท่ากับ 1 ไบต์
- ขนาดบล็อกเท่ากับ 16 ไบต์

**จงตอบคำถามต่อไปนี้**

- สำหรับที่อยู่หน่วยความจำหลัก `F0010`, `01234` และ `CABBE` จงหาค่า `tag`, ที่อยู่บรรทัดแคช (Cache Line Address) และออฟเซตของคำ (Word Offsets) สำหรับแคชแบบแมปโดยตรง
- จงยกตัวอย่างที่อยู่หน่วยความจำหลัก 2 ค่า ที่มีแท็กต่างกัน แต่แมปเข้าสู่ช่องแคช (Cache Slot) เดียวกันในกรณีแคชแบบแมปโดยตรง
- สำหรับที่อยู่ `F0010` และ `CABBE` จงหาค่า `tag` และ `offset` ของแคชแบบแอสโซซิเอทีฟสมบูรณ์ (Fully Associative Cache)
- สำหรับที่อยู่ `F0010` และ `CABBE` จงหาค่า `tag`, `cache set` และ `offset` ของแคชแบบเซต-แอสโซซิเอทีฟ 2 ทาง

### 3.19 โจทย์ 4.19

### 3.21 โจทย์ 4.21

พิจารณาแคชระดับเดียว (Single-Level Cache) ที่มีเวลาเข้าถึง `2.5 ns` มีขนาดบรรทัด `64 bytes` และมีอัตราการฮิต `H = 0.95` หน่วยความจำหลักรองรับการถ่ายโอนแบบบล็อก โดยมีเวลาเข้าถึงคำแรกขนาด 4 ไบต์เท่ากับ `50 ns` และใช้เวลาเพิ่มอีก `5 ns` สำหรับแต่ละคำถัดไป

**จงตอบคำถามต่อไปนี้**

- เวลาเข้าถึงเมื่อเกิดแคชมิสเป็นเท่าใด หากสมมติว่าแคชจะรอจนดึงทั้งบรรทัดจากหน่วยความจำหลักได้ครบ แล้วจึงเริ่มการอ้างอิงใหม่อีกครั้งในลักษณะฮิต
- หากเพิ่มขนาดบรรทัดเป็น `128 bytes` แล้วทำให้อัตราการฮิตเพิ่มขึ้นเป็น `0.97` การเปลี่ยนแปลงนี้จะช่วยลดเวลาเข้าถึงเฉลี่ยของหน่วยความจำหรือไม่

### 3.22 โจทย์ 4.22

คอมพิวเตอร์เครื่องหนึ่งมีแคช หน่วยความจำหลัก และดิสก์ที่ใช้สำหรับหน่วยความจำเสมือน (Virtual Memory) หากคำที่อ้างอิงอยู่ในแคช จะใช้เวลาเข้าถึง `20 ns` หากคำนั้นอยู่ในหน่วยความจำหลักแต่ไม่อยู่ในแคช จะใช้เวลา `60 ns` ในการโหลดเข้าสู่แคช แล้วจึงเริ่มการอ้างอิงใหม่อีกครั้ง และหากคำนั้นไม่อยู่ในหน่วยความจำหลัก จะต้องใช้เวลา `12 ms` เพื่อนำคำดังกล่าวจากดิสก์เข้าสู่หน่วยความจำหลัก ตามด้วย `60 ns` เพื่อนำเข้าสู่แคช แล้วจึงเริ่มอ้างอิงใหม่อีกครั้ง

กำหนดให้อัตราการฮิตของแคชเท่ากับ `0.9` และอัตราการฮิตของหน่วยความจำหลักเท่ากับ `0.6`

**จงหาค่าเวลาเฉลี่ยในหน่วยนาโนวินาทีที่ระบบต้องใช้ในการเข้าถึงคำหนึ่งคำ**

### 3.23 โจทย์ 4.23

พิจารณาแคชที่มีขนาดบรรทัด `64 bytes` และสมมติว่าโดยเฉลี่ยแล้ว `30%` ของบรรทัดทั้งหมดในแคชอยู่ในสถานะสกปรก (Dirty) หนึ่งคำ (Word) มีขนาด `8 bytes`

**จงตอบคำถามต่อไปนี้**

- หากอัตราการมิสเท่ากับ `3%` หรือกล่าวอีกอย่างคือมีอัตราการฮิต `0.97` จงคำนวณปริมาณทราฟฟิกของหน่วยความจำหลักในหน่วย `bytes per instruction` สำหรับทั้งนโยบายการเขียนผ่าน (Write Through) และการเขียนกลับ (Write Back) โดยสมมติว่าเมื่ออ่านจากหน่วยความจำจะอ่านเข้ามาทีละหนึ่งบรรทัด และในกรณี Write Back สามารถเขียนคืนเพียงหนึ่งคำจากแคชสู่หน่วยความจำหลักได้
- ทำซ้ำข้อก่อนหน้าสำหรับอัตราการมิส `5%`
- ทำซ้ำข้อก่อนหน้าสำหรับอัตราการมิส `7%`
- จากผลลัพธ์ทั้งหมด สามารถสรุปเชิงวิเคราะห์ได้อย่างไร

### 3.24 โจทย์ 4.24

ในไมโครโปรเซสเซอร์ Motorola 68020 การเข้าถึงแคชใช้เวลา 2 คล็อกไซเคิล ขณะที่การเข้าถึงข้อมูลจากหน่วยความจำหลักผ่านบัสมายังโปรเซสเซอร์ใช้เวลา 3 คล็อกไซเคิลในกรณีที่ไม่มีการแทรกสถานะรอ (Wait State) โดยข้อมูลจะถูกส่งไปยังโปรเซสเซอร์พร้อมกับส่งเข้ายังแคชแบบขนาน

### 3.25 โจทย์ 4.25

สมมติว่าโปรเซสเซอร์หนึ่งมีเวลาในหนึ่งรอบหน่วยความจำ (Memory Cycle Time) เท่ากับ `300 ns` และมีอัตราการประมวลผลคำสั่ง `1 MIPS` โดยเฉลี่ยแล้วแต่ละคำสั่งต้องใช้บัสหน่วยความจำหนึ่งรอบสำหรับการดึงคำสั่ง และอีกหนึ่งรอบสำหรับตัวดำเนินการ (Operand) ที่เกี่ยวข้อง

**จงตอบคำถามต่อไปนี้**

- คำนวณอัตราการใช้บัส (Bus Utilization) ของโปรเซสเซอร์
- หากโปรเซสเซอร์นี้ถูกติดตั้งแคชคำสั่ง (Instruction Cache) และอัตราการฮิตของแคชมีค่า `0.5` จงวิเคราะห์ผลกระทบต่ออัตราการใช้บัส

### 3.26 โจทย์ 4.26

สมรรถนะของระบบแคชระดับเดียวสำหรับการอ่านข้อมูลสามารถอธิบายได้ด้วยสมการ

```text
Ta = Tc + (1 - H)Tm
```

โดยที่

- `Ta` คือเวลาเข้าถึงเฉลี่ย (Average Access Time)
- `Tc` คือเวลาเข้าถึงแคช (Cache Access Time)
- `Tm` คือเวลาเข้าถึงหน่วยความจำหลักจากหน่วยความจำสู่รีจิสเตอร์ของโปรเซสเซอร์
- `H` คืออัตราการฮิต (Hit Ratio)

เพื่อให้ง่ายต่อการวิเคราะห์ สมมติว่าคำที่ร้องขอถูกโหลดเข้าสู่แคชพร้อมกันกับการโหลดเข้าสู่รีจิสเตอร์ของโปรเซสเซอร์ ซึ่งมีรูปแบบเดียวกับสมการ (4.2)

**จงตอบคำถามต่อไปนี้**

- กำหนดให้ `Tb` คือเวลาที่ใช้ถ่ายโอนหนึ่งบรรทัดระหว่างแคชกับหน่วยความจำหลัก และ `W` คือสัดส่วนของการอ้างอิงแบบเขียน (Write References) จงปรับสมการข้างต้นให้รองรับการเขียนควบคู่กับการอ่าน ภายใต้นโยบายการเขียนผ่าน (Write Through)
- กำหนดให้ `Wb` คือความน่าจะเป็นที่บรรทัดหนึ่งในแคชถูกแก้ไขแล้ว จงเขียนสมการของ `Ta` สำหรับนโยบายการเขียนกลับ (Write Back)

### 3.27 โจทย์ 4.27

สำหรับระบบที่มีแคช 2 ระดับ กำหนดให้

- `Tc2` = เวลาเข้าถึงแคชระดับที่สอง
- `Tm` = เวลาเข้าถึงหน่วยความจำหลัก
- `H1` = อัตราการฮิตของแคชระดับที่หนึ่ง
- `H2` = อัตราการฮิตรวมของแคชระดับที่หนึ่งและระดับที่สอง

**จงเขียนสมการสำหรับเวลาเข้าถึงเฉลี่ย `Ta` ในกรณีของการอ่าน**

### 3.28 โจทย์ 4.28

สมมติว่าระบบมีคุณลักษณะด้านสมรรถนะดังนี้เมื่อเกิดแคชมิสในการอ่าน

- ใช้เวลา 1 คล็อกไซเคิลเพื่อส่งแอดเดรสไปยังหน่วยความจำหลัก
- ใช้เวลา 4 คล็อกไซเคิลเพื่อเข้าถึงคำขนาด 32 บิตจากหน่วยความจำหลักและถ่ายโอนมายังโปรเซสเซอร์กับแคช

**จงตอบคำถามต่อไปนี้**

- หากขนาดบรรทัดแคชเท่ากับ 1 คำ มิสเพนัลตี (Miss Penalty) หรือเวลาส่วนเพิ่มที่ต้องจ่ายเมื่อเกิดแคชมิสในการอ่าน มีค่าเท่าใด
- หากขนาดบรรทัดแคชเท่ากับ 4 คำ และมีการถ่ายโอนแบบหลายครั้งที่ไม่ใช่เบิร์สต์ (Multiple, Non-Burst Transfer) มิสเพนัลตีมีค่าเท่าใด
- หากขนาดบรรทัดแคชเท่ากับ 4 คำ และใช้การถ่ายโอนที่กินเวลาเพียง 1 คล็อกไซเคิลต่อหนึ่งคำ มิสเพนัลตีมีค่าเท่าใด

### 3.29 โจทย์ 4.29

สำหรับการออกแบบแคชในโจทย์ข้อก่อนหน้า สมมติว่าการเพิ่มขนาดบรรทัดจาก 1 คำเป็น 4 คำ ทำให้อัตราการมิสจากการอ่านลดลงจาก `3.2%` เหลือ `1.1%`

**ทั้งในกรณีการถ่ายโอนแบบไม่เป็นเบิร์สต์ (Nonburst Transfer) และแบบเบิร์สต์ (Burst Transfer) จงหาค่ามิสเพนัลตีเฉลี่ยเมื่อคิดเฉลี่ยตลอดการอ่านทั้งหมด สำหรับขนาดบรรทัดทั้งสองแบบ**

## 4. สรุปเชิงวิชาการของหัวข้อ 4.5

แม้ว่าหัวข้อ 4.5 จะมีลักษณะเป็นภาคสรุปท้ายบท แต่ในเชิงวิชาการแล้ว ส่วนนี้ทำหน้าที่เป็น **ดัชนีความรู้ (Knowledge Index)** ของทั้งบทเกี่ยวกับแคชได้อย่างชัดเจน กล่าวคือ คำศัพท์สำคัญช่วยกำหนดกรอบนิยาม คำถามทบทวนช่วยตรวจสอบความเข้าใจในระดับแนวคิด และโจทย์ปัญหาช่วยผลักผู้เรียนจากการ “จำ” ไปสู่การ “วิเคราะห์ ออกแบบ และคำนวณ”

**หากผู้เรียนสามารถอธิบายคำศัพท์ทั้งหมด ตอบคำถามทบทวนได้อย่างมีเหตุผล และแก้โจทย์ปัญหาเหล่านี้ได้ด้วยตนเอง ย่อมสะท้อนว่ามีความเข้าใจเรื่องแคช (Cache) ในระดับที่พร้อมต่อยอดไปสู่ประเด็นขั้นสูงของสถาปัตยกรรมคอมพิวเตอร์**
