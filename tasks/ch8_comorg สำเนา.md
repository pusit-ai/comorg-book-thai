# บทที่ 8: การสนับสนุนของระบบปฏิบัติการ (Operating System Support)

---

# 8.1 ภาพรวมของระบบปฏิบัติการ

แม้ว่าหนังสือนี้จะเน้นเรื่อง **ฮาร์ดแวร์ของคอมพิวเตอร์** แต่มีซอฟต์แวร์ประเภทหนึ่งที่จำเป็นต้องทำความเข้าใจ นั่นคือ **ระบบปฏิบัติการ (Operating System – OS)**

ระบบปฏิบัติการคือโปรแกรมที่ทำหน้าที่

* จัดการทรัพยากรของคอมพิวเตอร์
* ให้บริการแก่โปรแกรมเมอร์
* ควบคุมการทำงานของโปรแกรมอื่น ๆ

การเข้าใจระบบปฏิบัติการช่วยให้เราเข้าใจกลไกที่ **CPU ใช้ควบคุมระบบคอมพิวเตอร์** ได้ดีขึ้น โดยเฉพาะเรื่อง

* การจัดการ interrupt
* การจัดการลำดับชั้นของหน่วยความจำ

บทนี้จะกล่าวถึงสองหน้าที่สำคัญของ OS ได้แก่

* **Scheduling**
* **Memory Management**

---

# วัตถุประสงค์และหน้าที่ของระบบปฏิบัติการ

ระบบปฏิบัติการทำหน้าที่เป็น **ตัวกลางระหว่างโปรแกรมประยุกต์ (Application) กับฮาร์ดแวร์คอมพิวเตอร์**

ระบบปฏิบัติการมีวัตถุประสงค์หลัก 2 ข้อ

### 1. ความสะดวกในการใช้งาน (Convenience)

ทำให้คอมพิวเตอร์ใช้งานได้ง่ายขึ้นสำหรับผู้ใช้

### 2. ประสิทธิภาพ (Efficiency)

ช่วยให้ทรัพยากรของระบบคอมพิวเตอร์ถูกใช้อย่างมีประสิทธิภาพ

---

# ระบบปฏิบัติการในฐานะตัวกลางระหว่างผู้ใช้กับคอมพิวเตอร์

โครงสร้างของระบบคอมพิวเตอร์สามารถมองเป็น **ชั้น (Layered Structure)** ได้ดังนี้

1. Hardware
2. Instruction Set Architecture
3. Operating System
4. Libraries / Utilities
5. Application Programs
6. End User

ผู้ใช้ส่วนใหญ่จะไม่สนใจรายละเอียดของสถาปัตยกรรมคอมพิวเตอร์ แต่จะมองระบบผ่าน **Application**

เพื่อให้การพัฒนาโปรแกรมง่ายขึ้น จึงมี **System Programs** ช่วยเหลือ เช่น

* Utilities
* File management tools
* I/O control programs

โปรแกรมที่สำคัญที่สุดในกลุ่มนี้คือ **Operating System**

OS จะ

* ซ่อนรายละเอียดฮาร์ดแวร์จากโปรแกรมเมอร์
* ให้ interface ที่ใช้งานง่าย

---

# บริการที่ระบบปฏิบัติการให้

โดยทั่วไป OS จะให้บริการดังนี้

## 1. การสร้างโปรแกรม (Program Creation)

ให้เครื่องมือ เช่น

* Editor
* Debugger

เพื่อช่วยสร้างโปรแกรม

---

## 2. การรันโปรแกรม (Program Execution)

ก่อนที่โปรแกรมจะทำงานได้ ต้องมีขั้นตอนต่าง ๆ เช่น

* โหลดคำสั่งและข้อมูลเข้าสู่หน่วยความจำ
* เตรียมอุปกรณ์ I/O
* เตรียมไฟล์ที่เกี่ยวข้อง

OS จะจัดการขั้นตอนเหล่านี้ทั้งหมด

---

## 3. การเข้าถึงอุปกรณ์ I/O

อุปกรณ์แต่ละชนิดมีคำสั่งควบคุมแตกต่างกัน

OS จะจัดการรายละเอียดเหล่านี้ ทำให้โปรแกรมเมอร์สามารถใช้คำสั่งง่าย ๆ เช่น

* read
* write

---

## 4. การเข้าถึงไฟล์

OS จะควบคุม

* รูปแบบไฟล์
* วิธีการจัดเก็บข้อมูล

รวมถึงให้ **ระบบป้องกันการเข้าถึงไฟล์**

---

## 5. การเข้าถึงระบบ

ในระบบที่มีผู้ใช้หลายคน OS จะควบคุม

* การเข้าใช้งานระบบ
* การใช้ทรัพยากร

เพื่อป้องกันการใช้งานที่ไม่ได้รับอนุญาต

---

## 6. การตรวจจับข้อผิดพลาด

ข้อผิดพลาดที่อาจเกิดขึ้น เช่น

* memory error
* device failure
* arithmetic overflow
* การเข้าถึงหน่วยความจำที่ไม่ได้รับอนุญาต

OS จะจัดการและตอบสนองต่อข้อผิดพลาดเหล่านี้

---

## 7. การเก็บสถิติการใช้งาน (Accounting)

OS จะเก็บข้อมูล เช่น

* การใช้ CPU
* response time
* การใช้ทรัพยากรต่าง ๆ

ข้อมูลนี้ช่วย

* ปรับปรุงประสิทธิภาพระบบ
* ใช้ในการคิดค่าใช้จ่ายในระบบหลายผู้ใช้

---

# อินเตอร์เฟซสำคัญในระบบคอมพิวเตอร์

## 1. Instruction Set Architecture (ISA)

กำหนดชุดคำสั่งระดับเครื่อง (Machine Instructions)

เป็นขอบเขตระหว่าง

* Hardware
* Software

---

## 2. Application Binary Interface (ABI)

กำหนดมาตรฐานสำหรับ

* System calls
* การเข้าถึงทรัพยากรระบบ

---

## 3. Application Programming Interface (API)

เป็นอินเตอร์เฟซระดับสูงที่โปรแกรมใช้ในการเข้าถึง

* hardware
* services

ผ่าน library

---

# ระบบปฏิบัติการในฐานะผู้จัดการทรัพยากร

คอมพิวเตอร์มีทรัพยากร เช่น

* CPU
* Memory
* I/O devices
* Files

OS ทำหน้าที่ **จัดการทรัพยากรเหล่านี้**

สิ่งที่น่าสนใจคือ

* OS เองก็เป็นโปรแกรม
* และต้องทำงานบน CPU เหมือนโปรแกรมทั่วไป

การทำงานของ OS จะเป็นลักษณะ

1. OS ควบคุมระบบ
2. OS ให้ CPU ไปทำงานของโปรแกรมผู้ใช้
3. OS กลับมาควบคุมอีกครั้ง

---

# ประเภทของระบบปฏิบัติการ

ระบบปฏิบัติการสามารถแบ่งตามสองลักษณะหลัก

### 1. Batch vs Interactive

**Batch System**

* ผู้ใช้ส่งงานให้ operator
* โปรแกรมหลายงานถูกจัดเป็นชุด
* ผลลัพธ์ถูกพิมพ์ออกมาภายหลัง

**Interactive System**

ผู้ใช้โต้ตอบกับคอมพิวเตอร์โดยตรง

---

### 2. Uniprogramming vs Multiprogramming

**Uniprogramming**

* ทำงานทีละโปรแกรม

**Multiprogramming**

* มีหลายโปรแกรมในหน่วยความจำ
* CPU สลับทำงานระหว่างโปรแกรม

---

# ระบบคอมพิวเตอร์ยุคแรก

คอมพิวเตอร์ยุคแรก (1940s–1950s)

* ไม่มีระบบปฏิบัติการ
* โปรแกรมเมอร์ควบคุมฮาร์ดแวร์โดยตรง

ปัญหาหลักคือ

## 1. Scheduling

ผู้ใช้ต้องจองเวลาใช้เครื่อง

ทำให้เกิด

* เวลาว่างของเครื่อง
* การใช้งานไม่เต็มประสิทธิภาพ

---

## 2. Setup Time

การรันโปรแกรมต้อง

* โหลด compiler
* compile program
* โหลด object program

ใช้เวลามาก

---

# Simple Batch Systems

เพื่อเพิ่มประสิทธิภาพ จึงเกิด **Batch OS**

ผู้ใช้ส่งงานให้ operator

OS (monitor) จะ

1. อ่านงาน
2. โหลดโปรแกรม
3. รันโปรแกรม
4. เมื่อเสร็จ โหลดงานถัดไป

---

# Multiprogramming

ปัญหาใหญ่ของ batch system คือ

**CPU เร็วกว่า I/O มาก**

CPU จึงต้องรอ I/O บ่อย

วิธีแก้คือ **Multiprogramming**

แนวคิดคือ

* โหลดหลายโปรแกรมในหน่วยความจำ
* เมื่อโปรแกรมหนึ่งรอ I/O
* CPU ไปทำงานอีกโปรแกรม

ผลคือ

* CPU utilization สูงขึ้น
* throughput เพิ่มขึ้น

---

# Time Sharing Systems

Multiprogramming นำไปสู่ **Time Sharing**

หลายผู้ใช้ใช้เครื่องเดียวกันพร้อมกัน

CPU จะแบ่งเวลาให้แต่ละผู้ใช้

เรียกว่า **time slice**

ผู้ใช้แต่ละคนจะรู้สึกเหมือนมีคอมพิวเตอร์ของตัวเอง

---

# 8.2 Scheduling

Scheduling คือกระบวนการตัดสินใจว่า

**โปรแกรมใดจะได้ใช้ CPU**

มี 4 ประเภท

| ประเภท                 | หน้าที่                       |
| ---------------------- | ----------------------------- |
| Long-term scheduling   | เลือกงานเข้าสู่ระบบ           |
| Medium-term scheduling | ควบคุมจำนวน process ใน memory |
| Short-term scheduling  | เลือก process ที่จะใช้ CPU    |
| I/O scheduling         | เลือก process สำหรับ I/O      |

---

# สถานะของ Process

Process สามารถมีสถานะได้ 5 แบบ

1. **New** – เพิ่งสร้าง
2. **Ready** – พร้อมใช้ CPU
3. **Running** – กำลังทำงาน
4. **Waiting / Blocked** – รอ I/O
5. **Exit / Halted** – สิ้นสุดการทำงาน

---

# Process Control Block (PCB)

OS เก็บข้อมูลของแต่ละ process ใน **PCB**

ข้อมูลที่เก็บ เช่น

* Process ID
* State
* Priority
* Program Counter
* Memory pointers
* CPU registers
* I/O status
* Accounting information

---

# 8.3 Memory Management

ในระบบ multiprogramming

หน่วยความจำต้องแบ่งให้หลาย process

OS ต้องจัดการ

* การจัดสรร memory
* การป้องกัน memory
* การเพิ่มประสิทธิภาพการใช้ memory

---

# เทคนิคสำคัญ

## 1. Swapping

Process สามารถ

* ย้ายออกจาก memory ไป disk
* และโหลดกลับมาเมื่อจำเป็น

---

## 2. Partitioning

หน่วยความจำถูกแบ่งเป็นส่วน

สองแบบ

### Fixed Partition

ขนาดคงที่

ข้อเสีย

* memory สูญเปล่า

---

### Dynamic Partition

ขนาดเปลี่ยนได้ตาม process

แต่เกิดปัญหา

**memory fragmentation**

---

## 3. Paging

แบ่ง

* process → pages
* memory → frames

ข้อดี

* ลด fragmentation

ต้องมี **Page Table**

สำหรับแปลง

```
Logical Address → Physical Address
```

---

# Virtual Memory

Virtual memory ทำให้

**โปรแกรมใหญ่กว่า RAM ได้**

ใช้เทคนิค

**Demand Paging**

คือ

โหลด page เฉพาะตอนที่ต้องใช้

---

ถ้า page ไม่อยู่ใน memory

จะเกิด

**Page Fault**

OS จะโหลด page จาก disk

---

# Translation Lookaside Buffer (TLB)

TLB คือ

**cache ของ page table**

ช่วยให้

* การแปลง address เร็วขึ้น

---

# Segmentation

Segmentation แบ่ง memory เป็น

**segment ตามโครงสร้างโปรแกรม**

เช่น

* code segment
* data segment
* stack segment

ข้อดี

* จัดโปรแกรมง่าย
* แชร์ข้อมูลได้
* ควบคุมสิทธิ์ได้ดี

---

# 8.4 การจัดการหน่วยความจำของ Intel x86

x86 รองรับทั้ง

* **Segmentation**
* **Paging**

สามารถใช้งานได้ 4 แบบ

1. Unsegmented + Unpaged
2. Unsegmented + Paged
3. Segmented + Unpaged
4. Segmented + Paged
