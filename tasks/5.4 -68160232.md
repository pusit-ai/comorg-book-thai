## 5.4 แฟลชเมโมรี

**แฟลชเมโมรี (Flash Memory)** เป็นรูปแบบอีกอย่างหนึ่งของหน่วยความจำเซมิคอนดักเตอร์ ใช้ทั้งสำหรับแอปพลิเคชันหน่วยความจำภายในและภายนอก แฟลชเมโมรีแนะนำครั้งแรกในช่วงกลางทศวรรษ 1980

---

### การทำงาน

รูปที่ 5.15 แสดงการทำงานพื้นฐานของแฟลชเมโมรี:

#### โครงสร้างทรานซิสเตอร์ปกติ (รูปที่ 5.15a)

ทรานซิสเตอร์ใช้ประโยชน์จากคุณสมบัติของเซมิคอนดักเตอร์ เพื่อให้แรงดันเล็กน้อยที่ส่งไปยังเกตสามารถควบคุมการไหลของกระแสขนาดใหญ่ระหว่าง source และ drain

#### เซลล์แฟลชเมโมรี

ในเซลล์แฟลชเมโมรี เกตที่สอง — เรียกว่า **floating gate** เนื่องจากถูกแยกโดยชั้นออกไซด์บางๆ — ถูกเพิ่มเข้าไปในทรานซิสเตอร์:

- **สถานะเริ่มต้น (binary 1)** (รูปที่ 5.15b): Floating gate ไม่รบกวนการทำงานของทรานซิสเตอร์ เซลล์ถูกถือว่าแทน binary 1
- **สถานะ 0 (binary 0)** (รูปที่ 5.15c): การส่งแรงดันสูงผ่านชั้นออกไซด์ทำให้อิเล็กตรอนทะลุผ่านและติดอยู่บน floating gate ซึ่งยังคงอยู่แม้ไฟจะถูกตัด เซลล์ถูกถือว่าแทน binary 0

สถานะของเซลล์สามารถอ่านได้โดยใช้วงจรภายนอกเพื่อทดสอบว่าทรานซิสเตอร์ทำงานหรือไม่ การส่งแรงดันสูงในทิศทางตรงกันข้ามจะลบอิเล็กตรอนออกจาก floating gate ทำให้กลับสู่สถานะ binary 1

#### ความคงทนของแฟลชเมโมรี

คุณสมบัติสำคัญของแฟลชเมโมรีคือมันเป็น **หน่วยความจำถาวร (persistent memory)** ซึ่งหมายความว่ามันยังคงข้อมูลเมื่อไม่มีไฟจ่าย ดังนั้นจึงมีประโยชน์สำหรับการจัดเก็บรอง (ภายนอก) และเป็นทางเลือกแทน RAM ในคอมพิวเตอร์

---

### NOR และ NAND Flash Memory

แฟลชเมโมรีมีสองประเภทที่แตกต่างกัน คือ **NOR** และ **NAND** (รูปที่ 5.16)

#### NOR Flash Memory

- หน่วยการเข้าถึงพื้นฐานคือบิต เรียกว่าเซลล์หน่วยความจำ
- เซลล์ใน NOR flash เชื่อมต่อแบบขนานกับสาย bit ดังนั้นแต่ละเซลล์สามารถอ่าน/เขียน/ลบได้ทีละเซลล์
- หาก cell หน่วยความจำใดของอุปกรณ์ถูกเปิดโดย word line ที่สอดคล้อง สาย bit จะต่ำ ซึ่งคล้ายในหน้าที่กับ NOR logic gate
- **จุดเด่น**: เข้าถึงสุ่มความเร็วสูง สามารถอ่านและเขียนข้อมูลไปยังตำแหน่งเฉพาะ และอ้างอิงและดึงข้อมูลหนึ่งไบต์ได้

#### NAND Flash Memory

- จัดในอาร์เรย์ทรานซิสเตอร์ที่มีทรานซิสเตอร์ 16 หรือ 32 ตัวต่อเนื่องกัน
- สาย bit จะต่ำเฉพาะเมื่อทรานซิสเตอร์ทั้งหมดใน word lines ที่สอดคล้องถูกเปิด ซึ่งคล้ายในหน้าที่กับ NAND logic gate
- อ่านและเขียนในบล็อกขนาดเล็ก ต้องอ่านข้อมูลแบบ blockwise (รู้จักกันว่าเป็นการเข้าถึงหน้า) โดยแต่ละบล็อกมีหลายร้อยถึงหลายพันบิต
- **จุดเด่น**: ความหนาแน่นบิตสูงกว่า NOR และความเร็วเขียนที่มากกว่า ไม่มีบัสที่อยู่ภายนอกแบบเข้าถึงสุ่ม

#### เปรียบเทียบ NOR vs NAND

| คุณสมบัติ | NOR | NAND |
|---|---|---|
| การเข้าถึง | สุ่มระดับบิต | บล็อก (page) |
| ความหนาแน่น | ต่ำกว่า | สูงกว่า |
| ความเร็วอ่าน | เร็วกว่า | ช้ากว่า |
| ความเร็วเขียน | ช้ากว่า | เร็วกว่า |
| ต้นทุนต่อบิต | สูงกว่า | ต่ำกว่า |
| การใช้งานหลัก | หน่วยความจำภายใน ระบบ embedded | หน่วยความจำภายนอก (USB, SD card, SSD) |
