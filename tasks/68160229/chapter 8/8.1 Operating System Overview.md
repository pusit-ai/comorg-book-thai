# 🖥️ Operating System Overview

## 1. วัตถุประสงค์และหน้าที่ของระบบปฏิบัติการ (Operating System Objectives and Functions)

ระบบปฏิบัติการ (Operating System: OS) คือโปรแกรมที่ควบคุมการทำงานของโปรแกรมประยุกต์ (Application Programs) และทำหน้าที่เป็นตัวกลางระหว่างแอปพลิเคชันกับฮาร์ดแวร์คอมพิวเตอร์

โดยมีวัตถุประสงค์หลัก 2 ประการ คือ:

* **ความสะดวกสบาย (Convenience)**: ช่วยให้คอมพิวเตอร์ใช้งานได้ง่ายขึ้น
* **ประสิทธิภาพ (Efficiency)**: ช่วยให้ทรัพยากรของระบบคอมพิวเตอร์ถูกใช้งานได้อย่างมีประสิทธิภาพสูงสุด

---

## 2. OS ในฐานะส่วนต่อประสานระหว่างผู้ใช้กับคอมพิวเตอร์ (User/Computer Interface)

ผู้ใช้ส่วนใหญ่จะมองเห็นระบบคอมพิวเตอร์ผ่านทางแอปพลิเคชัน โดยมีโครงสร้างการทำงานเป็นลำดับชั้น ดังนี้:

```
User → Application → OS → Hardware
```

### บริการที่ OS มอบให้แก่ผู้ใช้และนักพัฒนา:

* **การรันโปรแกรม (Program execution)**:
  จัดการการโหลดโปรแกรมและข้อมูลลงในหน่วยความจำและเตรียมทรัพยากรที่จำเป็น

* **การเข้าถึงอุปกรณ์ I/O (Access to I/O devices)**:
  ช่วยให้นักพัฒนาไม่ต้องจัดการกับรายละเอียดที่ซับซ้อนของฮาร์ดแวร์แต่ละตัว

* **การควบคุมการเข้าถึงไฟล์ (Controlled access to files)**:
  จัดการรูปแบบไฟล์และสื่อบันทึกข้อมูล รวมถึงการป้องกันการเข้าถึงที่ไม่ได้รับอนุญาต

* **การเข้าถึงระบบ (System access)**:
  ควบคุมการเข้าถึงระบบส่วนรวมและทรัพยากรเฉพาะ เพื่อป้องกันข้อมูลและแก้ไขข้อขัดแย้งในการใช้ทรัพยากร

* **การตรวจจับและการตอบสนองต่อข้อผิดพลาด (Error detection and response)**:
  จัดการเมื่อเกิดข้อผิดพลาดทั้งในฮาร์ดแวร์และซอฟต์แวร์เพื่อให้กระทบกับโปรแกรมน้อยที่สุด

* **การจัดทำบัญชี (Accounting)**:
  เก็บสถิติการใช้งานทรัพยากรเพื่อใช้ในการปรับปรุงประสิทธิภาพหรือคิดค่าบริการ

---

## 3. อินเทอร์เฟซหลัก (Key Interfaces)

ในการทำงานร่วมกันระหว่างฮาร์ดแวร์และซอฟต์แวร์ มีจุดเชื่อมต่อที่สำคัญ 3 อย่าง:

* **ISA (Instruction Set Architecture)**:
  กำหนดชุดคำสั่งเครื่องที่คอมพิวเตอร์เข้าใจ ถือเป็นรอยต่อระหว่างฮาร์ดแวร์และซอฟต์แวร์

* **ABI (Application Binary Interface)**:
  กำหนดมาตรฐานเพื่อให้โปรแกรมสามารถทำงานร่วมกันได้ (Portability) และกำหนดวิธีที่โปรแกรมจะเรียกใช้บริการของ OS (System Call)

* **API (Application Programming Interface)**:
  ช่วยให้โปรแกรมสามารถเข้าถึงทรัพยากรฮาร์ดแวร์ผ่านระบบบริการของ OS

---

## 4. OS ในฐานะผู้จัดการทรัพยากร (Resource Manager)

ระบบปฏิบัติการมีหน้าที่ควบคุมการเคลื่อนย้าย การจัดเก็บ และการประมวลผลข้อมูล:

* **การจัดการโปรเซสเซอร์ (Processor management)**:
  กำหนดเวลาที่ CPU จะใช้ในการรันโปรแกรมแต่ละตัว

* **การจัดการหน่วยความจำ (Memory management)**:
  จัดสรรพื้นที่ในหน่วยความจำหลักให้แก่โปรแกรมต่าง ๆ

* **การจัดการอุปกรณ์ I/O และไฟล์**:
  ควบคุมการใช้งานอุปกรณ์รับเข้า/ส่งออก และจัดการพื้นที่จัดเก็บข้อมูลในระยะยาว

---

## 🔚 สรุป

ระบบปฏิบัติการคือซอฟต์แวร์พื้นฐานที่สำคัญที่สุด ซึ่งคอยบริหารจัดการทรัพยากรฮาร์ดแวร์ที่มีอยู่อย่างจำกัดให้เกิดประโยชน์สูงสุด และอำนวยความสะดวกให้ผู้ใช้งานและนักพัฒนาสามารถรันโปรแกรมต่าง ๆ ได้โดยไม่ต้องกังวลถึงความซับซ้อนของเครื่องจักร
