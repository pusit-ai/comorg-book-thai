## 7.1 อุปกรณ์ภายนอก

นอกจากโปรเซสเซอร์และชุดโมดูลหน่วยความจำแล้ว องค์ประกอบหลักที่สามของระบบคอมพิวเตอร์คือชุดโมดูล I/O โดยแต่ละโมดูลเชื่อมต่อกับบัสระบบหรือสวิตช์กลาง และควบคุมอุปกรณ์ต่อพ่วงหนึ่งชิ้นหรือมากกว่า

### เหตุผลที่ต้องใช้โมดูล I/O

ไม่สามารถเชื่อมต่ออุปกรณ์ต่อพ่วงเข้ากับบัสระบบโดยตรงได้ เนื่องจากสาเหตุดังต่อไปนี้

- อุปกรณ์ต่อพ่วงมีหลากหลายประเภทและวิธีการทำงาน การรวมตรรกะที่จำเป็นไว้ในโปรเซสเซอร์เพื่อควบคุมอุปกรณ์ทุกชนิดนั้นเป็นไปได้ยาก
- อัตราการถ่ายโอนข้อมูลของอุปกรณ์ต่อพ่วงส่วนใหญ่ช้ากว่าหน่วยความจำหรือโปรเซสเซอร์มาก การใช้บัสระบบความเร็วสูงเพื่อสื่อสารโดยตรงกับอุปกรณ์จึงไม่คุ้มค่า
- ในทางกลับกัน อุปกรณ์บางชนิดมีอัตราการถ่ายโอนเร็วกว่าหน่วยความจำหรือโปรเซสเซอร์ ซึ่งหากไม่จัดการอย่างเหมาะสมก็จะนำไปสู่ความไม่มีประสิทธิภาพ
- อุปกรณ์ต่อพ่วงมักใช้รูปแบบข้อมูลและความยาวคำที่แตกต่างจากคอมพิวเตอร์ที่เชื่อมต่ออยู่

### หน้าที่หลักของโมดูล I/O

โมดูล I/O มีหน้าที่หลัก 2 ประการ ได้แก่

1. เชื่อมต่อกับโปรเซสเซอร์และหน่วยความจำผ่านบัสระบบหรือสวิตช์กลาง
2. เชื่อมต่อกับอุปกรณ์ต่อพ่วงหนึ่งชิ้นหรือมากกว่าผ่านลิงก์ข้อมูลเฉพาะทาง

### ประเภทของอุปกรณ์ภายนอก

อุปกรณ์ภายนอกแบ่งออกเป็น 3 ประเภทหลัก ได้แก่

**อุปกรณ์สำหรับมนุษย์ (Human-Readable Devices):** ใช้สำหรับการสื่อสารกับผู้ใช้คอมพิวเตอร์ เช่น จอแสดงผลวิดีโอ แป้นพิมพ์ เมาส์ เครื่องพิมพ์

**อุปกรณ์สำหรับเครื่องจักร (Machine-Readable Devices):** ใช้สำหรับการสื่อสารกับอุปกรณ์อิเล็กทรอนิกส์ เช่น อุปกรณ์จัดเก็บข้อมูล เซ็นเซอร์ ตัวกระตุ้น

**อุปกรณ์สื่อสาร (Communication Devices):** ใช้สำหรับสื่อสารกับอุปกรณ์ระยะไกล

## 7.2 โมดูล I/O

โมดูล I/O เป็นองค์ประกอบที่ทำหน้าที่เป็นสะพานเชื่อมระหว่างบัสระบบและอุปกรณ์ต่อพ่วง

### หน้าที่ของโมดูล I/O

โมดูล I/O ทำหน้าที่ดังต่อไปนี้

**การควบคุมและการกำหนดเวลา (Control and Timing):** ประสานงานการไหลของข้อมูลระหว่างทรัพยากรภายในและอุปกรณ์ภายนอก โดยทรัพยากรภายใน เช่น หน่วยความจำหลักและบัสของระบบ จะต้องใช้ร่วมกันระหว่างอุปกรณ์หลายชิ้น

**การสื่อสารกับโปรเซสเซอร์ (Processor Communication):** รับคำสั่ง สถานะ ข้อมูล และที่อยู่จากโปรเซสเซอร์

**การสื่อสารกับอุปกรณ์ (Device Communication):** รับและส่งคำสั่ง สถานะ และข้อมูลไปยังอุปกรณ์ภายนอก

**การบัฟเฟอร์ข้อมูล (Data Buffering):** เนื่องจากอัตราการถ่ายโอนข้อมูลระหว่างหน่วยความจำหลักและบัสแตกต่างจากอุปกรณ์ต่อพ่วง โมดูล I/O จึงต้องรับข้อมูลจากอุปกรณ์และบัฟเฟอร์ไว้จนกระทั่งสามารถส่งให้บัสได้

**การตรวจจับข้อผิดพลาด (Error Detection):** ตรวจจับและรายงานข้อผิดพลาดจากอุปกรณ์ต่อพ่วง

### รีจิสเตอร์ของโมดูล I/O

โมดูล I/O มีรีจิสเตอร์สำคัญ ดังนี้

- **รีจิสเตอร์ข้อมูล (Data Register):** เก็บข้อมูลที่รับ/ส่งระหว่างโปรเซสเซอร์และอุปกรณ์
- **รีจิสเตอร์สถานะ (Status Register):** แสดงสถานะปัจจุบันของโมดูลและอุปกรณ์
- **รีจิสเตอร์ควบคุม (Control Register):** รับคำสั่งจากโปรเซสเซอร์

## 7.3 I/O แบบโปรแกรม

I/O แบบโปรแกรม (Programmed I/O) คือวิธีการที่โปรเซสเซอร์ออกคำสั่ง I/O แล้วรอจนกว่าการดำเนินการ I/O จะเสร็จสิ้น

### ประเภทของคำสั่ง I/O

โมดูล I/O สามารถรับคำสั่งจากโปรเซสเซอร์ได้ 4 ประเภท ได้แก่

| ประเภทคำสั่ง | คำอธิบาย |
|--------------|-----------|
| **Control (ควบคุม)** | ใช้เปิดใช้งานอุปกรณ์และบอกอุปกรณ์ว่าต้องทำอะไร |
| **Test (ทดสอบ)** | ใช้ตรวจสอบเงื่อนไขสถานะต่างๆ ที่เกี่ยวข้องกับโมดูล I/O และอุปกรณ์ |
| **Read (อ่าน)** | สั่งให้โมดูล I/O รับข้อมูลจากอุปกรณ์และเก็บในบัฟเฟอร์ข้อมูลภายใน |
| **Write (เขียน)** | สั่งให้โมดูล I/O นำข้อมูลจากบัสข้อมูลและส่งไปยังอุปกรณ์ |

### วิธีการทำงานของ I/O แบบโปรแกรม

ขั้นตอนการทำงานมีดังนี้

1. โปรเซสเซอร์ออกคำสั่ง I/O ไปยังโมดูล I/O
2. โมดูล I/O ดำเนินการตามที่ร้องขอ แล้วตั้งค่าบิตที่เหมาะสมในรีจิสเตอร์สถานะ
3. **โปรเซสเซอร์ต้องตรวจสอบสถานะของโมดูล I/O เป็นระยะๆ** จนกว่าจะพบว่าการดำเนินการเสร็จสมบูรณ์ (การวนรอบรอ หรือ **Polling/Busy-Wait**)

**ข้อเสียสำคัญ:** โปรเซสเซอร์เสียเวลาส่วนใหญ่ไปกับการรอและตรวจสอบสถานะโดยไม่จำเป็น ทำให้ประสิทธิภาพโดยรวมของระบบลดลงอย่างมาก

## 7.4 I/O แบบขับเคลื่อนด้วยอินเทอร์รัปต์

I/O แบบขับเคลื่อนด้วยอินเทอร์รัปต์ (Interrupt-Driven I/O) เป็นวิธีที่มีประสิทธิภาพมากกว่า I/O แบบโปรแกรม เนื่องจากโปรเซสเซอร์ไม่ต้องเสียเวลารอ

### ลำดับการทำงาน

**จากมุมมองของโปรเซสเซอร์:**

1. โปรเซสเซอร์ออกคำสั่ง I/O ไปยังโมดูล I/O
2. โปรเซสเซอร์ดำเนินการงานอื่นต่อไปในขณะเดียวกัน
3. เมื่อสิ้นสุดทุกรอบคำสั่ง โปรเซสเซอร์จะตรวจสอบการเกิดอินเทอร์รัปต์
4. เมื่อโมดูล I/O พร้อมรับ/ส่งข้อมูล จะส่งสัญญาณอินเทอร์รัปต์ไปยังโปรเซสเซอร์
5. โปรเซสเซอร์บันทึกบริบทของโปรแกรมปัจจุบัน แล้วประมวลผลอินเทอร์รัปต์
6. โปรเซสเซอร์อ่านข้อมูลจากโมดูล I/O และเก็บในหน่วยความจำ
7. โปรเซสเซอร์คืนค่าบริบทของโปรแกรมและดำเนินการต่อ

**จากมุมมองของโมดูล I/O:**

1. โมดูล I/O รับคำสั่ง READ จากโปรเซสเซอร์
2. โมดูล I/O อ่านข้อมูลจากอุปกรณ์ต่อพ่วง
3. เมื่อข้อมูลอยู่ในรีจิสเตอร์ข้อมูล โมดูลส่งสัญญาณอินเทอร์รัปต์ผ่านสายควบคุม
4. โมดูลรอจนกว่าโปรเซสเซอร์จะร้องขอข้อมูล
5. เมื่อได้รับคำขอ โมดูลวางข้อมูลบนบัสข้อมูล แล้วพร้อมสำหรับการดำเนินการ I/O ครั้งต่อไป

### ข้อดีและข้อเสีย

**ข้อดี:** โปรเซสเซอร์สามารถดำเนินงานอื่นได้ในระหว่างรอ ทำให้ประสิทธิภาพโดยรวมสูงขึ้น

**ข้อเสีย:** ยังคงต้องใช้เวลาของโปรเซสเซอร์มาก เพราะทุกคำข้อมูลที่ส่งระหว่างหน่วยความจำกับโมดูล I/O หรือจากโมดูล I/O ไปยังหน่วยความจำต้องผ่านโปรเซสเซอร์

### การจัดการอินเทอร์รัปต์หลายตัว

ในระบบที่มีอุปกรณ์หลายชิ้น สามารถจัดการอินเทอร์รัปต์หลายตัวได้ 2 วิธี ได้แก่

**การปิดใช้งานอินเทอร์รัปต์ (Disabled Interrupt):** โปรเซสเซอร์จะไม่รับอินเทอร์รัปต์ใหม่ขณะกำลังประมวลผลอินเทอร์รัปต์อยู่ อินเทอร์รัปต์ที่เกิดขึ้นจะถูกค้างรอไว้และตรวจสอบหลังจากอินเทอร์รัปต์ปัจจุบันเสร็จสิ้น

**ระดับความสำคัญของอินเทอร์รัปต์ (Interrupt Priority):** กำหนดระดับความสำคัญให้กับอินเทอร์รัปต์ต่างๆ เพื่อให้อินเทอร์รัปต์ที่สำคัญกว่าสามารถขัดจังหวะอินเทอร์รัปต์ที่สำคัญน้อยกว่าได้

## 7.5 การเข้าถึงหน่วยความจำโดยตรง (DMA)

การเข้าถึงหน่วยความจำโดยตรง (Direct Memory Access: DMA) เป็นวิธีที่มีประสิทธิภาพสูงสุดในการถ่ายโอนข้อมูลจำนวนมากระหว่างหน่วยความจำและอุปกรณ์ I/O

### หลักการทำงานของ DMA

ในการทำงานแบบ DMA มีการเพิ่มโมดูลพิเศษที่เรียกว่า **ตัวควบคุม DMA (DMA Controller)** เข้ามาในระบบ โดยตัวควบคุม DMA สามารถเลียนแบบโปรเซสเซอร์และควบคุมการถ่ายโอนข้อมูลระหว่างหน่วยความจำและโมดูล I/O ได้โดยตรง โดยไม่ต้องผ่านโปรเซสเซอร์

### ลำดับการทำงานของ DMA

1. โปรเซสเซอร์ส่งคำสั่งไปยังตัวควบคุม DMA โดยระบุ ที่อยู่หน่วยความจำ, อุปกรณ์ I/O, ทิศทางการถ่ายโอน (อ่านหรือเขียน) และจำนวนข้อมูลที่ต้องการถ่ายโอน
2. โปรเซสเซอร์ดำเนินงานอื่นต่อไป
3. ตัวควบคุม DMA ทำการถ่ายโอนข้อมูลโดยตรงระหว่างหน่วยความจำกับอุปกรณ์ I/O
4. เมื่อการถ่ายโอนเสร็จสิ้น ตัวควบคุม DMA ส่งสัญญาณอินเทอร์รัปต์ไปยังโปรเซสเซอร์

### ข้อดีของ DMA

- โปรเซสเซอร์ถูกขัดจังหวะเพียงครั้งเดียวต่อการถ่ายโอนข้อมูลหนึ่งบล็อก (แทนที่จะเป็นทุกคำข้อมูล)
- ประหยัดเวลาของโปรเซสเซอร์ได้มาก โดยเฉพาะเมื่อถ่ายโอนข้อมูลจำนวนมาก

### ปัญหาการแย่งชิงบัส (Cycle Stealing)

ตัวควบคุม DMA ต้องแย่งการใช้บัสระบบกับโปรเซสเซอร์ ซึ่งอาจทำให้โปรเซสเซอร์ต้องรอในบางครั้ง อย่างไรก็ตาม ยังมีประสิทธิภาพดีกว่า I/O แบบอินเทอร์รัปต์อย่างมาก เนื่องจากโปรเซสเซอร์ถูกขัดจังหวะน้อยลงมาก

### การกำหนดค่าของ DMA

DMA สามารถกำหนดค่าได้หลายรูปแบบ ได้แก่

**แบบเดี่ยว (Single Bus, Detached DMA):** ตัวควบคุม DMA และโมดูล I/O แต่ละตัวต่างเชื่อมต่อกับบัสระบบโดยตรง

**แบบรวม (Single Bus, Integrated DMA-I/O):** ตัวควบคุม DMA รวมเข้ากับโมดูล I/O ลดปริมาณการสื่อสารบนบัสระบบ

**แบบ I/O Bus:** โมดูล I/O เชื่อมต่อกันผ่าน I/O Bus และเชื่อมต่อกับตัวควบคุม DMA เพียงตัวเดียว

---

## 7.6 การเข้าถึงแคชโดยตรง (DCA)

การเข้าถึงแคชโดยตรง (Direct Cache Access: DCA) เป็นเทคนิคขั้นสูงที่ช่วยปรับปรุงประสิทธิภาพของระบบเครือข่ายความเร็วสูง

### ปัญหาของ DMA แบบเดิม

ในการดำเนินการ DMA แบบดั้งเดิม ข้อมูลจะถูกแลกเปลี่ยนระหว่างหน่วยความจำหลักกับอุปกรณ์ I/O ผ่านโครงสร้างการเชื่อมต่อของระบบ ดังนั้นข้อมูลใดๆ ที่ถ่ายโอนผ่านระบบนี้จะต้องผ่านไปมาผ่านแคช L3

ปัญหาคือ แคช L3 ไม่มีส่วนร่วมในธุรกรรมนี้ และอาจต้องการการอ่านหน่วยความจำนอกชิปจำนวนหนึ่งครั้งหรือมากกว่า ทำให้มีความล่าช้าเพิ่มขึ้น

### หลักการทำงานของ DCA

DCA ช่วยให้ข้อมูลที่เข้ามาจากอุปกรณ์เครือข่ายสามารถเขียนเข้าไปในแคชของโปรเซสเซอร์โดยตรง แทนที่จะเขียนไปยังหน่วยความจำหลัก ทำให้เมื่อแอปพลิเคชันต้องการข้อมูล ข้อมูลนั้นจะอยู่ในแคชแล้ว ลดการเข้าถึงหน่วยความจำหลักที่ช้ากว่า

## 7.7 ช่องทาง I/O และโปรเซสเซอร์

### ช่องทาง I/O (I/O Channel)

ช่องทาง I/O เป็นวิธีการขยายความสามารถของ DMA โดยช่องทาง I/O มีความสามารถในการดำเนินการคำสั่ง I/O ได้ด้วยตนเอง ซึ่งหมายความว่าโปรเซสเซอร์สามารถตั้งโปรแกรมช่องทาง I/O ให้ดำเนินการลำดับงาน I/O ที่ซับซ้อนได้

### ประเภทของช่องทาง I/O

ช่องทาง I/O มี 2 ประเภทหลัก ได้แก่

**ช่องทางเลือก (Selector Channel):** ควบคุมอุปกรณ์ความเร็วสูงหลายตัว แต่ในช่วงเวลาหนึ่งๆ จะเชื่อมต่อกับอุปกรณ์เพียงตัวเดียว เหมาะสำหรับการถ่ายโอนข้อมูลจำนวนมาก

**ช่องทางมัลติเพล็กซ์ (Multiplexor Channel):** จัดการการดำเนินการ I/O จากอุปกรณ์หลายตัวพร้อมกันโดยการสลับกัน แบ่งออกเป็น 2 ชนิด คือ

- **ช่องทางมัลติเพล็กซ์แบบไบต์ (Byte Multiplexor Channel):** สลับระหว่างอุปกรณ์หลายตัวในระดับไบต์ต่อไบต์ เหมาะสำหรับอุปกรณ์ความเร็วต่ำ
- **ช่องทางมัลติเพล็กซ์แบบบล็อก (Block Multiplexor Channel):** สลับระหว่างอุปกรณ์หลายตัวในระดับบล็อก เหมาะสำหรับอุปกรณ์ความเร็วสูงหลายตัวพร้อมกัน

### โปรเซสเซอร์ I/O (I/O Processor)

โปรเซสเซอร์ I/O เป็นวิวัฒนาการต่อจากช่องทาง I/O มีความสามารถครบถ้วนในการประมวลผลงาน I/O โดยอิสระ ช่วยแบ่งเบาภาระของโปรเซสเซอร์หลักได้อย่างมาก

ตัวอย่างที่โดดเด่น ได้แก่ **Intel 8089** ซึ่งเป็นโปรแกรมได้และออกแบบมาเป็นตัวควบคุมอุปกรณ์ I/O สำหรับไมโครโปรเซสเซอร์และระบบฝังตัว

## 7.8 มาตรฐานการเชื่อมต่อภายนอก

### USB (Universal Serial Bus)

USB เป็นอินเทอร์เฟซอนุกรมความเร็วสูงที่ได้รับความนิยมอย่างแพร่หลาย ใช้เป็นค่าเริ่มต้นสำหรับอุปกรณ์ความเร็วต่ำ เช่น แป้นพิมพ์ และอุปกรณ์ชี้ตำแหน่ง แต่ยังนิยมใช้กับอุปกรณ์ความเร็วสูง เช่น เครื่องพิมพ์ ไดรฟ์ดิสก์ และอแดปเตอร์เครือข่าย

USB ได้ผ่านการพัฒนามาหลายรุ่น ดังนี้

| เวอร์ชัน | อัตราข้อมูล |
|---------|------------|
| USB 1.0 | 1.5 Mbps (Low Speed) / 12 Mbps (Full Speed) |
| USB 2.0 | 480 Mbps (High Speed) |
| USB 3.0 | 5 Gbps (SuperSpeed) |

### FireWire (IEEE 1394)

FireWire หรือ IEEE 1394 เป็นอินเทอร์เฟซแบบอนุกรมที่รองรับข้อมูลแบบ Isochronous (ในเวลาจริง) ซึ่งเหมาะสำหรับวิดีโอและเสียง รองรับอัตราข้อมูลตั้งแต่ 100 ถึง 3,200 Mbps

### Thunderbolt

Thunderbolt เป็นอินเทอร์เฟซความเร็วสูงรุ่นใหม่ที่รวม PCI Express และ DisplayPort เข้าด้วยกัน รองรับอัตราข้อมูลสูงถึง 40 Gbps (Thunderbolt 3/4)

### InfiniBand

InfiniBand เป็นมาตรฐานการเชื่อมต่อความเร็วสูงที่ออกแบบมาสำหรับเซิร์ฟเวอร์และสภาพแวดล้อมการประมวลผลประสิทธิภาพสูง รองรับอัตราข้อมูลสูงมากและมีความหน่วงต่ำ

## 7.9 โครงสร้าง I/O ของ IBM zEnterprise EC12

IBM zEnterprise EC12 เป็นโปรเซสเซอร์สมรรถนะสูงของ Intel ที่ใช้ในเซิร์ฟเวอร์ เวิร์กสเตชันประสิทธิภาพสูง และซูเปอร์คอมพิวเตอร์ โดยสมาชิกหลายรุ่นในตระกูล Xeon ใช้ระบบการเชื่อมต่อแบบวงแหวน

### สถาปัตยกรรมวงแหวน (Ring Architecture)

ระบบ zEnterprise EC12 ใช้การเชื่อมต่อแบบวงแหวนสองทิศทาง (Bidirectional High-Speed Ring Interconnect) เชื่อมต่อระหว่างคอร์, แคช LLC, QPI, และตัวควบคุมหน่วยความจำแบบรวม (IMC)

หลักการทำงานของวงแหวน

- แต่ละองค์ประกอบที่เชื่อมต่อกับวงแหวน (kQPI, LLC, แคช L2, คอร์) ถือเป็นตัวแทนวงแหวนและใช้ตรรกะตัวแทนวงแหวน
- ตัวแทนวงแหวนทำงานร่วมกันผ่านโปรโตคอลแบบกระจาย เพื่อร้องขอและจัดสรรการเข้าถึงวงแหวนในรูปแบบของช่องเวลา
- เมื่อตัวแทนมีข้อมูลที่จะส่ง จะเลือกทิศทางวงแหวนที่ทำให้เส้นทางสั้นที่สุดไปยังปลายทาง และส่งเมื่อมีช่องเวลาว่าง

### การทำงานของ DMA ใน Intel Xeon

ในการดำเนินการ DMA แบบดั้งเดิมใน zEnterprise EC12

- ไดรเวอร์ I/O ที่ทำงานบนคอร์จะส่งคำสั่ง DMA ไปยังตัวควบคุม DMA (ซึ่งระบุเป็น kQPI) พร้อมตำแหน่งและขนาดของบัฟเฟอร์ในหน่วยความจำหลัก
- ตัวควบคุม DMA ออกคำขออ่านที่ส่งไปยังตัวควบคุมหน่วยความจำ (MRH) ซึ่งเข้าถึงข้อมูลใน DDR memory แล้ววางลงบน System Ring เพื่อส่งให้ตัวควบคุม DMA
- แคช L3 ไม่เกี่ยวข้องในธุรกรรมนี้

### โครงสร้างกายภาพของ zEnterprise EC12

ระบบประกอบด้วยกรอบขนาดใหญ่ 2 ตัว เรียกว่าเฟรม ซึ่งรองรับส่วนประกอบต่างๆ ของ zEnterprise EC12

**เฟรมขวา** ประกอบด้วย
- กรงโปรเซสเซอร์พร้อมช่องสล็อตสำหรับบุ๊กโปรเซสเซอร์สูงสุด 4 บุ๊ก
- แต่ละบุ๊กมีโมดูลมัลติชิป (MCM), การ์ดหน่วยความจำ และตัวเชื่อมต่อกรง I/O

**เฟรมซ้าย** ประกอบด้วย
- แบตเตอรีภายใน, แหล่งจ่ายไฟ และองค์ประกอบสนับสนุน
- ช่องสล็อตสำหรับตัวดึง I/O สองตัวหรือมากกว่า

### การเชื่อมต่อ InfiniBand และ PCIe

การเชื่อมต่อ InfiniBand จากบุ๊กโปรเซสเซอร์ไปยังกรง I/O และตู้ I/O ผ่าน Host Channel Adapter (HCA) fanout ซึ่งมีลิงก์ InfiniBand ไปยัง InfiniBand multiplexors ในกรง I/O หรือตู้ I/O

InfiniBand multiplexors ใช้ในการเชื่อมต่อเซิร์ฟเวอร์, อุปกรณ์โครงสร้างพื้นฐานการสื่อสาร, อุปกรณ์จัดเก็บข้อมูล และระบบฝังตัว โดยสนับสนุนเทคโนโลยี I/O หลายประเภท เช่น

- **ESCON (Enterprise Systems Connection):** รองรับการเชื่อมต่อกับดิสก์, เทป และอุปกรณ์เครื่องพิมพ์ผ่านเทคโนโลยีไฟเบอร์
- **Ethernet:** รองรับการเชื่อมต่อความเร็ว 1 Gbps และ 10 Gbps กับอุปกรณ์หลากหลาย
