## 6.1 ดิสก์แม่เหล็ก (Magnetic Disk)

ดิสก์แม่เหล็กเป็นอุปกรณ์จัดเก็บข้อมูลที่สำคัญที่สุดสำหรับคอมพิวเตอร์ส่วนใหญ่ และเป็นตัวเลือกหลักสำหรับการจัดเก็บข้อมูลภายนอก ดิสก์แม่เหล็กมีคุณสมบัติที่สำคัญดังนี้

- จัดเก็บข้อมูลได้ในปริมาณมากด้วยต้นทุนต่ำ
- ข้อมูลไม่สูญหายเมื่อปิดเครื่อง (Non-Volatile)
- เข้าถึงข้อมูลได้แบบสุ่ม (Random Access)
- เร็วกว่าเทปแม่เหล็ก แต่ช้ากว่าหน่วยความจำหลัก

---

### กลไกการอ่านและเขียนด้วยแม่เหล็ก

ข้อมูลถูกบันทึกบนดิสก์โดยอาศัยการเหนี่ยวนำแม่เหล็ก หัวอ่านเขียน (Read/Write Head) ลอยอยู่เหนือพื้นผิวดิสก์ระยะเพียงเล็กน้อย

**กลไกการเขียน:** กระแสไฟฟ้าไหลผ่านขดลวดในหัวเขียน ทำให้เกิดสนามแม่เหล็กซึ่งทำให้พื้นผิวบริเวณนั้นถูกทำให้มีขั้วแม่เหล็กตามทิศทางที่กำหนด

**กลไกการอ่าน:** มี 2 วิธี
- **วิธีดั้งเดิม:** สนามแม่เหล็กที่เคลื่อนที่ผ่านหัวอ่านจะเหนี่ยวนำให้เกิดกระแสไฟฟ้าในขดลวด
- **วิธีสมัยใหม่ (Magnetoresistive):** ใช้คุณสมบัติที่ว่าความต้านทานไฟฟ้าของวัสดุบางชนิดเปลี่ยนแปลงตามสนามแม่เหล็ก ทำให้มีความไวสูงกว่า

```
                 ┌─────────────────────────────────────┐
                 │         หัวอ่านเขียน (R/W Head)      │
                 └───────────────┬─────────────────────┘
                                 │
                                 ▼ (ลอยเหนือพื้นผิว)
    ┌──────────────────────────────────────────────────────┐
    │  N S N S N S S N N S S N    ← พื้นผิวแม่เหล็ก        │
    │  ████████████████████████    ← จานดิสก์               │
    └──────────────────────────────────────────────────────┘
         ↑         ↑
      แทร็ก     เซกเตอร์
```

---

### การจัดระเบียบข้อมูลและการฟอร์แมต

ดิสก์แม่เหล็กมีโครงสร้างทางกายภาพดังนี้

```
┌─────────────────────────────────────────────────────────────┐
│                    โครงสร้างของดิสก์                         │
│                                                             │
│                    ┌──────────┐                             │
│              ┌─────│  แทร็ก  │─────┐                        │
│         ┌───┤     │  (Track) │     ├───┐                    │
│    ┌────┤   │     └──────────┘     │   ├────┐               │
│    │    │   │                      │   │    │               │
│    │    │   │      ◉  สปินเดิล     │   │    │               │
│    │    │   │                      │   │    │               │
│    └────┘   │     ┌──────────┐     │   └────┘               │
│         └───┘     │ เซกเตอร์  │     └───┘                    │
│                   │ (Sector) │                              │
│                   └──────────┘                             │
│                                                             │
│  แต่ละแทร็กแบ่งออกเป็นเซกเตอร์ (Sectors)                    │
│  กลุ่มของแทร็กที่อยู่ตรงกันในทุกจานเรียกว่า ซิลินเดอร์     │
│  (Cylinder)                                                 │
└─────────────────────────────────────────────────────────────┘
```

**องค์ประกอบสำคัญ:**

- **แทร็ก (Track):** วงแหวนแม่เหล็กบนพื้นผิวดิสก์ แต่ละจานมีหลายร้อยถึงหลายพันแทร็ก
- **เซกเตอร์ (Sector):** ส่วนย่อยของแทร็ก โดยทั่วไปเก็บข้อมูลได้ 512 ไบต์หรือ 4 KB
- **ซิลินเดอร์ (Cylinder):** กลุ่มของแทร็กที่มีรัศมีเท่ากันในทุกจาน

---

### คุณลักษณะทางกายภาพ

| คุณลักษณะ | คำอธิบาย |
|-----------|----------|
| จำนวนหน้า | ดิสก์หนึ่งชุดมีได้ 1–5 จาน (Platter) แต่ละจานมี 2 หน้า |
| หัวอ่านเขียน | แบบหัวเคลื่อนที่ (Movable-Head) หรือแบบหัวคงที่ (Fixed-Head) |
| แบบหัวเคลื่อนที่ | หัวหนึ่งชุดต่อหน้าดิสก์ หัวเคลื่อนเข้า-ออกเพื่อเข้าถึงแทร็กต่างๆ |
| แบบหัวคงที่ | มีหัวอ่านเขียนหนึ่งหัวต่อแทร็ก เร็วกว่าแต่ราคาแพงกว่า |

---

### พารามิเตอร์ประสิทธิภาพของดิสก์

เวลาในการเข้าถึงข้อมูลบนดิสก์ประกอบด้วย 3 ส่วนหลัก

```
┌──────────────────────────────────────────────────────────────┐
│              เวลาในการเข้าถึงข้อมูลบนดิสก์                    │
│                                                              │
│  เวลาในการเข้าถึง (Access Time) =                            │
│                                                              │
│  ┌─────────────┐   ┌──────────────┐   ┌─────────────────┐   │
│  │ Seek Time   │ + │ Rotational   │ + │ Transfer Time   │   │
│  │ (เวลาซีก)   │   │ Latency      │   │ (เวลาถ่ายโอน)   │   │
│  │             │   │ (หน่วงหมุน)  │   │                 │   │
│  └─────────────┘   └──────────────┘   └─────────────────┘   │
│        │                  │                   │              │
│   เวลาที่หัวอ่าน    เวลาที่รอให้           เวลาที่ใช้ใน      │
│   เคลื่อนไปยัง      เซกเตอร์ที่ต้องการ     การถ่ายโอนข้อมูล  │
│   แทร็กที่ต้องการ   หมุนมาถึงหัวอ่าน                         │
└──────────────────────────────────────────────────────────────┘
```

**สูตรคำนวณเวลาถ่ายโอน:**

```
Transfer Time (T_t) = b / (r × N)

โดยที่:
  b = จำนวนไบต์ที่จะถ่ายโอน
  N = จำนวนไบต์ต่อแทร็ก
  r = ความเร็วรอบ (รอบ/วินาที)
```

**ตัวอย่างพารามิเตอร์ดิสก์ทั่วไป:**

| พารามิเตอร์ | Enterprise | Desktop | Laptop |
|------------|-----------|---------|--------|
| ความจุ | 1 TB | 1 TB | 500 GB |
| เวลาซีกเฉลี่ย | 3.5 ms | 9 ms | 12 ms |
| ความเร็วรอบ | 15,000 rpm | 7,200 rpm | 5,400 rpm |
| อัตราถ่ายโอน | 600 MB/s | 150 MB/s | 100 MB/s |

---

### ตัวอย่างการคำนวณเวลาถ่ายโอน

**กรณีที่ 1 — การเข้าถึงแบบลำดับ (Sequential Access):**

พิจารณาดิสก์ที่มีเวลาซีกเฉลี่ย 4 ms, ความเร็ว 15,000 rpm, เซกเตอร์ขนาด 512 ไบต์, 500 เซกเตอร์ต่อแทร็ก ต้องการอ่านไฟล์ขนาด 2,500 เซกเตอร์

```
อ่านแทร็กแรก:
  เวลาซีกเฉลี่ย          =  4 ms
  เวลาหน่วงหมุนเฉลี่ย    =  2 ms
  อ่าน 500 เซกเตอร์       =  4 ms
                            ──────
                            10 ms

แทร็กที่เหลือ (4 แทร็ก): 2 + 4 = 6 ms ต่อแทร็ก

เวลารวม = 10 + (4 × 6) = 34 ms = 0.034 วินาที
```

**กรณีที่ 2 — การเข้าถึงแบบสุ่ม (Random Access):**

```
ต่อเซกเตอร์:
  เวลาซีกเฉลี่ย          =  4.000 ms
  เวลาหน่วงหมุน           =  2.000 ms
  อ่าน 1 เซกเตอร์         =  0.008 ms
                            ─────────
                             6.008 ms

เวลารวม = 2,500 × 6.008 = 15,020 ms = 15.02 วินาที
```

ผลลัพธ์แสดงให้เห็นว่าลำดับการเข้าถึงเซกเตอร์มีผลอย่างมากต่อประสิทธิภาพ I/O

## 6.2 RAID

**RAID** ย่อมาจาก **Redundant Array of Independent Disks** (อาร์เรย์ดิสก์อิสระแบบซ้ำซ้อน) เป็นเทคโนโลยีที่ใช้ดิสก์หลายตัวทำงานร่วมกันเพื่อเพิ่มประสิทธิภาพและ/หรือความน่าเชื่อถือ

**คุณสมบัติร่วมกันของ RAID ทุกระดับ:**
1. ระบบปฏิบัติการมองเห็นเป็นดิสก์ลอจิกตัวเดียว
2. ข้อมูลกระจายอยู่บนดิสก์ฟิสิคัลตามรูปแบบที่เรียกว่า **Striping**
3. ดิสก์ซ้ำซ้อน (Redundant Disk) ใช้เก็บข้อมูลพาริตีเพื่อกู้คืนข้อมูล

---

### โครงสร้างพื้นฐานของ RAID

```
┌─────────────────────────────────────────────────────────────────┐
│              โครงสร้างพื้นฐานของ RAID                           │
│                                                                 │
│   ดิสก์ลอจิก         ซอฟต์แวร์จัดการ          ดิสก์ฟิสิคัล     │
│   (มุมมองOS)          อาร์เรย์                                   │
│                                                                 │
│  ┌──────────┐      ┌─────────────┐    ┌────┐ ┌────┐ ┌────┐     │
│  │ strip 0  │      │   Array     │    │Disk│ │Disk│ │Disk│     │
│  │ strip 1  │─────▶│ Management  │───▶│ 0  │ │ 1  │ │ 2  │     │
│  │ strip 2  │      │  Software   │    └────┘ └────┘ └────┘     │
│  │ strip 3  │      └─────────────┘                             │
│  └──────────┘                                                   │
└─────────────────────────────────────────────────────────────────┘
```

---

### RAID ระดับ 0 (Striping ไม่มีพาริตี)

RAID 0 กระจายข้อมูลไปยังดิสก์หลายตัวโดยไม่มีการซ้ำซ้อน ทำให้ได้ประสิทธิภาพสูงสุด แต่ไม่มีการป้องกันความผิดพลาด

```
ดิสก์ลอจิก       ดิสก์ฟิสิคัล 0   ดิสก์ฟิสิคัล 1   ดิสก์ฟิสิคัล 2   ดิสก์ฟิสิคัล 3
┌─────────┐       ┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│ strip 0  │──────▶│  strip 0  │    │  strip 1  │    │  strip 2  │    │  strip 3  │
│ strip 1  │       ├───────────┤    ├───────────┤    ├───────────┤    ├───────────┤
│ strip 2  │       │  strip 4  │    │  strip 5  │    │  strip 6  │    │  strip 7  │
│ strip 3  │       ├───────────┤    ├───────────┤    ├───────────┤    ├───────────┤
│ strip 4  │       │  strip 8  │    │  strip 9  │    │  strip 10 │    │  strip 11 │
│   ...    │       │  strip 12 │    │  strip 13 │    │  strip 14 │    │  strip 15 │
└─────────┘       └───────────┘    └───────────┘    └───────────┘    └───────────┘

✓ ประสิทธิภาพสูง (อ่าน/เขียนพร้อมกันหลายดิสก์)
✗ ไม่มีความทนทานต่อความผิดพลาด (ดิสก์เสียหายตัวเดียว = ข้อมูลสูญหายทั้งหมด)
```

---

### RAID ระดับ 1 (Mirroring)

RAID 1 สร้างความซ้ำซ้อนโดยการสำเนาข้อมูลทั้งหมดไปยังดิสก์คู่ขนาน

```
ดิสก์ลอจิก       ดิสก์ฟิสิคัล 0   ดิสก์ฟิสิคัล 1   ดิสก์ฟิสิคัล 2   ดิสก์ฟิสิคัล 3
                   (ข้อมูล)          (กระจก)          (ข้อมูล)          (กระจก)
┌─────────┐       ┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│ strip 0  │──────▶│  strip 0  │    │  strip 0  │    │  strip 2  │    │  strip 2  │
│ strip 1  │       ├───────────┤    ├───────────┤    ├───────────┤    ├───────────┤
│ strip 2  │       │  strip 4  │    │  strip 4  │    │  strip 6  │    │  strip 6  │
└─────────┘       └───────────┘    └───────────┘    └───────────┘    └───────────┘
                         ↑──────── สำเนา ────────↑

✓ ทนทานต่อความผิดพลาด: ดิสก์เสียได้ 1 ตัว
✓ อ่านได้จากทั้งสองดิสก์ (ประสิทธิภาพอ่านสูงขึ้น)
✓ กู้คืนง่าย: ดิสก์เสียยังเข้าถึงข้อมูลจากดิสก์คู่ได้
✗ ต้นทุนสูง (ต้องการดิสก์เป็น 2 เท่า)
✗ ไม่เพิ่มประสิทธิภาพการเขียน
```

---

### RAID ระดับ 2 (Bit-Level Striping พร้อม Hamming Code)

RAID 2 ใช้การกระจายข้อมูลระดับบิตและใช้รหัสแฮมมิงสำหรับแก้ไขข้อผิดพลาด ในปัจจุบันไม่นิยมใช้แล้วเนื่องจากดิสก์สมัยใหม่มีการตรวจสอบข้อผิดพลาดในตัว

```
ดิสก์ข้อมูล:  D0  D1  D2  D3       ดิสก์พาริตี: P1  P2  P4
               ├───┼───┼───┤              ├───┼───┼───┤
               │ b0│ b1│ b2│ b3           │ p1│ p2│ p4│  ← บิตพาริตีแฮมมิง
               └───┴───┴───┘              └───┴───┴───┘

✓ สามารถแก้ไขข้อผิดพลาด 1 บิตได้
✗ ต้องการดิสก์พาริตีจำนวนมาก
✗ ไม่นิยมในปัจจุบัน
```

---

### RAID ระดับ 3 (Byte-Level Striping พร้อมพาริตีเดียว)

RAID 3 กระจายข้อมูลระดับไบต์และใช้ดิสก์พาริตีเพียงตัวเดียว

```
ดิสก์ 0       ดิสก์ 1       ดิสก์ 2       ดิสก์ 3       ดิสก์ P (พาริตี)
┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────────┐
│  ไบต์ 0  │  │  ไบต์ 1  │  │  ไบต์ 2  │  │  ไบต์ 3  │  │  P = b0⊕b1  │
│  ไบต์ 4  │  │  ไบต์ 5  │  │  ไบต์ 6  │  │  ไบต์ 7  │  │     ⊕b2⊕b3  │
└──────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────────┘

✓ ทนทานต่อความผิดพลาดของดิสก์ 1 ตัว
✓ อัตราถ่ายโอนข้อมูลสูง
✗ ดิสก์พาริตีกลายเป็นคอขวด
✗ ประสิทธิภาพต่ำสำหรับคำร้องขนาดเล็กหลายคำร้องพร้อมกัน
```

---

### RAID ระดับ 4 (Block-Level Striping พร้อมพาริตีเดียว)

RAID 4 คล้าย RAID 3 แต่กระจายข้อมูลระดับบล็อก ทำให้สามารถอ่านบล็อกแต่ละบล็อกจากดิสก์ตัวเดียวได้

```
ดิสก์ 0       ดิสก์ 1       ดิสก์ 2       ดิสก์ 3       ดิสก์ P (พาริตี)
┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────────┐
│  บล็อก 0 │  │  บล็อก 1 │  │  บล็อก 2 │  │  บล็อก 3 │  │ P(0,1,2,3)   │
│  บล็อก 4 │  │  บล็อก 5 │  │  บล็อก 6 │  │  บล็อก 7 │  │ P(4,5,6,7)   │
└──────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────────┘

✓ รองรับคำร้องอ่านหลายรายการพร้อมกันได้
✗ ดิสก์พาริตียังคงเป็นคอขวดในการเขียน
```

---

### RAID ระดับ 5 (Block-Level Striping พร้อมพาริตีแบบกระจาย)

RAID 5 แก้ปัญหาคอขวดของ RAID 4 โดยกระจายบิตพาริตีออกไปบนดิสก์ทุกตัว

```
ดิสก์ 0       ดิสก์ 1       ดิสก์ 2       ดิสก์ 3       ดิสก์ 4

### RAID ระดับ 6 (Block-Level Striping พร้อมพาริตีคู่)

RAID 6 เพิ่มพาริตีชุดที่สองเพื่อให้ทนทานต่อดิสก์เสียได้ 2 ตัวพร้อมกัน

```
ดิสก์ 0       ดิสก์ 1       ดิสก์ 2       ดิสก์ 3       ดิสก์ 4       ดิสก์ 5
┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│  บล็อก 0 │  │  บล็อก 1 │  │  บล็อก 2 │  │  บล็อก 3 │  │  P(0-3)  │  │  Q(0-3)  │
│   ...    │  │   ...    │  │   ...    │  │   ...    │  │   ...    │  │   ...    │
└──────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────┘
                                                          ↑ พาริตี P   ↑ พาริตี Q

✓ ทนทานต่อดิสก์เสียพร้อมกันได้ 2 ตัว
✗ ต้องการดิสก์เพิ่ม 2 ตัวสำหรับพาริตี
✗ ประสิทธิภาพการเขียนต่ำกว่า RAID 5
```

### สรุปการเปรียบเทียบระดับ RAID

| ระดับ | จำนวนดิสก์ขั้นต่ำ | ดิสก์ซ้ำซ้อน | ทนทาน | ประสิทธิภาพอ่าน | ประสิทธิภาพเขียน | การใช้งาน |
|-------|-----------------|-------------|------|--------------|----------------|----------|
| 0 | 2 | ไม่มี | ไม่มี | สูงมาก | สูงมาก | ประสิทธิภาพสูง ไม่สำคัญข้อมูล |
| 1 | 2 | N | 1 ตัว | สูง | ปานกลาง | ระบบปฏิบัติการ ฐานข้อมูลขนาดเล็ก |
| 2 | 3 | ตาม Hamming | 1 ตัว | สูง | ช้า | ไม่นิยมแล้ว |
| 3 | 3 | 1 | 1 ตัว | สูง | ช้า | วิดีโอสตรีมมิ่ง |
| 4 | 3 | 1 | 1 ตัว | สูง | ช้า (คอขวด) | ไม่นิยมแล้ว |
| 5 | 3 | 1 | 1 ตัว | สูง | ปานกลาง | ระบบทั่วไป |
| 6 | 4 | 2 | 2 ตัว | สูง | ต่ำ | ระบบที่ต้องการความเชื่อถือสูง |

### การคำนวณพาริตีใน RAID 5

พาริตีคำนวณด้วยการดำเนินการ XOR ระหว่างข้อมูลในดิสก์ทุกตัว

```
สมมุติมีดิสก์ข้อมูล 4 ตัว: D0, D1, D2, D3
พาริตี P = D0 ⊕ D1 ⊕ D2 ⊕ D3

เมื่อ D1 เสียหาย:
D1_กู้คืน = P ⊕ D0 ⊕ D2 ⊕ D3
           = (D0 ⊕ D1 ⊕ D2 ⊕ D3) ⊕ D0 ⊕ D2 ⊕ D3
           = D1  ✓
```

### ประเด็นปฏิบัติของ SSD

**ปัญหาการแตกกระจาย (Fragmentation) และการลบข้อมูล:**

เนื่องจากหน่วยความจำแฟลชต้องลบทั้งบล็อกก่อนจึงจะเขียนใหม่ได้ กระบวนการเขียนหน้าแต่ละหน้าจึงเป็นดังนี้

```
ขั้นตอนที่ 1: อ่านบล็อกทั้งหมดจากแฟลชเมโมรีไปยังบัฟเฟอร์
ขั้นตอนที่ 2: ลบบล็อกทั้งหมดในแฟลชเมโมรี
              (ไม่สามารถลบเพียงหน้าเดียวได้)
ขั้นตอนที่ 3: เขียนบล็อกทั้งหมดจากบัฟเฟอร์กลับไปยังแฟลชเมโมรี
```

**การสึกหรอ (Wear Leveling):**

เซลล์แฟลชมีจำนวนครั้งการเขียนที่จำกัด (โดยทั่วไป 10,000–100,000 ครั้ง) ตัวควบคุม SSD จึงใช้อัลกอริทึมกระจายการเขียนออกไปทั่วทุกบล็อก เพื่อป้องกันไม่ให้บล็อกใดบล็อกหนึ่งถูกใช้งานมากเกินไป

**คำสั่ง TRIM:**

คำสั่ง TRIM ช่วยให้ระบบปฏิบัติการแจ้งให้ SSD ทราบว่าบล็อกใดไม่ได้ใช้งานแล้ว SSD สามารถทำการลบบล็อกเหล่านั้นในเวลาที่ระบบว่างได้ ซึ่งช่วยรักษาประสิทธิภาพการเขียนในระยะยาว

## 6.5 เทปแม่เหล็ก (Magnetic Tape)

เทปแม่เหล็กใช้เทคนิคการอ่านและบันทึกแบบเดียวกับดิสก์แม่เหล็ก แต่อยู่บนแถบโพลีเอสเตอร์ยืดหยุ่นที่เคลือบด้วยวัสดุแม่เหล็ก

### ลักษณะทางกายภาพ

- ความกว้างของเทปอยู่ระหว่าง 0.38 ซม. ถึง 1.27 ซม.
- เทปสมัยใหม่ถูกบรรจุในตลับ (Cartridge)
- ข้อมูลบันทึกเป็นแทร็กขนานตามความยาวของเทป

### โครงสร้างข้อมูลบนเทป

### วิธีการบันทึก

**การบันทึกแบบขนาน (Parallel Recording):**
บันทึกข้อมูล 1 ไบต์พร้อมกันทั้ง 9 แทร็ก (8 บิตข้อมูล + 1 บิตพาริตี) เป็นเทคนิคดั้งเดิม

**การบันทึกแบบอนุกรม (Serial Recording):**
บันทึกบิตข้อมูลเรียงต่อกันตามแทร็ก เหมือนกับการบันทึกบนดิสก์แม่เหล็ก เป็นเทคนิคที่ใช้ในระบบสมัยใหม่

**การบันทึกแบบงู (Serpentine Recording):**
เทคนิคที่นิยมในเทปสมัยใหม่ โดยบันทึกข้อมูลสลับทิศทางไปมา

```
ทิศทางการบันทึก:
แทร็ก 1:  ────────────────────────────────────────────▶
แทร็ก 2:  ◀────────────────────────────────────────────
แทร็ก 3:  ────────────────────────────────────────────▶
แทร็ก 4:  ◀────────────────────────────────────────────
                    ... สลับทิศทางต่อไปเรื่อยๆ
```

└────────────────────────┴─────────────────┴──────────────────────┘
```

เทปแม่เหล็กยังคงเป็นสมาชิกที่มีต้นทุนต่ำที่สุดและความเร็วต่ำที่สุดในลำดับชั้นหน่วยความจำ นิยมใช้สำหรับการสำรองข้อมูล (Backup) และจัดเก็บข้อมูลถาวร (Archive) ในระยะยาว

## สรุปคำศัพท์สำคัญ

| คำศัพท์ | ความหมาย |
|---------|---------|
| Seek Time | เวลาที่หัวอ่านเคลื่อนไปยังแทร็กที่ต้องการ |
| Rotational Latency | เวลาที่รอให้เซกเตอร์ที่ต้องการหมุนมาถึงหัวอ่าน |
| Transfer Time | เวลาที่ใช้ในการถ่ายโอนข้อมูลจริง |
| Track | วงแหวนแม่เหล็กบนพื้นผิวดิสก์ |
| Sector | ส่วนย่อยของแทร็ก หน่วยเล็กที่สุดในการถ่ายโอนข้อมูล |
| Cylinder | กลุ่มแทร็กที่มีรัศมีเท่ากันในทุกจาน |
| RAID | Redundant Array of Independent Disks |
| Striping | การกระจายข้อมูลออกไปยังดิสก์หลายตัว |
| Mirroring | การสำเนาข้อมูลทั้งหมดไปยังดิสก์คู่ขนาน |
| Parity | ข้อมูลซ้ำซ้อนที่ใช้กู้คืนเมื่อดิสก์เสียหาย |
| SSD | Solid State Drive อุปกรณ์จัดเก็บข้อมูลที่ใช้แฟลชเมโมรี |
| Wear Leveling | อัลกอริทึมกระจายการเขียนเพื่อยืดอายุ SSD |
| TRIM | คำสั่งที่แจ้ง SSD ให้ลบบล็อกที่ไม่ใช้แล้ว |
| CD-ROM | Compact Disc Read-Only Memory ความจุ ~682 MB |
| DVD-ROM | Digital Versatile Disc ความจุสูงสุด 17 GB |
| Blu-ray | ออปติคัลดิสก์ความละเอียดสูง ความจุ 25–100 GB |
| Serpentine Recording | การบันทึกเทปแบบสลับทิศทาง |
| Parallel Recording | การบันทึกเทปแบบขนาน (เก่า) |
| Interrecord Gap | ช่องว่างระหว่างบล็อกบนเทปแม่เหล็ก |
