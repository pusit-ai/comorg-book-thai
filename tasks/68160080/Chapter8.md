## 8.1 ภาพรวมของระบบปฏิบัติการ

### วัตถุประสงค์และหน้าที่ของระบบปฏิบัติการ

ระบบปฏิบัติการ (Operating System: OS) ทำหน้าที่เป็นตัวกลางระหว่างโปรแกรมประยุกต์และฮาร์ดแวร์คอมพิวเตอร์ โดยมีวัตถุประสงค์หลัก ดังนี้

**ความสะดวกสบาย (Convenience):** ระบบปฏิบัติการทำให้การใช้งานคอมพิวเตอร์ง่ายขึ้นสำหรับผู้ใช้

**ประสิทธิภาพ (Efficiency):** ระบบปฏิบัติการช่วยให้ทรัพยากรของระบบคอมพิวเตอร์ถูกใช้งานอย่างมีประสิทธิภาพ

**ความสามารถในการพัฒนา (Ability to Evolve):** ระบบปฏิบัติการควรสามารถสร้าง ทดสอบ และนำเสนอฟังก์ชันใหม่ๆ ของระบบได้ โดยไม่รบกวนการทำงานของโปรแกรมที่มีอยู่

### อินเทอร์เฟซหลักในระบบคอมพิวเตอร์

ระบบคอมพิวเตอร์ทั่วไปมีอินเทอร์เฟซสำคัญ 3 ประเภท ได้แก่

**ชุดคำสั่ง (Instruction Set Architecture: ISA)** กำหนดรายการคำสั่งภาษาเครื่องที่คอมพิวเตอร์สามารถปฏิบัติตามได้ อินเทอร์เฟซนี้เป็นขอบเขตระหว่างฮาร์ดแวร์และซอฟต์แวร์ โดยทั้งโปรแกรมประยุกต์และยูทิลิตีสามารถเข้าถึง ISA ได้โดยตรง ส่วน OS มีสิทธิ์เข้าถึงคำสั่งเพิ่มเติมสำหรับการจัดการทรัพยากรระบบ

**อินเทอร์เฟซไบนารีประยุกต์ (Application Binary Interface: ABI)** กำหนดมาตรฐานสำหรับความสามารถในการพกพาไบนารีข้ามโปรแกรมต่างๆ โดยกำหนดอินเทอร์เฟซการเรียกระบบไปยัง OS และฮาร์ดแวร์ผ่าน User ISA

**อินเทอร์เฟซการเขียนโปรแกรมประยุกต์ (Application Programming Interface: API)** ให้โปรแกรมเข้าถึงทรัพยากรฮาร์ดแวร์และบริการในระบบผ่าน User ISA ร่วมกับการเรียกใช้ไลบรารี ทำให้สามารถนำโปรแกรมไปใช้ในระบบอื่นได้ง่ายผ่านการคอมไพล์ใหม่

### หน้าที่หลักของระบบปฏิบัติการ

ระบบปฏิบัติการทำหน้าที่ในการ

- **การจัดการโปรแกรม:** ควบคุมการทำงานของโปรแกรม รวมถึงการสร้างและยุติการทำงานของโปรเซส
- **การจัดการ I/O:** จัดการการรับและส่งข้อมูลระหว่างอุปกรณ์ต่างๆ
- **การจัดการไฟล์:** ควบคุมการสร้าง ลบ อ่าน และเขียนไฟล์
- **การควบคุมการเข้าถึง:** ป้องกันการเข้าถึงทรัพยากรและข้อมูลโดยไม่ได้รับอนุญาต
- **การตรวจจับและตอบสนองต่อข้อผิดพลาด:** จัดการกับข้อผิดพลาดทั้งจากฮาร์ดแวร์และซอฟต์แวร์
- **การบัญชีทรัพยากร:** รวบรวมข้อมูลสถิติการใช้งานทรัพยากร

### ประเภทของระบบปฏิบัติการ

ระบบปฏิบัติการสามารถจำแนกตามลักษณะการใช้งานได้หลายประเภท ได้แก่

- **ระบบแบบกลุ่ม (Batch System):** ประมวลผลงานเป็นชุดโดยไม่มีการโต้ตอบกับผู้ใช้
- **ระบบแบบโต้ตอบ (Interactive System):** รองรับการโต้ตอบโดยตรงกับผู้ใช้
- **ระบบแบ่งเวลา (Time-Sharing System):** ให้บริการผู้ใช้หลายคนพร้อมกัน
- **ระบบเวลาจริง (Real-Time System):** ตอบสนองต่อเหตุการณ์ภายในระยะเวลาที่กำหนด

## 8.2 การจัดตารางเวลา

การจัดตารางเวลา (Scheduling) เป็นหน้าที่สำคัญของ OS ในการจัดสรรทรัพยากรโปรเซสเซอร์ให้กับงานต่างๆ โดยแบ่งออกเป็น 3 ระดับ

### การจัดตารางเวลาระยะยาว (Long-Term Scheduling)

การจัดตารางเวลาระยะยาวทำหน้าที่กำหนดว่าโปรแกรมใดจะได้รับอนุญาตให้เข้าสู่ระบบและกลายเป็นโปรเซส ในระบบแบบกลุ่ม งานที่ส่งเข้ามาจะถูกเก็บไว้ในคิว และตัวจัดตารางระยะยาวจะสร้างโปรเซสจากคิวเหล่านี้ตามความเหมาะสม

### การจัดตารางเวลาระยะกลาง (Medium-Term Scheduling)

การจัดตารางเวลาระยะกลางเป็นส่วนหนึ่งของกระบวนการสลับ (Swapping) โดยพิจารณาจากความจำเป็นในการควบคุมระดับของมัลติโปรแกรมมิง และบนระบบที่ไม่ใช้หน่วยความจำเสมือน การจัดการหน่วยความจำก็เป็นปัจจัยสำคัญด้วย

### การจัดตารางเวลาระยะสั้น (Short-Term Scheduling)

ตัวจัดตารางระยะสั้น หรือที่เรียกว่า **ตัวจัดส่ง (Dispatcher)** ทำงานบ่อยครั้งและตัดสินใจอย่างละเอียดว่างานใดควรทำงานถัดไป

### สถานะของโปรเซส (Process States)

ในช่วงอายุการทำงานของโปรเซส สถานะจะเปลี่ยนแปลงหลายครั้ง โดยมีสถานะพื้นฐาน 5 สถานะ ได้แก่

| สถานะ | คำอธิบาย |
|-------|-----------|
| **ใหม่ (New)** | โปรแกรมได้รับการรับเข้าสู่ระบบแต่ยังไม่พร้อมทำงาน |
| **พร้อม (Ready)** | โปรเซสรอใช้โปรเซสเซอร์ |
| **ทำงาน (Running)** | โปรเซสกำลังดำเนินการบนโปรเซสเซอร์ |
| **ถูกบล็อก (Blocked)** | โปรเซสรอเหตุการณ์บางอย่าง เช่น การ I/O |
| **สิ้นสุด (Exit)** | โปรเซสได้รับการปล่อยจากโปรเซสเซอร์ |

### ข้อมูลในบล็อกควบคุมโปรเซส (Process Control Block)

บล็อกควบคุมโปรเซสประกอบด้วยข้อมูลสำคัญ ได้แก่

- **ตัวระบุโปรเซส (Process Identification):** หมายเลขเฉพาะของโปรเซส
- **ข้อมูลสถานะโปรเซสเซอร์ (Processor State Information):** เนื้อหาของรีจิสเตอร์ต่างๆ
- **ข้อมูลควบคุมโปรเซส (Process Control Information):** ข้อมูลสำหรับการจัดตารางและจัดการโปรเซส

### อัลกอริทึมการจัดตารางเวลา

ตัวอย่างผลกระทบของการจัดตารางเวลาแบบต่างๆ ต่อการใช้งานทรัพยากร

| ตัวแปร | การจัดตารางเวลาแบบ Uniprocessing | การจัดตารางเวลาแบบ Multiprocessing |
|--------|----------------------------------|-------------------------------------|
| เวลาการประมวลผล (%) | 20 | 40 |
| เวลาหน่วยความจำ (%) | 33 | 67 |
| เวลา I/O (%) | 33 | 67 |
| เวลาผ่าน (%) | 33 | 67 |
| เวลาบล็อก (นาที) | 30 | 15 |
| อัตราการตอบสนอง (ต่อชั่วโมง) | 13 | 10 |

## 8.3 การจัดการหน่วยความจำ

การจัดการหน่วยความจำเป็นหน้าที่สำคัญของ OS ที่เกี่ยวข้องกับการจัดสรรหน่วยความจำให้กับโปรเซสต่างๆ

### การสลับ (Swapping)

การสลับเป็นกระบวนการย้ายโปรเซสระหว่างหน่วยความจำหลักและหน่วยจัดเก็บข้อมูลสำรอง เพื่อให้สามารถรองรับโปรเซสได้มากขึ้นกว่าที่หน่วยความจำจะรองรับได้พร้อมกัน

### การแบ่งพาร์ทิชัน (Partitioning)

**การแบ่งพาร์ทิชันแบบคงที่ (Fixed Partitioning):** หน่วยความจำถูกแบ่งออกเป็นส่วนขนาดคงที่ล่วงหน้า ข้อเสียคืออาจเกิดการสูญเสียหน่วยความจำภายใน (Internal Fragmentation)

**การแบ่งพาร์ทิชันแบบพลวัต (Dynamic Partitioning):** พาร์ทิชันถูกสร้างขึ้นตามความต้องการของแต่ละโปรเซส โดยพาร์ทิชันจะมีขนาดเท่ากับที่โปรเซสต้องการพอดี อาจเกิดการกระจัดกระจายภายนอก (External Fragmentation)

### การแบ่งหน้า (Paging)

การแบ่งหน้าช่วยแก้ปัญหาการกระจัดกระจายโดยแบ่งหน่วยความจำกายภาพออกเป็นกรอบขนาดคงที่ (Frames) และแบ่งหน่วยความจำตรรกะออกเป็นหน้า (Pages) ขนาดเท่ากัน

กระบวนการแปลงที่อยู่ในการแบ่งหน้า

1. ที่อยู่ตรรกะประกอบด้วยหมายเลขหน้าและออฟเซต
2. ตารางหน้า (Page Table) แปลงหมายเลขหน้าเป็นหมายเลขกรอบ
3. ที่อยู่กายภาพได้จากการรวมหมายเลขกรอบกับออฟเซต

### หน่วยความจำเสมือน (Virtual Memory)

**การแบ่งหน้าตามความต้องการ (Demand Paging)** เป็นพื้นฐานของหน่วยความจำเสมือน โดยหน้าของโปรเซสจะถูกโหลดเข้าหน่วยความจำเฉพาะเมื่อจำเป็นเท่านั้น (On Demand)

**หลักการสำคัญ:** ณ เวลาใดๆ จะมีเพียงหน้าบางส่วนของโปรเซสเท่านั้นที่อยู่ในหน่วยความจำ ส่งผลให้สามารถรักษาโปรเซสได้มากขึ้น และ **โปรเซสสามารถมีขนาดใหญ่กว่าหน่วยความจำหลักทั้งหมดได้**

**ข้อผิดพลาดหน้า (Page Fault):** เมื่อโปรแกรมอ้างอิงข้อมูลในหน้าที่ไม่ได้อยู่ในหน่วยความจำ จะเกิด Page Fault ซึ่ง OS จะโหลดหน้าที่ต้องการเข้ามา

**การแทนที่หน้า (Page Replacement):** เมื่อหน่วยความจำเต็ม OS ต้องนำหน้าที่ไม่จำเป็นออกไปก่อน เทคนิคที่มีประสิทธิภาพ ได้แก่ Least Recently Used (LRU)

**การดีดตัว (Thrashing):** ปัญหาที่โปรเซสใช้เวลาส่วนใหญ่ในการสลับหน้ามากกว่าการดำเนินการคำสั่ง

### บัฟเฟอร์การค้นหาแบบเร่งความเร็ว (Translation Lookaside Buffer: TLB)

TLB เป็นแคชฮาร์ดแวร์ที่จัดเก็บการแปลงที่อยู่ตารางหน้าล่าสุด เพื่อเร่งความเร็วในการเข้าถึงหน่วยความจำ

กระบวนการทำงาน

1. เมื่อมีการอ้างอิงหน่วยความจำ โปรเซสเซอร์จะค้นหาใน TLB ก่อน
2. หากพบ (TLB Hit) จะใช้หมายเลขกรอบโดยตรง
3. หากไม่พบ (TLB Miss) จะค้นหาในตารางหน้าและอัปเดต TLB

### การแบ่งส่วน (Segmentation)

การแบ่งส่วนแบ่งหน่วยความจำตรรกะออกเป็นส่วนต่างๆ ที่มีความหมาย เช่น โค้ด, ข้อมูล, สแตก

ที่อยู่ตรรกะในการแบ่งส่วนประกอบด้วยหมายเลขส่วนและออฟเซต โดยตารางส่วน (Segment Table) จะแปลงเป็นที่อยู่กายภาพ

## 8.4 การจัดการหน่วยความจำของ Intel x86

### พื้นที่ที่อยู่ (Address Spaces)

สถาปัตยกรรม x86 รองรับสองโหมดการทำงาน

**โหมดปกป้อง (Protected Mode):** รองรับการแบ่งส่วนและการแบ่งหน้า มีกลไกการปกป้องหน่วยความจำ

**โหมดจริง (Real Mode):** โหมดดั้งเดิมสำหรับความเข้ากันได้กับซอฟต์แวร์รุ่นเก่า

### การแบ่งส่วนใน x86

ระบบ x86 มีการแบ่งส่วนที่ซับซ้อน ซึ่งใช้ตารางพิเศษหลายตาราง ได้แก่

- **Global Descriptor Table (GDT):** สำหรับส่วนที่ใช้ร่วมกันระหว่างโปรเซส
- **Local Descriptor Table (LDT):** สำหรับส่วนเฉพาะของแต่ละโปรเซส
- **Interrupt Descriptor Table (IDT):** สำหรับจัดการอินเทอร์รัปต์

### รูปแบบรายการตารางส่วน

รายการในตารางส่วน (Segment Descriptor) ประกอบด้วยฟิลด์ต่างๆ ดังนี้

| ฟิลด์ | คำอธิบาย |
|-------|-----------|
| ขีดจำกัด (Limit) | ขนาดของส่วน |
| ที่อยู่ฐาน (Base Address) | ที่อยู่เริ่มต้นของส่วน |
| ประเภท (Type) | ประเภทและสิทธิ์การเข้าถึง |
| ระดับสิทธิ์ (DPL) | ระดับสิทธิ์ในการเข้าถึง |

### การแบ่งหน้าใน x86

x86 รองรับการแบ่งหน้าขนาด 4 KB โดยใช้โครงสร้างตารางสองระดับ

- **ไดเรกทอรีหน้า (Page Directory):** ระดับแรกมี 1,024 รายการ
- **ตารางหน้า (Page Table):** ระดับสองมี 1,024 รายการต่อไดเรกทอรี

## 8.5 การจัดการหน่วยความจำของ ARM

### การจัดระบบหน่วยความจำ

สถาปัตยกรรม ARM มีระบบการจัดการหน่วยความจำที่ยืดหยุ่น รองรับหน้าขนาดต่างๆ รวมถึงขนาด 4 KB, 64 KB, 1 MB และ 16 MB

### การแปลงที่อยู่หน่วยความจำเสมือน

ARM ใช้ Translation Table Base Register (TTBR) ในการชี้ไปยังตารางการแปลงที่อยู่ โดยมีขั้นตอน ดังนี้

1. โปรเซสเซอร์สร้างที่อยู่เสมือน
2. MMU ค้นหาในตารางการแปลงที่อยู่
3. ที่อยู่กายภาพถูกสร้างขึ้นพร้อมกับการตรวจสอบสิทธิ์

### รูปแบบการจัดการหน่วยความจำ

รายการตารางการแปลงที่อยู่ของ ARM มีฟิลด์สำคัญ ดังนี้

| ฟิลด์ | คำอธิบาย |
|-------|-----------|
| ที่อยู่ฐาน (Base Address) | ที่อยู่เริ่มต้นของหน้าหรือส่วน |
| AP (Access Permission) | การควบคุมการเข้าถึงสำหรับผู้ใช้และโปรแกรมควบคุม |
| C (Cacheable) | ระบุว่าพื้นที่นี้สามารถแคชได้หรือไม่ |
| B (Bufferable) | ระบุว่าพื้นที่นี้สามารถบัฟเฟอร์ได้หรือไม่ |
| Domain | โดเมนการปกป้อง |
| Type Extension | รองรับประเภทหน่วยความจำเพิ่มเติม |

### การควบคุมการเข้าถึง (Access Control)

ARM รองรับโดเมนการเข้าถึง (Access Domains) หลายโดเมน ซึ่งแต่ละโดเมนสามารถตั้งค่าสิทธิ์การเข้าถึงแยกกันได้

**ระดับสิทธิ์:** ARM แบ่งระดับสิทธิ์เป็นโหมดผู้ใช้ (User Mode) และโหมดโปรแกรมควบคุม (Privileged Mode)

**Domain Access Control Register:** รีจิสเตอร์พิเศษที่ควบคุมการเข้าถึงโดเมนทั้งหมด 16 โดเมน แต่ละโดเมนใช้บิต 2 บิตในการกำหนดสิทธิ์

## คำศัพท์สำคัญ

| คำศัพท์ภาษาอังกฤษ | คำศัพท์ภาษาไทย | คำอธิบาย |
|-------------------|----------------|-----------|
| Batch System | ระบบแบบกลุ่ม | ประมวลผลงานเป็นชุด |
| Demand Paging | การแบ่งหน้าตามความต้องการ | โหลดหน้าเฉพาะเมื่อจำเป็น |
| Interactive OS | ระบบปฏิบัติการแบบโต้ตอบ | รองรับการโต้ตอบกับผู้ใช้ |
| Interrupt | อินเทอร์รัปต์ | สัญญาณขัดจังหวะการทำงาน |
| Job Control Language (JCL) | ภาษาควบคุมงาน | ภาษาสำหรับควบคุมการทำงานของงาน |
| Kernel | เคอร์เนล | แกนกลางของระบบปฏิบัติการ |
| Logical Address | ที่อยู่ตรรกะ | ที่อยู่ที่โปรแกรมมองเห็น |
| Long-Term Scheduling | การจัดตารางเวลาระยะยาว | กำหนดว่าโปรเซสใดเข้าระบบ |
| Medium-Term Scheduling | การจัดตารางเวลาระยะกลาง | ควบคุมระดับมัลติโปรแกรมมิง |
| Memory Management | การจัดการหน่วยความจำ | จัดสรรและควบคุมหน่วยความจำ |
| Memory Protection | การปกป้องหน่วยความจำ | ป้องกันการเข้าถึงโดยไม่ได้รับอนุญาต |
| Multiprogramming | มัลติโปรแกรมมิง | รันหลายโปรแกรมพร้อมกัน |
| Multitasking | มัลติทาสกิง | รับมือกับหลายงานพร้อมกัน |
| Nucleus | นิวเคลียส | อีกชื่อหนึ่งของเคอร์เนล |
| Operating System (OS) | ระบบปฏิบัติการ | ซอฟต์แวร์ควบคุมหลักของระบบ |
| Page Table | ตารางหน้า | ตารางแปลงที่อยู่หน้าเป็นกรอบ |
| Paging | การแบ่งหน้า | แบ่งหน่วยความจำเป็นหน้าขนาดคงที่ |
| Partitioning | การแบ่งพาร์ทิชัน | แบ่งหน่วยความจำออกเป็นส่วน |
| Physical Address | ที่อยู่กายภาพ | ที่อยู่จริงในฮาร์ดแวร์ |
| Privileged Instruction | คำสั่งพิเศษ | คำสั่งที่ใช้ได้เฉพาะโหมด OS |
| Process | โปรเซส | โปรแกรมที่กำลังทำงาน |
| Process Control Block | บล็อกควบคุมโปรเซส | โครงสร้างข้อมูลของโปรเซส |
| Process State | สถานะโปรเซส | สถานะปัจจุบันของโปรเซส |
| Real Memory | หน่วยความจำจริง | หน่วยความจำกายภาพ |
| Resident Monitor | มอนิเตอร์ประจำการ | ส่วนของ OS ที่อยู่ในหน่วยความจำเสมอ |
| Segmentation | การแบ่งส่วน | แบ่งหน่วยความจำตามความหมาย |
| Short-Term Scheduling | การจัดตารางเวลาระยะสั้น | ตัดสินใจว่าโปรเซสใดทำงานต่อไป |
| Swapping | การสลับ | ย้ายโปรเซสระหว่างหน่วยความจำและดิสก์ |
| Thrashing | การดีดตัว | ใช้เวลาสลับหน้ามากกว่าทำงาน |
| Time-Sharing System | ระบบแบ่งเวลา | แบ่งเวลาโปรเซสเซอร์ให้ผู้ใช้หลายคน |
| Translation Lookaside Buffer (TLB) | บัฟเฟอร์การค้นหาแบบเร่งความเร็ว | แคชของตารางหน้า |
| Utility | ยูทิลิตี | โปรแกรมเสริมสำหรับการดูแลระบบ |
| Virtual Memory | หน่วยความจำเสมือน | หน่วยความจำที่ขยายเกินขนาดจริง |
