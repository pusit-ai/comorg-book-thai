## 4. วิธีการ Mapping (RAM ↔ Cache)

| วิธีการ Mapping | ลักษณะการทำงาน | ข้อดี | ข้อเสีย |
| :--- | :--- | :--- | :--- |
| **Direct Mapping** | Block ของ RAM ไปได้แค่ตำแหน่งเดียวที่กำหนดไว้ใน Cache | เร็ว, ฮาร์ดแวร์ไม่ซับซ้อน, ราคาถูก | เกิด Cache Thrashing (การแย่งที่กัน) ได้ง่าย |
| **Fully Associative** | Block สามารถไปวางที่ Cache line ไหนก็ได้ที่ว่างอยู่ | มีอัตราการ Hit สูงที่สุด | ฮาร์ดแวร์มีความซับซ้อนและราคาสูง |
| **Set Associative** | แบ่ง Cache เป็น Set และในแต่ละ Set จะมีหลาย Line | ลดปัญหา Thrashing และเป็นวิธีที่นิยมใช้จริง | - |
