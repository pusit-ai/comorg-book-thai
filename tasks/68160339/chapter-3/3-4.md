# 3.4 การเชื่อมต่อด้วยระบบบัส (Bus Interconnection)

----

ระบบบัสเป็นวิธีการหลักในการเชื่อมต่อส่วนประกอบคอมพิวเตอร์มาอย่างยาวนาน แม้คอมพิวเตอร์ปัจจุบันจะหันไปใช้การเชื่อมต่อแบบ จุดต่อจุด (Point-to-Point) มากขึ้น แต่ระบบบัสยังคงสำคัญมากในระบบฝังตัว (Embedded Systems)

นิยามของบัส: คือเส้นทางการสื่อสารที่เป็น ตัวกลางการส่งข้อมูลที่ใช้ร่วมกัน (Shared Transmission Medium) อุปกรณ์ที่เชื่อมต่ออยู่ทั้งหมดสามารถรับสัญญาณที่ส่งออกมาได้ แต่ในขณะหนึ่งจะมีอุปกรณ์เพียง ตัวเดียว เท่านั้นที่สามารถส่งข้อมูลได้สำเร็จ เพื่อป้องกันสัญญาณซ้อนทับกัน

## โครงสร้างของระบบบัส (System Bus)
บัสที่เชื่อมต่อส่วนประกอบหลักเข้าด้วยกันเรียกว่า ระบบบัส (System Bus) ประกอบด้วยสายสัญญาณ 3 กลุ่มหลัก
* 1. สายข้อมูล (Data Lines): หรือ บัสข้อมูล (Data Bus) ใช้เคลื่อนย้ายข้อมูล ความกว้างของบัส (จำนวนสาย) เป็นตัวกำหนดประสิทธิภาพการส่งข้อมูลในหนึ่งรอบ
* 2. สายที่อยู่ (Address Lines): หรือ บัสที่อยู่ (Address Bus) ใช้ระบุแหล่งที่มาหรือปลายทางของข้อมูล ความกว้างของบัสนี้เป็นตัวกำหนด ความจุหน่วยความจำสูงสุด ที่ระบบจะรองรับได้
* 3. สายควบคุม (Control Lines): ใช้สำหรับควบคุมการเข้าถึงและการใช้งานสายข้อมูลและสายที่อยู่

ต้องมีวิธีการในการควบคุมการใช้งานสิ่งเหล่านั้น สัญญาณควบคุม (Control signals) จะทำหน้าที่รับส่งข้อมูลทั้งในส่วนของคำสั่ง (command) และข้อมูลด้านเวลา (timing) ระหว่างโมดูลต่างๆ ของระบบ สัญญาณด้านเวลาจะบ่งบอกถึงความถูกต้องของข้อมูลและข้อมูลตำแหน่งที่อยู่ (address) ส่วนสัญญาณคำสั่งจะระบุการปฏิบัติการที่ต้องดำเนินการ สายควบคุมที่สำคัญประกอบด้วย
* Memory write: สั่งให้ข้อมูลที่อยู่บนบัสถูกเขียนลงในตำแหน่งที่อยู่ที่ระบุไว้
* Memory read: สั่งให้ข้อมูลจากตำแหน่งที่อยู่ที่ระบุถูกนำมาวางไว้บนบัส
* I/O write: สั่งให้ข้อมูลบนบัสถูกส่งออกไปยังพอร์ต I/O ที่ระบุ
* I/O read: สั่งให้ข้อมูลจากพอร์ต I/O ที่ระบุถูกนำมาวางไว้บนบัส
* Transfer ACK (Acknowledge): บ่งบอกว่าข้อมูลได้รับการตอบรับจากบัสหรือถูกวางไว้บนบัสเรียบร้อยแล้ว
* Bus request: บ่งบอกว่าโมดูลหนึ่งต้องการเข้าควบคุมการใช้งานบัส
* Bus grant: บ่งบอกว่าโมดูลที่ร้องขอได้รับการอนุญาตให้เข้าควบคุมบัสแล้ว
* Interrupt request: บ่งบอกว่ามีการขัดจังหวะที่ค้างอยู่ (pending)
* Interrupt ACK: ตอบรับว่าการขัดจังหวะที่ค้างอยู่นั้นได้รับการรับรู้แล้ว
* Clock: ใช้สำหรับการประสานจังหวะการทำงาน (synchronize) ของการปฏิบัติการต่างๆ
* Reset: กำหนดค่าเริ่มต้นให้กับโมดูลทั้งหมดในระบบ

การทำงานของบัสมีลักษณะดังนี้: หากโมดูลหนึ่งต้องการส่งข้อมูลไปยังอีกโมดูลหนึ่ง จะต้องดำเนินการสองประการ คือ (1) ได้รับสิทธิ์การใช้งานบัส และ (2) ถ่ายโอนข้อมูลผ่านทางบัส ในทางกลับกัน หากโมดูลหนึ่งต้องการร้องขอข้อมูลจากอีกโมดูลหนึ่ง จะต้อง (1) ได้รับสิทธิ์การใช้งานบัส และ (2) ส่งคำร้องขอไปยังอีกโมดูลหนึ่งผ่านทางสายควบคุมและสายระบุตำแหน่งที่เหมาะสม จากนั้นจึงต้องรอให้โมดูลที่สองส่งข้อมูลกลับมา
