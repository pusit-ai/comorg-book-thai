# สรุปบทที่ 8.5: การจัดการหน่วยความจำของ ARM (ARM Memory Management)

สถาปัตยกรรม ARM มีระบบการจัดการหน่วยความจำที่ยืดหยุ่น เพื่อรองรับตั้งแต่อุปกรณ์ฝังตัวขนาดเล็กไปจนถึงระบบปฏิบัติการที่ซับซ้อน โดยมีประเด็นสำคัญดังนี้:

## 1. ระบบจัดการหน่วยความจำ (Memory System Organization)
* **MMU (Memory Management Unit)**: ARM ใช้ MMU ในการแปลที่อยู่เสมือน (Virtual Address) ให้เป็นที่อยู่จริง (Physical Address)
* **Micro TLB**: เป็นแคชขนาดเล็กที่เก็บข้อมูลการแปลที่อยู่ล่าสุด เพื่อให้เข้าถึงข้อมูลได้รวดเร็ว (ความล่าช้าต่ำ)
* **Main TLB**: หากไม่พบข้อมูลใน Micro TLB ระบบจะมาหาที่ Main TLB ซึ่งมีขนาดใหญ่กว่าแต่ทำงานช้ากว่าเล็กน้อย



## 2. การแปลที่อยู่หน่วยความจำเสมือน (Virtual Memory Address Translation)
ARM รองรับการจัดการพื้นที่หน่วยความจำหลายรูปแบบ เพื่อให้เหมาะสมกับขนาดของโปรแกรม:
* **Sections**: หน่วยความจำขนาดใหญ่ 1 MB (เหมาะสำหรับแมปโค้ดระบบปฏิบัติการหรือบัฟเฟอร์ขนาดใหญ่)
* **Supersections**: หน่วยความจำขนาด 16 MB
* **Pages**: แบ่งย่อยออกเป็น:
    * **Large Pages**: ขนาด 64 KB
    * **Small Pages**: ขนาด 4 KB (เป็นขนาดมาตรฐานที่ใช้ในระบบปฏิบัติการทั่วไป)

## 3. รูปแบบการจัดการหน่วยความจำ (Memory-Management Formats)
การแปลที่อยู่ของ ARM มักใช้การทำงานแบบ 2 ระดับ (Two-level translation):
1. **First-level Table (L1 Descriptor)**: เก็บข้อมูลชี้ไปยังระดับที่สอง หรือชี้ไปยัง Section ขนาด 1 MB โดยตรง
2. **Second-level Table (L2 Descriptor)**: เก็บข้อมูลที่ชี้ไปยังหน้าหน่วยความจำ (Pages) ขนาดเล็ก



## 4. การควบคุมการเข้าถึง (Access Control)
ARM มีกลไกความปลอดภัยในการเข้าถึงหน่วยความจำที่เข้มงวด:
* **Domain**: แบ่งหน่วยความจำออกเป็นกลุ่ม (สูงสุด 16 โดเมน) เพื่อกำหนดสิทธิ์การเข้าถึงแบบกลุ่ม
* **Access Permissions (AP)**: กำหนดว่าส่วนของหน่วยความจำนั้นเป็น Read-only, Read/Write หรือห้ามเข้าถึง (No Access)
* **Execute Never (XN)**: เป็นบิตที่ป้องกันไม่ให้ CPU รันคำสั่งจากพื้นที่หน่วยความจำนั้น (ช่วยป้องกันการโจมตีแบบ Buffer Overflow)

## 5. การใช้ตารางหน้าหน่วยความจำ (Page Table Entry)
ในแต่ละรายการของตาราง (Entry) จะมีบิตควบคุมสถานะ เช่น:
* **C (Cacheable)**: กำหนดว่าข้อมูลนี้สามารถเก็บใน Cache ได้หรือไม่
* **B (Bufferable)**: กำหนดว่าข้อมูลนี้สามารถเขียนผ่าน Write Buffer ได้หรือไม่
* **S (Shared)**: บ่งบอกว่าหน่วยความจำนี้ใช้ร่วมกับโปรเซสเซอร์ตัวอื่นหรือไม่

---

### สรุปจุดเด่น
การจัดการหน่วยความจำของ ARM ออกแบบมาเพื่อ **"ประสิทธิภาพในการใช้พลังงาน"** และ **"ความยืดหยุ่น"** โดยการให้ระบบปฏิบัติการสามารถเลือกได้ว่าจะใช้การแบ่งหน้าขนาดเล็ก (Paging) เพื่อประหยัดพื้นที่ หรือใช้การแบ่งส่วนขนาดใหญ่ (Sections) เพื่อลดภาระการทำงานของ MMU
