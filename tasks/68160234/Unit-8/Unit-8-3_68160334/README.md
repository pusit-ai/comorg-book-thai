# สรุปเนื้อหาบทที่ 8.3: การจัดการหน่วยความจำ (Memory Management)

ในระบบที่ทำงานแบบโปรแกรมเดียว (Uniprogramming) หน่วยความจำหลักจะถูกแบ่งเป็นสองส่วน คือ ส่วนของระบบปฏิบัติการ (OS) และส่วนของโปรแกรมที่กำลังทำงาน อย่างไรก็ตาม ในระบบ **Multiprogramming** ส่วนของผู้ใช้จะถูกแบ่งย่อยออกไปอีกเพื่อรองรับหลายโปรเซสพร้อมกัน ซึ่งหน้าที่ในการแบ่งส่วนนี้เรียกว่า **การจัดการหน่วยความจำ (Memory Management)**

## 1. การสลับหน้าสมุด (Swapping)
ปัญหาหลักของระบบคือ I/O ทำงานช้ากว่าการประมวลผลมาก จนบางครั้งทุกโปรเซสในหน่วยความจำต่างก็รอ I/O ทำให้ CPU ว่างงาน
* **กลไก:** เมื่อไม่มีโปรเซสใดในหน่วยความจำพร้อมทำงาน (อยู่ในสถานะ Ready) OS จะย้ายโปรเซสที่รออยู่ (Blocked) ออกไปไว้ที่ดิสก์ (Intermediate Queue)
* **การทำงาน:** จากนั้น OS จะนำโปรเซสใหม่จากคิวระยะยาว (Long-term Queue) หรือโปรเซสที่พร้อมทำงานจากดิสก์กลับเข้ามาแทนที่ เพื่อให้ CPU ทำงานต่อไปได้
* **ประสิทธิภาพ:** แม้การสลับหน้าจะเป็นการทำงานแบบ I/O แต่เนื่องจากดิสก์มีความเร็วสูงกว่าอุปกรณ์ I/O อื่นๆ จึงช่วยเพิ่มประสิทธิภาพโดยรวมของระบบได้

## 2. การแบ่งส่วน (Partitioning)
คือวิธีการแบ่งหน่วยความจำเพื่อบรรจุโปรเซส
* **Fixed Partitioning (การแบ่งขนาดตายตัว):** แบ่งหน่วยความจำตามขนาดที่กำหนดไว้ล่วงหน้า แม้จะมีขนาดไม่เท่ากันแต่ก็ยังเกิดปัญหา "Internal Fragmentation" หรือการเสียพื้นที่เปล่าภายในช่องหากโปรเซสมีขนาดเล็กกว่าช่องนั้น
* **Variable Partitioning (การแบ่งขนาดตามจริง):** โปรเซสจะได้รับหน่วยความจำเท่าที่ต้องการพอดี แต่เมื่อมีการเข้าและออกของโปรเซสบ่อยๆ จะเกิด "รูพรุน" ขนาดเล็กกระจายอยู่ทั่วไป เรียกว่า "External Fragmentation"
* **การแก้ไข:** ใช้การ **Compaction** เพื่อเลื่อนโปรเซสในหน่วยความจำมาติดกันและรวมพื้นที่ว่างที่กระจัดกระจายให้เป็นก้อนเดียว แต่กรรมวิธีนี้กินเวลาการประมวลผลสูง

## 3. การแบ่งหน้า (Paging)
เป็นการแก้ปัญหาความไม่คุ้มค่าของการแบ่งส่วนแบบเดิม
* **Frames และ Pages:** แบ่งหน่วยความจำหลักเป็นช่องเล็กๆ ขนาดเท่ากันเรียกว่า **Frames** และแบ่งโปรเซสเป็นส่วนเล็กๆ ขนาดเดียวกันเรียกว่า **Pages**
* **Page Table:** OS จะรักษา "ตารางหน้า" สำหรับทุกโปรเซส เพื่อระบุว่า Page ใดของโปรเซสถูกเก็บไว้ที่ Frame ไหนในหน่วยความจำจริง
* **Logical vs Physical Address:** โปรแกรมจะใช้ที่อยู่เชิงตรรกะ (Logical Address) ซึ่งประกอบด้วยเลขหน้าและตำแหน่งในหน้านั้น จากนั้นฮาร์ดแวร์จะใช้ Page Table แปลงเป็นที่อยู่จริง (Physical Address)

## 4. หน่วยความจำเสมือน (Virtual Memory)
เป็นแนวคิดที่พัฒนาต่อยอดจากการแบ่งหน้า โดยใช้เทคนิค **Demand Paging**
* **Demand Paging:** จะโหลด Page ของโปรเซสเข้าสู่หน่วยความจำหลักเฉพาะเมื่อมีการเรียกใช้งานจริงเท่านั้น
* **Page Fault:** หากโปรเซสเรียกใช้ Page ที่ไม่อยู่ในหน่วยความจำหลัก จะเกิดสัญญาณขัดจังหวะเพื่อให้ OS ไปดึงหน้านั้นจากดิสก์เข้ามา
* **Real vs Virtual Memory:** หน่วยความจำจริงคือพื้นที่ในแรม (Main Memory) ส่วนหน่วยความจำเสมือนคือพื้นที่ทั้งหมดบนดิสก์ที่โปรแกรมเมอร์มองเห็น ทำให้สามารถรันโปรแกรมที่มีขนาดใหญ่กว่าแรมที่มีอยู่จริงได้
* **Thrashing:** สถานะที่ระบบเสียเวลาส่วนใหญ่ไปกับการสลับหน้า Page ไปมาจนไม่ได้ทำงานจริง

## 5. Translation Lookaside Buffer (TLB)
เนื่องจาก Page Table มักถูกเก็บไว้ในหน่วยความจำหลัก การเข้าถึงข้อมูลหนึ่งครั้งจึงต้องเข้าถึงหน่วยความจำสองครั้ง (ครั้งแรกหาเลข Frame ครั้งที่สองดึงข้อมูลจริง) ซึ่งทำให้ระบบช้าลง
* **หน้าที่:** TLB คือ Cache พิเศษที่เก็บข้อมูล Page Table ที่ถูกเรียกใช้ล่าสุด
* **ประสิทธิภาพ:** เมื่อมีการเรียกใช้ที่อยู่หน่วยความจำ ฮาร์ดแวร์จะเช็กที่ TLB ก่อน หากพบข้อมูลจะสามารถแปลที่อยู่ได้ทันทีโดยไม่ต้องไปดูที่หน่วยความจำหลัก ช่วยเพิ่มความเร็วได้อย่างมาก

## 6. การแบ่งเซกเมนต์ (Segmentation)
ต่างจากการแบ่งหน้าตรงที่การแบ่งเซกเมนต์มักจะ **มองเห็นได้โดยโปรแกรมเมอร์**
* ช่วยให้โปรแกรมเมอร์แบ่งโปรแกรมและข้อมูลออกเป็นส่วนๆ (Segments) ตามโครงสร้างตามธรรมชาติของโปรแกรม เช่น ส่วนของโค้ดคำสั่ง, ส่วนของข้อมูล หรือส่วนของ Stack
* แต่ละ Segment สามารถมีขนาดไม่เท่ากันและมีการขยายตัวได้ ซึ่งเอื้อต่อการแชร์ข้อมูลและการป้องกันความปลอดภัยของข้อมูลระหว่างโปรเซส
