# เจาะลึกการจัดการหน่วยความจำของ Intel x86 (Intel x86 Memory Management)

สถาปัตยกรรม Intel x86 มีระบบการจัดการหน่วยความจำที่ทรงพลังและยืดหยุ่นมากที่สุดระบบหนึ่ง โดยใช้การผสมผสานระหว่าง **Segmentation** (การแบ่งส่วน) และ **Paging** (การแบ่งหน้า) เพื่อรองรับความต้องการที่หลากหลายของระบบปฏิบัติการสมัยใหม่

---

## 1. รูปแบบพื้นที่ที่อยู่ (Address Spaces)
ฮาร์ดแวร์ x86 ยอมรับการกำหนดค่ารูปแบบหน่วยความจำได้ 4 ประเภทหลัก ตามความต้องการของ OS:

1.  **Unsegmented Unpaged Memory**: ที่อยู่เสมือนเท่ากับที่อยู่จริงโดยตรง เหมาะสำหรับงานฝังตัว (Embedded) ที่ต้องการความเร็วสูงสุด
2.  **Unsegmented Paged Memory**: มองเป็นพื้นที่เชิงเส้น (Linear) และใช้ Paging จัดการ นิยมใช้ใน Berkeley UNIX
3.  **Segmented Unpaged Memory**: แบ่งหน่วยความจำตามลักษณะข้อมูลทางตรรกะ ป้องกันข้อมูลได้ระดับไบต์ และให้เวลาการเข้าถึงที่คงที่
4.  **Segmented Paged Memory**: ใช้ Segmentation แบ่งขอบเขตสิทธิการเข้าถึง และใช้ Paging จัดสรรพื้นที่ภายในส่วนนั้น นิยมใช้ใน UNIX System V

---

## 2. กลไกการแบ่งส่วนหน่วยความจำ (Segmentation)
เมื่อเปิดใช้งาน การอ้างอิงที่อยู่ทางตรรกะ (Logical Address) จะประกอบด้วย **Segment Selector (16 บิต)** และ **Offset (32 บิต)**

### ระดับสิทธิการเข้าถึง (Privilege Levels)
x86 แบ่งระดับความปลอดภัยออกเป็น 4 ระดับ (Ring 0-3):
* **ระดับ 0 (Highest)**: สำหรับฟังก์ชันวิกฤตของ OS เช่น การจัดการหน่วยความจำ
* **ระดับ 1**: สำหรับบริการทั่วไปของ OS
* **ระดับ 2**: สำหรับระบบย่อยของแอปพลิเคชัน (เช่น Database)
* **ระดับ 3 (Lowest)**: สำหรับโปรแกรมประยุกต์ทั่วไป (User Applications)

### โครงสร้าง Segment Selector:
* **Index**: ระบุตำแหน่ง Entry ในตาราง Segment
* **TI (Table Indicator)**: เลือกว่าจะใช้ตาราง Global (GDT) หรือ Local (LDT)
* **RPL**: ระดับสิทธิที่ร้องขอ



---

## 3. กลไกการแบ่งหน้าหน่วยความจำ (Paging)
x86 ใช้การค้นหาตารางแบบ 2 ระดับ (Two-level Table Lookup) เพื่อแปลง Linear Address เป็น Physical Address

### การแบ่งส่วนที่อยู่เชิงเส้น (Linear Address Partitioning):
* **Directory (Bit 31-22)**: ดัชนีชี้ไปยัง Page Directory Entry
* **Table (Bit 21-12)**: ดัชนีชี้ไปยัง Page Table Entry
* **Offset (Bit 11-0)**: ตำแหน่งข้อมูลภายในหน้า (Page) ขนาด 4 KB

### ส่วนขยายและเทคโนโลยีสนับสนุน:
* **Page Directory**: ต้องอยู่ในหน่วยความจำหลักเสมอ มีขนาด 1,024 Entries
* **TLB (Translation Lookaside Buffer)**: Cache พิเศษเก็บข้อมูลการแปลงที่อยู่ล่าสุด 32 รายการ
* **PSE (Page Size Extension)**: หากเปิดใช้งาน จะสามารถใช้หน้าขนาด **4 Mbyte** ได้ ซึ่งจะตัดขั้นตอนการค้นหาจาก 2 ระดับเหลือเพียงระดับเดียว ช่วยลดภาระของระบบเมื่อมีหน่วยความจำหลักขนาดใหญ่



---

## 4. รายละเอียดพารามิเตอร์ที่สำคัญ (Table 8.5)
ข้อมูลที่จัดเก็บใน Segment Descriptor และ Page Table Entry มีดังนี้:

| ชื่อพารามิเตอร์ | หน้าที่และรายละเอียด |
| :--- | :--- |
| **Base Address** | จุดเริ่มต้นของ Segment หรือ Page ในพื้นที่ 4-Gbyte |
| **Limit** | ขนาดสูงสุดของพื้นที่นั้นๆ (หน่วยเป็น Byte หรือ 4 KB) |
| **Present (P)** | ระบุว่าหน้าหรือส่วนนั้นอยู่ในหน่วยความจำจริงหรือไม่ |
| **Accessed (A)** | ระบบจะตั้งเป็น 1 เมื่อมีการอ่านหรือเขียนข้อมูลในหน้านั้น |
| **Dirty (D)** | ระบบจะตั้งเป็น 1 เมื่อมีการ "เขียน" ข้อมูลลงในหน้า (ใช้ในการ Swap) |
| **Read/Write (RW)** | กำหนดสิทธิว่าอ่านได้อย่างเดียว หรือเขียนได้ด้วย |
| **User/Supervisor (US)** | แยกสิทธิระหว่างโปรแกรมทั่วไปและระบบปฏิบัติการ |
| **PCD (Cache Disable)** | ระบุว่าห้ามเก็บข้อมูลจากหน้านี้ลงใน Cache |

---

## 5. สรุปขั้นตอนการแปลงที่อยู่ (Address Translation Flow)
1.  **Logical Address** (Selector + Offset)
2.  ค้นหาใน **Segment Table** เพื่อหา Base Address
3.  นำ Base Address + Offset = **Linear Address**
4.  (หากเปิด Paging) นำ Linear Address ไปค้นหาใน **Page Directory** และ **Page Table**
5.  ได้ผลลัพธ์เป็น **Physical Address** เพื่อเข้าถึงข้อมูลในแรมจริง
