# สรุปเนื้อหาบทที่ 8.2: การกำหนดการ (Scheduling)

การกำหนดการ (Scheduling) คือหัวใจสำคัญของระบบปฏิบัติการที่รองรับการทำงานหลายโปรแกรมพร้อมกัน (Multiprogramming) เพื่อให้การใช้งานหน่วยประมวลผล (Processor) เกิดประโยชน์สูงสุด โดยแบ่งออกเป็น 3 ระดับหลัก ดังนี้:

## 1. การกำหนดการระยะยาว (Long-Term Scheduling)
* **หน้าที่**: ตัดสินใจว่าโปรแกรมใดจะได้เข้าสู่ระบบเพื่อเปลี่ยนสภาพเป็น "โพรเซส" (Process)
* **การทำงาน**: ควบคุมจำนวนโพรเซสที่ทำงานอยู่ในระบบ (Degree of Multiprogramming) โดยการเลือกงานจากคิวในหน่วยเก็บข้อมูลสำรองมาสร้างเป็นโพรเซสในหน่วยความจำ
* **เป้าหมาย**: พยายามรักษาสมดุลระหว่างโพรเซสที่เน้นการคำนวณ (CPU-bound) และโพรเซสที่เน้นการรับส่งข้อมูล (I/O-bound)



## 2. การกำหนดการระยะกลาง (Medium-Term Scheduling)
* **หน้าที่**: จัดการเรื่องการสลับโพรเซสเข้า-ออก (Swapping)
* **การทำงาน**: ย้ายโพรเซสบางส่วนออกจากหน่วยความจำหลักไปพักไว้ในหน่วยเก็บข้อมูลสำรอง (Disk) ชั่วคราว เมื่อหน่วยความจำเต็ม หรือโพรเซสนั้นอยู่ในสถานะรอคอย (Blocked) นานเกินไป
* **เป้าหมาย**: เพื่อเพิ่มพื้นที่ว่างในหน่วยความจำให้โพรเซสอื่นทำงานได้ และช่วยบริหารจัดการประสิทธิภาพของระบบเมื่อมีโพรเซสมากเกินไป

## 3. การกำหนดการระยะสั้น (Short-Term Scheduling)
* **หน้าที่**: ตัดสินใจว่าโพรเซสใดในสถานะ "พร้อม" (Ready) จะได้สิทธิ์เข้าใช้งาน CPU เป็นรายต่อไป
* **ความถี่**: ทำงานบ่อยมาก (ระดับมิลลิวินาที) เนื่องจากต้องตอบสนองต่อเหตุการณ์ต่างๆ เช่น:
    * การขัดจังหวะจากนาฬิกา (Clock Interrupts)
    * การขัดจังหวะจากการรับส่งข้อมูล (I/O Interrupts)
    * การเรียกใช้บริการของระบบ (System Calls)
* **เป้าหมาย**: เพิ่มความเร็วในการประมวลผล (Throughput) และลดเวลาตอบสนอง (Response Time)

## ตารางสรุปเปรียบเทียบ

| ประเภท | ชื่อเรียก | ความถี่ในการทำงาน | หน้าที่หลัก |
| :--- | :--- | :--- | :--- |
| **ระยะยาว** | Job Scheduler | น้อยที่สุด | คัดเลือกงานใหม่เข้าสู่ระบบ |
| **ระยะกลาง** | Swapper | ปานกลาง | จัดการพื้นที่หน่วยความจำ (เข้า-ออก) |
| **ระยะสั้น** | Dispatcher | บ่อยที่สุด | เลือกโพรเซสให้ CPU ประมวลผล |

## กลไกการทำงานของคิว (Queuing Diagram)
ในการกำหนดการ ระบบจะจัดการโพรเซสผ่านคิวต่างๆ ดังนี้:
1. **Long-term Queue**: งานที่รอเข้าสู่ระบบ
2. **Ready Queue**: โพรเซสที่พร้อมจะประมวลผลใน CPU
3. **I/O Queue**: โพรเซสที่รอการรับส่งข้อมูลจากอุปกรณ์ภายนอก

การจัดการคิวที่มีประสิทธิภาพจะช่วยให้ระบบทำงานได้ลื่นไหลและไม่มีทรัพยากรส่วนใดถูกปล่อยทิ้งไว้โดยเปล่าประโยชน์
