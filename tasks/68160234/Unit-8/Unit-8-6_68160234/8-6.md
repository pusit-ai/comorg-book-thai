เนื้อหาในส่วนนี้คือการรวบรวมคำศัพท์สำคัญทั้งหมดที่ปรากฏในบทที่ 8 รวมถึงคำถามเพื่อทดสอบความเข้าใจ และแนวคิดหลักของแบบฝึกหัดท้ายบทที่เกี่ยวกับการทำงานของระบบปฏิบัติการ (OS)

## 1. คำศัพท์สำคัญ (Key Terms)

กลุ่มคำศัพท์เหล่านี้เป็นคีย์เวิร์ดสำคัญในการทำความเข้าใจการจัดการทรัพยากรของระบบปฏิบัติการ:

* **Batch system (ระบบประมวลผลแบบกลุ่ม):** ระบบที่รวบรวมงาน (Jobs) เข้าด้วยกันแล้วประมวลผลตามลำดับโดยอัตโนมัติ ไม่มีการโต้ตอบกับผู้ใช้ระหว่างทำ
* **Demand paging (การเรียกหน้าหน่วยความจำเมื่อต้องการ):** เทคนิคที่ระบบจะโหลดเพจ (Page) เข้าสู่หน่วยความจำหลักก็ต่อเมื่อโปรแกรมมีการเรียกใช้งานเพจนั้นจริงๆ เท่านั้น
* **Interactive operating system (ระบบปฏิบัติการแบบโต้ตอบ):** ระบบที่อนุญาตให้ผู้ใช้สื่อสารสั่งงานคอมพิวเตอร์และได้รับผลลัพธ์ตอบกลับโดยตรง
* **Interrupt (การขัดจังหวะ):** สัญญาณฮาร์ดแวร์หรือซอฟต์แวร์ที่สั่งให้ CPU หยุดงานปัจจุบันชั่วคราว เพื่อไปจัดการกับเหตุการณ์ที่สำคัญกว่า
* **Job control language / JCL (ภาษาควบคุมงาน):** ภาษาพิเศษที่ใช้เขียนคำสั่งระบุความต้องการของงาน เพื่อส่งให้ระบบปฏิบัติการแบบ Batch
* **Kernel / Nucleus (เคอร์เนล / นิวเคลียส):** ส่วนแกนกลางของระบบปฏิบัติการที่มีฟังก์ชันสำคัญที่สุด มักจะฝังตัวอยู่ในหน่วยความจำหลักตลอดเวลา
* **Logical address (ที่อยู่เชิงตรรกะ):** ตำแหน่งที่อยู่อ้างอิงที่สร้างขึ้นโดยโปรแกรม (ยังไม่ใช่ตำแหน่งจริงใน RAM)
* **Physical address / Real memory (ที่อยู่ทางกายภาพ / หน่วยความจำจริง):** ตำแหน่งจริงที่อยู่ในหน่วยความจำหลัก (RAM)
* **Long-term scheduling (การจัดตารางเวลาระยะยาว):** การตัดสินใจของ OS ว่าจะรับงานใหม่เข้ามาในระบบเป็นโปรเซสหรือไม่
* **Medium-term scheduling (การจัดตารางเวลาระยะกลาง):** การตัดสินใจสลับโปรเซสเข้า/ออกจากหน่วยความจำหลัก (ส่วนหนึ่งของการทำ Swapping)
* **Short-term scheduling (การจัดตารางเวลาระยะสั้น):** การตัดสินใจเลือกว่าโปรเซสใดในสถานะพร้อม (Ready) จะได้สิทธิใช้งาน CPU เป็นลำดับต่อไป
* **Memory management (การจัดการหน่วยความจำ):** กระบวนการที่ OS ใช้ในการจัดสรรและควบคุมพื้นที่หน่วยความจำสำหรับโปรเซสต่างๆ
* **Memory protection (การป้องกันหน่วยความจำ):** กลไกป้องกันไม่ให้โปรเซสหนึ่งเข้าไปอ่าน/เขียนข้อมูลในพื้นที่หน่วยความจำของโปรเซสอื่น
* **Multiprogramming / Multitasking (การทำหลายโปรแกรม / การทำหลายงาน):** การนำหลายโปรแกรมมาไว้ในหน่วยความจำและสลับการใช้งาน CPU อย่างรวดเร็วเพื่อให้ทำงานพร้อมกัน
* **Page table (ตารางหน้าหน่วยความจำ):** ตารางที่ใช้แปลง Logical Address (เลขเพจ) ให้กลายเป็น Physical Address (เลขเฟรม)
* **Paging (การแบ่งหน้าหน่วยความจำ):** เทคนิคการแบ่งหน่วยความจำทางกายภาพเป็นกรอบ (Frames) ขนาดเท่าๆ กัน และแบ่งโปรแกรมเป็นหน้า (Pages) ขนาดเท่ากัน
* **Partitioning (การแบ่งส่วน):** การแบ่งหน่วยความจำออกเป็นส่วนๆ (อาจมีขนาดคงที่หรือปรับเปลี่ยนได้) เพื่อนำโปรแกรมไปวาง
* **Privileged instruction (คำสั่งสงวนสิทธิ์):** ชุดคำสั่งเครื่องที่สามารถสั่งรันได้เฉพาะเมื่อ CPU อยู่ในโหมดของระบบปฏิบัติการเท่านั้น
* **Process (โปรเซส):** โปรแกรมที่กำลังถูกรัน หรือ "เอนทิตี" ที่กำลังถูกครอบครองโดย CPU
* **Process control block / PCB (บล็อกควบคุมโปรเซส):** โครงสร้างข้อมูลที่ OS ใช้เก็บข้อมูลสถานะและรายละเอียดของแต่ละโปรเซส
* **Process state (สถานะโปรเซส):** สภาวะปัจจุบันของโปรเซส เช่น New, Ready, Running, Blocked, หรือ Exit
* **Resident monitor (มอนิเตอร์ประจำที่):** ส่วนของโปรแกรม OS ยุคแรกที่ต้องอยู่ในหน่วยความจำหลักเสมอ เพื่อคอยจัดการดึงงานถัดไปมาทำ
* **Segmentation (การแบ่งเซกเมนต์):** การแบ่งโปรแกรมหรือข้อมูลออกเป็นส่วนๆ ตามตรรกะของผู้เขียนโปรแกรม (ขนาดแต่ละส่วนไม่จำเป็นต้องเท่ากัน)
* **Swapping (การสลับ):** การย้ายโปรเซสไปมาระหว่างหน่วยความจำหลัก (RAM) กับหน่วยความจำสำรอง (เช่น ดิสก์) เพื่อเพิ่มพื้นที่ว่าง
* **Thrashing (ภาวะแทรชชิง):** อาการที่ระบบเสียเวลาส่วนใหญ่ไปกับการสลับเพจ (Page in/out) มากกว่าการประมวลผลคำสั่งจริงๆ
* **Time-sharing system (ระบบแบ่งเวลา):** ระบบที่ให้ผู้ใช้หลายคนสามารถเข้าใช้งานคอมพิวเตอร์พร้อมกันได้ โดย OS จะสลับให้เวลาประมวลผลแก่ทุกคนอย่างรวดเร็ว
* **Translation lookaside buffer / TLB:** หน่วยความจำแคชความเร็วสูงที่ใช้เก็บรายการใน Page Table ที่เพิ่งถูกใช้งาน เพื่อลดเวลาในการแปลงที่อยู่หน่วยความจำ
* **Utility (โปรแกรมอรรถประโยชน์):** โปรแกรมระบบที่ช่วยทำงานพื้นฐานที่ใช้บ่อย เช่น การจัดการไฟล์ หรือตัว Editor
* **Virtual memory (หน่วยความจำเสมือน):** เทคนิคที่ใช้พื้นที่ในฮาร์ดดิสก์มาจำลองเป็นหน่วยความจำหลัก ทำให้โปรแกรมเมอร์สามารถเขียนโปรแกรมขนาดใหญ่กว่า RAM จริงได้

## 2. คำถามทบทวน (Review Questions)

คำถามพื้นฐานเพื่อเช็คความเข้าใจจากเนื้อหาบทที่ 8:

1. **(8.1)** ระบบปฏิบัติการ (Operating System) คืออะไร?
2. **(8.2)** จงระบุและให้คำจำกัดความสั้นๆ ถึงบริการหลักที่ OS จัดเตรียมไว้ให้
3. **(8.3)** จงระบุและให้คำจำกัดความสั้นๆ ถึงรูปแบบการจัดตารางเวลา (Scheduling) หลักของ OS
4. **(8.4)** ความแตกต่างระหว่าง "โปรเซส" (Process) และ "โปรแกรม" (Program) คืออะไร?
5. **(8.5)** จุดประสงค์ของการทำ Swapping คืออะไร?
6. **(8.6)** หากมีการจัดสรรโปรเซสไปยังตำแหน่งต่างๆ ในหน่วยความจำหลักได้แบบไดนามิก จะส่งผลกระทบต่อกลไกการอ้างอิงที่อยู่ (Addressing mechanism) อย่างไร?
7. **(8.7)** จำเป็นหรือไม่ที่ **ทุกเพจ** (Pages) ของโปรเซสจะต้องโหลดอยู่ในหน่วยความจำหลักขณะที่โปรเซสกำลังทำงาน?
8. **(8.8)** เพจของโปรเซสที่อยู่ในหน่วยความจำหลัก จำเป็นต้องวาง **ติดกัน** (Contiguous) หรือไม่?
9. **(8.9)** เพจของโปรเซสที่อยู่ในหน่วยความจำหลัก จำเป็นต้องวาง **เรียงตามลำดับ** (Sequential order) หรือไม่?
10. **(8.10)** จุดประสงค์ของ Translation Lookaside Buffer (TLB) คืออะไร?

## 3. สรุปแนวคิดของแบบฝึกหัด (Problems Overview)

แบบฝึกหัดท้ายบทที่ 8.1 - 8.20 เป็นการประยุกต์ทฤษฎี โดยเน้นประเด็นสำคัญ ดังนี้:

* **การประเมินประสิทธิภาพ CPU (CPU Performance & Scheduling):** การคำนวณหา Turnaround time, Throughput และอัตราการใช้งาน CPU ในสถานการณ์จำลองที่โปรแกรมมีการเรียกใช้งาน I/O สลับกับการใช้ประมวลผล 
* **การจัดการหน่วยความจำ (Memory Partitioning & Fragmentation):** การวิเคราะห์และพิสูจน์ปัญหาพื้นที่ว่างที่เสียไป (Fragmentation) ในการแบ่งหน่วยความจำแบบขนาดตายตัว (Fixed) และแบบไดนามิก (Dynamic)
* **กลไกของ Paging และ Virtual Memory:** * การคำนวณแปลงที่อยู่ (Logical เป็น Physical Address) จากตาราง Page Table
  * การติดตามลำดับของ Page Fault ที่เกิดขึ้นเมื่อใช้ Page Replacement Algorithm แบบต่างๆ (เช่น FIFO, LRU)
  * ผลกระทบของขนาด Page ที่มีต่อพื้นที่สูญเปล่า (Internal Fragmentation) และขนาดของตาราง Page Table
* **สถาปัตยกรรมเฉพาะเจาะจง:** การศึกษากลไก Memory Management ในระดับฮาร์ดแวร์ เช่น ระบบ Paging ควบคู่กับ Segmentation, แคช, และโครงสร้างของ ARM (รูปแบบ L1/L2 Page Table) และ IBM System/370
