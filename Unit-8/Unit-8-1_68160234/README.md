# สรุปบทที่ 8.1: ภาพรวมของระบบปฏิบัติการ (Operating System Overview)

ระบบปฏิบัติการ (Operating System - OS) คือโปรแกรมที่ทำหน้าที่จัดการทรัพยากรของคอมพิวเตอร์ ให้บริการแก่ผู้เขียนโปรแกรม และควบคุมการทำงานของโปรแกรมประยุกต์ต่างๆ

---

## 1. วัตถุประสงค์และฟังก์ชันของ OS (Objectives and Functions)
OS ถูกออกแบบมาเพื่อตอบโจทย์ 3 ด้านหลัก:
1.  **ความสะดวก (Convenience):** ทำให้คอมพิวเตอร์ใช้งานได้ง่ายขึ้น
2.  **ประสิทธิภาพ (Efficiency):** จัดสรรทรัพยากรให้ใช้งานได้อย่างคุ้มค่าที่สุด
3.  **ความสามารถในการพัฒนา (Ability to evolve):** ออกแบบมาให้รองรับการอัปเดตหรือเพิ่มฟังก์ชันใหม่ๆ โดยไม่กระทบการทำงานเดิม

### 1.1 การเป็นส่วนต่อประสาน (User/Computer Interface)
OS ช่วยลดความยุ่งยากในการสั่งงานฮาร์ดแวร์โดยตรงผ่านบริการต่างๆ:
* **การสร้างโปรแกรม (Program creation):** ให้บริการเครื่องมือจำพวก Debugger หรือ Editor (มักอยู่ในรูป Utility)
* **การรันโปรแกรม (Program execution):** จัดการโหลดโปรแกรมและข้อมูลลงหน่วยความจำ และเตรียมทรัพยากรที่จำเป็น
* **การเข้าถึงอุปกรณ์ I/O:** ช่วยให้ผู้ใช้เขียนคำสั่งอ่าน/เขียนแบบมาตรฐาน โดยไม่ต้องรู้รายละเอียดเชิงลึกของฮาร์ดแวร์แต่ละรุ่น
* **การเข้าถึงไฟล์ (File access control):** จัดการโครงสร้างไฟล์และระบบความปลอดภัยในการเข้าถึงข้อมูล
* **การเข้าถึงระบบ (System access):** ควบคุมการใช้งานทรัพยากรร่วมกันและป้องกันการบุกรุก
* **การตรวจจับและตอบสนองต่อข้อผิดพลาด:** จัดการปัญหาที่เกิดขึ้น (เช่น ฮาร์ดแวร์พัง, การหารด้วยศูนย์) อย่างเป็นระบบ
* **การทำบัญชี (Accounting):** เก็บสถิติการใช้งานเพื่อปรับปรุงประสิทธิภาพหรือเรียกเก็บค่าบริการ

### 1.2 การเป็นผู้จัดการทรัพยากร (Resource Manager)
OS มีหน้าที่ควบคุมการเคลื่อนย้าย, การจัดเก็บ และการประมวลผลข้อมูล โดยจัดการทรัพยากรหลักดังนี้:
* **Main Memory:** แบ่งพื้นที่สำหรับ Kernel (OS) และโปรแกรมของผู้ใช้
* **Processor (CPU):** กำหนดว่าโปรแกรมใดจะได้ใช้งาน CPU และนานเท่าไหร่
* **I/O Devices & Storage:** ควบคุมการรับส่งข้อมูลกับอุปกรณ์ภายนอก

## 2. ประเภทของระบบปฏิบัติการ (Types of Operating Systems)

ในการวิวัฒนาการของ OS สามารถแบ่งตามลักษณะการทำงานได้ดังนี้:

### 2.1 การประมวลผลแบบต่อเนื่อง (Serial Processing)
* **ยุคแรก (1940s-1950s):** ไม่มี OS ผู้ใช้ต้องจองเวลาใช้งานและตั้งค่าฮาร์ดแวร์เองทั้งหมด (ผ่าน Console)
* **ปัญหา:** การจัดคิวงานทำได้ยาก และเสียเวลาในการเตรียมระบบ (Setup time) นานเกินไป

### 2.2 ระบบประมวลผลแบบกลุ่ม (Simple Batch Systems)
* ใช้โปรแกรมที่เรียกว่า **Monitor** มาช่วยจัดการคิวงาน
* ผู้ใช้ส่งงานเป็นกลุ่ม (Batch) ให้โอเปอเรเตอร์ แล้ว Monitor จะโหลดงานถัดไปเข้าเครื่องอัตโนมัติ
* **Resident Monitor:** ส่วนของ OS ที่ต้องฝังตัวอยู่ในหน่วยความจำหลักตลอดเวลา

### 2.3 ระบบประมวลผลแบบกลุ่มหลายโปรแกรม (Multiprogrammed Batch Systems)
* ออกแบบมาเพื่อแก้ปัญหา CPU ว่างงาน (Idle) ขณะรออุปกรณ์ I/O
* **หลักการ:** โหลดหลายโปรแกรมลงในหน่วยความจำพร้อมกัน เมื่อโปรแกรมหนึ่งรอ I/O ระบบจะสลับไปรันโปรแกรมอื่นทันที
* ช่วยเพิ่ม **Throughput** (จำนวนงานที่ทำเสร็จต่อหน่วยเวลา) อย่างมาก

### 2.4 ระบบแบ่งเวลา (Time-Sharing Systems)
* พัฒนาเพื่อรองรับการใช้งานพร้อมกันหลายคน (Multi-user)
* **หลักการ:** CPU จะสลับการทำงานระหว่างผู้ใช้ด้วยความเร็วสูงมาก จนผู้ใช้แต่ละคนรู้สึกเหมือนได้ครองเครื่องคอมพิวเตอร์เพียงคนเดียว
* เน้นความเร็วในการตอบสนอง (**Response Time**) เป็นหลัก

## 3. สรุปความแตกต่างของคำศัพท์สำคัญ
| หัวข้อ | Multiprogramming | Time-Sharing |
| :--- | :--- | :--- |
| **เป้าหมายหลัก** | เพิ่มประสิทธิภาพการใช้ CPU | เพิ่มการโต้ตอบกับผู้ใช้ (Response time) |
| **เทคนิค** | สลับงานเมื่อมีการรอ I/O | สลับงานตามช่วงเวลา (Time slicing) |
