# ส่วนที่ 2: การตั้งค่า Git และ GitHub

## ทำความเข้าใจพื้นฐานก่อนเริ่มใช้งาน

| คำศัพท์ | ความหมาย |
| :--- | :--- |
| **Git** | ระบบควบคุมเวอร์ชันที่ติดตั้งในคอมพิวเตอร์ ใช้บันทึก ติดตาม และย้อนกลับประวัติการเปลี่ยนแปลงไฟล์ |
| **GitHub** | เว็บไซต์ที่เก็บข้อมูลและประวัติจาก Git บนอินเทอร์เน็ต ทำให้ทำงานร่วมกับผู้อื่นและสำรองข้อมูลได้ |

---

## ขั้นตอนที่ 1: ติดตั้ง Git

1. ไปที่ [git-scm.com/downloads](https://git-scm.com/downloads)
2. ดาวน์โหลดโปรแกรมตามระบบปฏิบัติการของท่าน (Windows / Mac / Linux)
3. ติดตั้งตามขั้นตอนที่โปรแกรมแนะนำ
4. เปิด **Command Prompt** (Windows) หรือ **Terminal** (Mac/Linux) เพื่อใช้งาน

---

## ขั้นตอนที่ 2: ตั้งค่าชื่อและอีเมล (ทำครั้งแรกในทุกเครื่อง)

**ขั้นตอนนี้สำคัญมาก** เพราะ Git จำเป็นต้องรู้ว่า "ใคร" เป็นผู้แก้ไขไฟล์

1. เปิด Command Prompt หรือ Terminal
2. พิมพ์คำสั่งต่อไปนี้ (เปลี่ยนชื่อและอีเมลเป็นของท่าน):

```bash
git config --global user.name "ชื่อ-นามสกุล หรือ Username"
git config --global user.email "อีเมลที่ใช้สมัคร_GitHub"
```

**ตัวอย่าง:**
```bash
git config --global user.name "สมชาย ใจดี"
git config --global user.email "somchai@email.com"
```

---

## ขั้นตอนที่ 3: ตรวจสอบค่าที่ตั้งไว้

พิมพ์คำสั่ง:

```bash
git config --global --list
```

ระบบจะแสดงค่าที่ตั้งไว้ทั้งหมด หากเห็น `user.name` และ `user.email` ตามที่กำหนด แสดงว่าตั้งค่าสำเร็จ

---

## ความแตกต่างระหว่าง Global และ Local Config

| ประเภท | ความหมาย | ใช้เมื่อใด |
| :--- | :--- | :--- |
| **Global** | ใช้กับทุก Repository ในเครื่องนี้ | การตั้งค่าปกติ สำหรับใช้บัญชีเดียว |
| **Local** | ใช้เฉพาะ Repository นั้น ๆ | ต้องการใช้ชื่อหรืออีเมลอื่นในโปรเจกต์บางอัน |

**หลักสำคัญ:** ค่า Local จะมีผลเหนือกว่า Global หากตั้งค่าทั้งสองที่

---

## คำสั่งพื้นฐานสำหรับจัดการ Repository

| การกระทำ | คำสั่ง |
| :--- | :--- |
| เริ่มโปรเจกต์ใหม่ในโฟลเดอร์ปัจจุบัน | `git init` |
| ดึงโปรเจกต์จาก GitHub ลงมา | `git clone https://github.com/username/repo-name.git` |

---

## การใช้งานกับหลายเครื่อง

ทุกเครื่องที่ใช้ Git กับ GitHub ต้อง:

1. ติดตั้ง Git
2. ตั้งค่า `user.name` และ `user.email` ด้วย `git config --global` ในเครื่องนั้น

**หมายเหตุ:** เมื่อ push หรือ pull ครั้งแรก ระบบจะถาม username และ password ของ GitHub (หรือ Personal Access Token หากเปิด Two-Factor Authentication)

---

## สรุป

- ใช้ Git ครั้งแรกในเครื่องใด ต้องตั้งค่าชื่อและอีเมลก่อนเสมอ
- แต่ละเครื่องต้องตั้งค่าของตน ไม่สามารถตั้งครั้งเดียวใช้ได้ทุกที่
- ใช้อีเมลเดียวกับที่สมัคร GitHub เพื่อให้เชื่อมโยงกับบัญชีได้ถูกต้อง
