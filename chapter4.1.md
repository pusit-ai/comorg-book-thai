# 4.1 Computer Memory System Overview

## สรุปภาพรวม
หน่วยความจำคอมพิวเตอร์มีหลายชนิด หลายเทคโนโลยี และมีความแตกต่างกันมากทั้งด้านโครงสร้าง ความเร็ว ความจุ และต้นทุน จึงไม่มีหน่วยความจำชนิดเดียวที่ตอบโจทย์ทุกอย่างได้พร้อมกัน ระบบคอมพิวเตอร์จริงจึงใช้ **memory hierarchy** หรือการจัดหน่วยความจำเป็นหลายระดับ เพื่อให้ได้สมดุลระหว่างความเร็ว ความจุ และราคา

## ลักษณะสำคัญของระบบหน่วยความจำ
หัวข้อหลักที่ใช้จำแนกหน่วยความจำ ได้แก่

- **Location**: อยู่ภายในระบบหรือภายนอกระบบ  
  - ภายใน: register, cache, main memory  
  - ภายนอก: disk, tape, optical storage

- **Capacity**: วัดเป็นจำนวนคำ (words) หรือจำนวนไบต์ (bytes)

- **Unit of Transfer**: ขนาดข้อมูลที่ย้ายในแต่ละครั้ง  
  - ภายในมักเป็น word หรือหลายบิตตาม bus width  
  - ภายนอกมักเป็น block

- **Access Method**  
  - Sequential: เข้าถึงตามลำดับ  
  - Direct: เข้าถึงบริเวณที่ต้องการแล้วค้นต่อ  
  - Random: เข้าถึงตำแหน่งใดก็ได้โดยตรง  
  - Associative: ค้นจากเนื้อหาของข้อมูล ไม่ใช่จาก address

- **Performance**  
  - Access time  
  - Memory cycle time  
  - Transfer rate

- **Physical Type**  
  - Semiconductor  
  - Magnetic  
  - Optical  
  - Magneto-optical

- **Physical Characteristics**  
  - Volatile / Nonvolatile  
  - Erasable / Nonerasable

- **Organization**  
  - การจัดเรียงบิตเป็นคำและเป็นโมดูลหน่วยความจำ

## ความหมายของตัวชี้วัดประสิทธิภาพ
- **Access time (latency)**: เวลาตั้งแต่ส่ง address จนได้ข้อมูลกลับมา หรือเขียนข้อมูลเสร็จ
- **Memory cycle time**: access time รวมกับเวลาพักที่ต้องใช้ก่อนเริ่มการเข้าถึงครั้งถัดไป
- **Transfer rate**: อัตราการย้ายข้อมูลเข้าออกหน่วยความจำ

## แนวคิด Memory Hierarchy
โจทย์ของการออกแบบหน่วยความจำมี 3 ข้อหลัก

1. ต้องการความจุเท่าไร
2. ต้องการความเร็วเท่าไร
3. ยอมรับต้นทุนได้แค่ไหน

โดยทั่วไปจะเกิด trade-off ดังนี้

- หน่วยความจำที่เร็วกว่า มักแพงกว่าต่อบิต
- หน่วยความจำที่จุมากกว่า มักถูกกว่าต่อบิต
- หน่วยความจำที่จุมากกว่า มักช้ากว่า

เพราะฉะนั้นระบบจึงใช้การจัดลำดับชั้น เช่น

- Registers
- Cache
- Main memory
- Magnetic disk
- Optical disk / Tape

เมื่อไล่ลงล่างในลำดับชั้น จะเกิดแนวโน้มดังนี้

- ราคาต่อบิตลดลง
- ความจุเพิ่มขึ้น
- เวลาเข้าถึงเพิ่มขึ้น
- ความถี่ที่ CPU เข้าถึงลดลง

## หลัก Locality of Reference
เหตุผลที่ memory hierarchy ใช้งานได้จริง คือโปรแกรมมักอ้างถึงข้อมูลเป็น “กลุ่ม” ในช่วงเวลาสั้น ๆ เช่น

- คำสั่งใน loop เดิมถูกเรียกซ้ำ
- subroutine เดิมถูกใช้งานซ้ำ
- ข้อมูลใน array หรือตารางมักถูกเข้าถึงติดกัน

แนวคิดนี้เรียกว่า **locality of reference** ทำให้ข้อมูลที่ถูกใช้งานบ่อยในช่วงเวลานั้นถูกเก็บไว้ในหน่วยความจำระดับบนที่เร็วกว่าได้

## ใจความสำคัญของหัวข้อนี้
- ไม่มีหน่วยความจำแบบเดียวที่เร็ว ใหญ่ และถูกพร้อมกัน
- จึงต้องใช้หลายระดับร่วมกัน
- ระดับบนเร็วแต่เล็กและแพง
- ระดับล่างช้าแต่ใหญ่และถูก
- ประสิทธิภาพของระบบดีขึ้นได้เพราะโปรแกรมมี locality ทำให้ข้อมูลที่ใช้งานบ่อยมักอยู่ใกล้และถูกเรียกซ้ำ
