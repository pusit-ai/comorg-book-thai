## เป้าหมาย

จัดทำหนังสือวิชาสถาปัตยกรรมคอมพิวเตอร์ฉบับภาษาไทย โดยใช้ระบบควบคุมเวอร์ชัน (Version Control) เพื่อให้สมาชิกในกลุ่มสามารถทำงานร่วมกันได้อย่างมีประสิทธิภาพ

---

## ขั้นตอนและเครื่องมือ

| ลำดับ | ขั้นตอน | รายละเอียด |
| :---: | :--- | :--- |
| 1 | **การแบ่งงาน** | แบ่งเนื้อหาจากไฟล์ต้นฉบับ PDF (เช่น บทที่ 4–8) ให้สมาชิกแต่ละคนรับผิดชอบตามหัวข้อย่อย |
| 2 | **การแปลงไฟล์เอกสาร** | ใช้เครื่องมือ `pdftotext` (Xpdf) เพื่อดึงข้อความออกจากเอกสาร PDF |
| 3 | **การแปลและเรียบเรียง** | นำข้อความที่สกัดได้มาแปลและเรียบเรียงเป็นภาษาทางการ โดยจัดรูปแบบด้วย Markdown (นามสกุล `.md`) |
| 4 | **การจัดการผ่าน Git และ GitHub** | ทำงานร่วมกันผ่าน Repository กลางบนอินเทอร์เน็ต ชื่อ `comorg-book-thai` |

---

## สิ่งที่สมาชิกทุกคนจะต้องทำ

- ใช้ Repository หลักร่วมกันชื่อ `comorg-book-thai`
- สมาชิกแต่ละคนสร้าง Branch แยกเพื่อทำงานในส่วนของตนเอง
- เมื่อเขียนและเรียบเรียงด้วย Markdown เสร็จ ให้ Merge กลับเข้าสู่ Branch หลัก (main)

---

## การจัดกลุ่มเมื่อมีหลายกลุ่ม

ใช้ Repository หลักร่วมกันชื่อ `comorg-book-thai` หากมีหลายกลุ่มแยกกันทำตามบท (เช่น กลุ่ม 1 รับผิดชอบบทที่ 1 กลุ่ม 2 รับผิดชอบบทที่ 2) จะไม่มีปัญหาเรื่องชื่อ Repository เพราะกลุ่มต่างๆ ทำงานในโฟลเดอร์บทของตนเอง (`chapter-1/`, `chapter-2/`, ฯลฯ) ภายใน Repository เดียวกัน

---

## คำสั่งเริ่มต้นสำหรับหัวหน้ากลุ่ม

```bash
mkdir comorg-book-thai
cd comorg-book-thai
git init
```
