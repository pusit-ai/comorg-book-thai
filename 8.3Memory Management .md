# 📑 สรุปบทที่ 8.3: การจัดการหน่วยความจำ (Memory Management)

ในระบบ **Multiprogramming** ระบบปฏิบัติการ (OS) มีหน้าที่จัดสรรหน่วยความจำหลักเพื่อให้สามารถรันหลายโปรเซสพร้อมกันได้อย่างมีประสิทธิภาพ โดยมีกลไกสำคัญดังนี้:

---

## 1. 🔄 Swapping (การสลับหน่วยความจำ)
ใช้แก้ปัญหาเมื่อ CPU ทำงานเร็วกว่าการรอ I/O มาก จนโปรเซสทั้งหมดในหน่วยความจำตกอยู่ในสถานะ "รอ" ทำให้ CPU ว่างงาน
* **กลไก:** OS จะย้ายโปรเซสที่รอ I/O ออกไปไว้ในดิสก์ (**Intermediate Queue**) และดึงโปรเซสใหม่จาก **Long-term Queue** เข้ามาทำงานแทน
* **ผลลัพธ์:** ช่วยให้ CPU มีงานทำตลอดเวลา (Maximize CPU Utilization)



---

## 2. 🗄️ Partitioning (การแบ่งส่วนหน่วยความจำ)

### Fixed Partitioning (การแบ่งขนาดตายตัว)
* แบ่งหน่วยความจำออกเป็นช่องขนาดคงที่ (ขนาดอาจเท่ากันหรือไม่ก็ได้)
* **ข้อเสีย:** เกิด **Internal Fragmentation** (พื้นที่เหลือทิ้งภายในช่อง เนื่องจากโปรเซสใช้ไม่เต็มพื้นที่ที่จองไว้)

### Dynamic Partitioning (การแบ่งขนาดตามจริง)
* แบ่งพื้นที่ตามขนาดที่แต่ละโปรเซสต้องการจริงในขณะที่โหลดเข้าเครื่อง
* **ข้อเสีย:** เมื่อเวลาผ่านไปจะเกิด **External Fragmentation** (เกิดช่องว่างขนาดเล็กกระจายอยู่ทั่วไปจนไม่สามารถลงโปรเซสใหม่ได้)
* **วิธีแก้:** การทำ **Compaction** (การบีบอัดข้อมูล) โดยการเลื่อนโปรเซสทั้งหมดมาติดกันเพื่อให้ได้พื้นที่ว่างก้อนใหญ่ก้อนเดียว



---

## 3. 📄 Paging (การแบ่งหน้า)
เทคนิคที่ช่วยให้โปรเซสไม่ต้องวางตัวติดกันในหน่วยความจำ (Non-contiguous Allocation)
* **Pages:** โปรเซสถูกแบ่งเป็นส่วนย่อยขนาดเท่ากัน
* **Frames:** หน่วยความจำหลัก (RAM) ถูกแบ่งเป็นช่องขนาดเท่ากับ Page
* **Page Table:** ตารางที่ OS ใช้จับคู่ว่า Page ของโปรเซสอยู่ที่ Frame ไหนในแรม ช่วยในการแปลงที่อยู่จาก Logical เป็น Physical



---

## 4. 🧠 Virtual Memory (หน่วยความจำเสมือน)
แนวคิดที่อนุญาตให้รันโปรเซสที่มีขนาด **ใหญ่กว่าหน่วยความจำจริง (RAM)** ได้
* **Demand Paging:** โหลดเฉพาะ Page ที่ต้องใช้งานจริงๆ เข้ามาในแรม
* **Page Fault:** เกิดขึ้นเมื่อ CPU เรียกหา Page ที่ไม่อยู่ในแรม ทำให้ OS ต้องไปดึงหน้านั้นมาจากดิสก์
* **Thrashing:** สภาวะที่ระบบใช้เวลาส่วนใหญ่ไปกับการสลับ Page เข้า-ออก จนแทบไม่ได้ประมวลผลงานจริง (เกิดจากแรมไม่พอ)
* **TLB (Translation Lookaside Buffer):** แคชพิเศษที่เก็บสำเนา Page Table ล่าสุด เพื่อให้การแปลงที่อยู่ทำได้รวดเร็วขึ้น



---

## 🧩 5. Segmentation (การแบ่งเซกเมนต์)
ต่างจาก Paging ตรงที่ผู้เขียนโปรแกรมมองเห็นได้ (Visible to Programmer)
* แบ่งหน่วยความจำตามโครงสร้างของโปรแกรม (เช่น ส่วน Code, Data, Stack)
* **ข้อดี:** ง่ายต่อการแชร์ข้อมูล (Sharing), การป้องกันข้อมูล (Protection) และการขยายขนาดโครงสร้างข้อมูลที่ทำได้ยืดหยุ่นกว่า

---

## 🔑 สรุปคำศัพท์สำคัญ (Key Terms)

| คำศัพท์ | ความหมาย |
| :--- | :--- |
| **Logical Address** | ที่อยู่ที่ถูกอ้างอิงภายในโปรแกรม (Relative Address) |
| **Physical Address** | ที่อยู่จริงในหน่วยความจำหลัก (RAM) |
| **Internal Fragmentation** | พื้นที่ว่างที่เสียเปล่าภายใน Partition เนื่องจากโปรเซสเล็กกว่าช่องที่จอง |
| **External Fragmentation** | ช่องว่างระหว่างโปรเซสที่กระจัดกระจายจนรวมกันเป็นก้อนใหญ่ไม่ได้ |
| **Page Fault** | เหตุการณ์ที่ CPU เรียกใช้ข้อมูลที่ยังไม่ได้ถูกโหลดเข้า RAM |
