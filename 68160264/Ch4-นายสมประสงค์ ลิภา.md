# บทที่ 4 หน่วยความจำแคช (Cache Memory)

## 4.1 ภาพรวมของระบบหน่วยความจำคอมพิวเตอร์ (Computer Memory System Overview)

ในระบบคอมพิวเตอร์ หน่วยความจำถือเป็นองค์ประกอบสำคัญที่ส่งผลโดยตรงต่อประสิทธิภาพของระบบ การออกแบบระบบหน่วยความจำจึงมักใช้แนวคิด **ลำดับชั้นของหน่วยความจำ (Memory Hierarchy)** เพื่อสร้างสมดุลระหว่างความเร็ว ความจุ และต้นทุน

โดยทั่วไป หน่วยความจำสามารถแบ่งตามลำดับความเร็วได้ดังนี้

1. Registers  
2. Cache Memory  
3. Main Memory (RAM)  
4. Secondary Storage เช่น SSD หรือ Hard Disk

หน่วยความจำที่อยู่ใกล้กับหน่วยประมวลผลกลาง (CPU) จะมีความเร็วสูงกว่าแต่มีขนาดเล็ก ในขณะที่หน่วยความจำที่อยู่ลำดับล่างจะมีความจุสูงกว่าแต่มีความเร็วต่ำกว่า

แนวคิดสำคัญที่ทำให้ระบบหน่วยความจำแบบลำดับชั้นทำงานได้อย่างมีประสิทธิภาพ คือ **หลักการของ Locality**

### Temporal Locality

หากข้อมูลหนึ่งถูกใช้งานในช่วงเวลาหนึ่ง มีแนวโน้มสูงว่าข้อมูลนั้นจะถูกใช้งานอีกครั้งในอนาคตอันใกล้

### Spatial Locality

หากตำแหน่งหนึ่งของหน่วยความจำถูกเข้าถึง มีแนวโน้มว่าตำแหน่งใกล้เคียงจะถูกเข้าถึงในเวลาใกล้กัน

หลักการทั้งสองนี้เป็นพื้นฐานสำคัญของการออกแบบ **Cache Memory**

---

# 4.2 หลักการทำงานของ Cache Memory

**Cache Memory** เป็นหน่วยความจำความเร็วสูงที่อยู่ระหว่าง CPU และ Main Memory ทำหน้าที่เก็บข้อมูลที่ถูกใช้งานบ่อย เพื่อให้ CPU สามารถเข้าถึงข้อมูลได้รวดเร็วขึ้น

กระบวนการทำงานมีลำดับดังนี้

1. CPU ส่งคำสั่งเพื่ออ่านข้อมูล
2. ระบบตรวจสอบข้อมูลใน Cache ก่อน
3. หากพบข้อมูล จะเรียกว่า **Cache Hit**
4. หากไม่พบข้อมูล จะเรียกว่า **Cache Miss**
5. ระบบจะดึงข้อมูลจาก Main Memory มาเก็บใน Cache แล้วส่งให้ CPU

การใช้ Cache Memory ช่วยลดเวลาการเข้าถึงหน่วยความจำ และเพิ่มประสิทธิภาพโดยรวมของระบบคอมพิวเตอร์

---

# 4.3 องค์ประกอบของการออกแบบ Cache (Elements of Cache Design)

การออกแบบ Cache Memory จำเป็นต้องพิจารณาปัจจัยสำคัญหลายประการ ได้แก่

## 1. ขนาดของ Cache (Cache Size)

Cache ที่มีขนาดใหญ่สามารถเก็บข้อมูลได้มากขึ้น ทำให้มีโอกาสเกิด Cache Hit สูงขึ้น อย่างไรก็ตาม Cache ที่มีขนาดใหญ่ย่อมมีต้นทุนสูงและอาจเพิ่มเวลาในการเข้าถึงข้อมูล

ดังนั้นการออกแบบจึงต้องคำนึงถึงความสมดุลระหว่าง **ขนาด ความเร็ว และต้นทุน**

---

## 2. วิธีการจัดวางข้อมูล (Mapping Function)

Mapping Function เป็นกลไกที่กำหนดว่าข้อมูลจาก Main Memory จะถูกจัดเก็บในตำแหน่งใดของ Cache

มีรูปแบบหลักดังนี้

### Direct Mapping

ในวิธีนี้ บล็อกข้อมูลจาก Main Memory สามารถถูกเก็บได้เพียงตำแหน่งเดียวใน Cache

ข้อดี

- โครงสร้างง่าย
- ใช้ฮาร์ดแวร์น้อย
- ความเร็วสูง

ข้อเสีย

- มีโอกาสเกิด Cache Miss บ่อย

---

### Fully Associative Mapping

บล็อกข้อมูลจาก Main Memory สามารถถูกเก็บในตำแหน่งใดก็ได้ใน Cache

ข้อดี

- ลดปัญหา Cache Miss

ข้อเสีย

- ต้องใช้ฮาร์ดแวร์ที่ซับซ้อน
- การค้นหาข้อมูลใช้เวลามากขึ้น

---

### Set Associative Mapping

เป็นการผสมระหว่าง Direct Mapping และ Fully Associative

Cache จะถูกแบ่งออกเป็นหลาย **Set** และแต่ละบล็อกข้อมูลสามารถถูกเก็บในตำแหน่งใดก็ได้ภายใน Set ที่กำหนด

วิธีนี้เป็นรูปแบบที่นิยมใช้ในโปรเซสเซอร์สมัยใหม่

---

# 4.4 อัลกอริทึมการแทนที่ข้อมูล (Replacement Algorithms)

เมื่อ Cache เต็มและต้องนำข้อมูลใหม่เข้ามา ระบบต้องเลือกข้อมูลบางส่วนออกจาก Cache เพื่อทำพื้นที่ว่าง

อัลกอริทึมที่ใช้กันทั่วไป ได้แก่

### LRU (Least Recently Used)

เลือกนำข้อมูลที่ไม่ได้ถูกใช้งานมานานที่สุดออกจาก Cache

ข้อดี

- มีประสิทธิภาพสูงในหลายกรณี

ข้อเสีย

- ต้องใช้โครงสร้างข้อมูลเพิ่มเติมเพื่อบันทึกการใช้งาน

---

### FIFO (First In First Out)

ข้อมูลที่ถูกนำเข้ามาใน Cache ก่อน จะถูกนำออกก่อน

ข้อดี

- โครงสร้างง่าย

ข้อเสีย

- อาจไม่เหมาะกับรูปแบบการใช้งานบางประเภท

---

### Random Replacement

เลือกข้อมูลใน Cache แบบสุ่มเพื่อนำออก

ข้อดี

- ใช้งานง่าย
- ไม่ต้องติดตามประวัติการใช้งาน

ข้อเสีย

- ประสิทธิภาพไม่แน่นอน

---

# 4.5 นโยบายการเขียนข้อมูล (Write Policies)

เมื่อ CPU ทำการเขียนข้อมูลลง Cache จำเป็นต้องกำหนดวิธีการจัดการกับ Main Memory

## Write Through

ข้อมูลจะถูกเขียนลงทั้ง Cache และ Main Memory พร้อมกัน

ข้อดี

- ข้อมูลมีความสอดคล้องกันตลอดเวลา

ข้อเสีย

- เพิ่มจำนวนการเข้าถึง Main Memory

---

## Write Back

ข้อมูลจะถูกเขียนเฉพาะใน Cache ก่อน และจะถูกเขียนกลับไปยัง Main Memory เมื่อบล็อกข้อมูลถูกแทนที่

ข้อดี

- ลดจำนวนการเขียนไปยัง Main Memory

ข้อเสีย

- ระบบควบคุมซับซ้อนกว่า

---

# 4.6 Cache ในระบบคอมพิวเตอร์สมัยใหม่

โปรเซสเซอร์สมัยใหม่มักมี Cache หลายระดับ เพื่อเพิ่มประสิทธิภาพในการเข้าถึงข้อมูล

### L1 Cache

- อยู่ใกล้ CPU มากที่สุด
- ความเร็วสูงมาก
- ขนาดเล็ก

### L2 Cache

- มีขนาดใหญ่กว่า L1
- ความเร็วรองลงมา

### L3 Cache

- ใช้ร่วมกันระหว่างหลาย Core
- ช่วยลดการเข้าถึง Main Memory

โครงสร้าง Cache หลายระดับช่วยเพิ่มประสิทธิภาพของระบบคอมพิวเตอร์ได้อย่างมาก

---

# สรุป

Cache Memory เป็นองค์ประกอบสำคัญของสถาปัตยกรรมคอมพิวเตอร์สมัยใหม่ เนื่องจากช่วยลดเวลาในการเข้าถึงข้อมูลของ CPU และเพิ่มประสิทธิภาพของระบบโดยรวม

การออกแบบ Cache จำเป็นต้องพิจารณาปัจจัยหลายด้าน เช่น

- ขนาดของ Cache
- วิธีการ Mapping
- Replacement Algorithm
- Write Policy

การผสมผสานองค์ประกอบเหล่านี้อย่างเหมาะสมจะช่วยให้ระบบคอมพิวเตอร์สามารถทำงานได้อย่างมีประสิทธิภาพสูง
