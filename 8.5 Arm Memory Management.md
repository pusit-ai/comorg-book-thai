สรุปเนื้อหา หัวข้อ 8.5: Arm Memory Management แบบเข้าใจง่าย 👇

🔹 แนวคิดหลัก

การจัดการหน่วยความจำของ ARM เน้นเรื่อง

ประสิทธิภาพ (performance)
ความปลอดภัย (security)
การรองรับหลายโปรแกรม (multitasking)

โดยใช้หน่วยที่เรียกว่า MMU (Memory Management Unit)

🔹 หน้าที่ของ MMU

MMU ในระบบ ARM ทำหน้าที่สำคัญ เช่น

แปลง Virtual Address → Physical Address
ป้องกันการเข้าถึงหน่วยความจำที่ไม่ได้รับอนุญาต
แบ่งพื้นที่หน่วยความจำให้แต่ละโปรแกรมแยกจากกัน
🔹 การแบ่งหน่วยความจำ (Memory Regions)

ARM ใช้วิธีแบ่งหน่วยความจำเป็นส่วน ๆ เช่น

Code (คำสั่งโปรแกรม)
Data (ข้อมูล)
Stack / Heap

แต่ละส่วนสามารถกำหนดสิทธิ์ได้ เช่น

อ่านอย่างเดียว (Read-only)
อ่าน/เขียน (Read/Write)
🔹 Page Table

ARM ใช้ Page Table เพื่อเก็บข้อมูลว่า

Virtual Address แต่ละตัวไปอยู่ที่ Physical Address ไหน
มีสิทธิ์ใช้งานแบบใด
🔹 Translation Lookaside Buffer (TLB)
เป็น cache ขนาดเล็กใน MMU
ช่วยให้การแปลง address เร็วขึ้น
ลดเวลาในการค้นหา Page Table
🔹 การป้องกันหน่วยความจำ

ARM รองรับระบบป้องกัน เช่น

จำกัดสิทธิ์ของแต่ละโปรแกรม
ป้องกันโปรแกรมหนึ่งไปแก้ข้อมูลของอีกโปรแกรม
เพิ่มความปลอดภัยของระบบปฏิบัติการ
🔹 จุดเด่นของ ARM Memory Management
ประหยัดพลังงาน (เหมาะกับมือถือ/embedded)
ยืดหยุ่นสูง
รองรับระบบสมัยใหม่ เช่น virtual memory และ multitasking
#✅ สรุปสั้นที่สุด

ARM ใช้ MMU + Page Table + TLB เพื่อ
👉 แปลงที่อยู่หน่วยความจำ
👉 ป้องกันข้อมูล
👉 เพิ่มความเร็วและประสิทธิภาพของระบบ
