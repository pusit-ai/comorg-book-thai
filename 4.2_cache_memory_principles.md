# 4.2 Cache Memory Principles

## แนวคิดหลัก
Cache memory ถูกออกแบบมาเพื่อรวมข้อดีของหน่วยความจำ 2 แบบเข้าด้วยกัน คือ

- ความเร็วของหน่วยความจำราคาแพงและเร็ว
- ความจุของหน่วยความจำราคาถูกกว่าแต่ช้ากว่า

โดย cache เป็นหน่วยความจำขนาดเล็กและเร็ว ที่เก็บ **สำเนาบางส่วนของ main memory** ไว้ใกล้ CPU

## การทำงานพื้นฐาน
เมื่อ CPU ต้องการอ่านข้อมูลจากหน่วยความจำ

1. ตรวจสอบก่อนว่าข้อมูลนั้นอยู่ใน cache หรือไม่
2. ถ้าอยู่ เรียกว่า **cache hit**  
   - ส่งข้อมูลให้ CPU ได้ทันที
3. ถ้าไม่อยู่ เรียกว่า **cache miss**  
   - ต้องไปอ่าน block ที่มีข้อมูลนั้นจาก main memory
   - โหลด block นั้นเข้ามาใน cache
   - แล้วจึงส่งคำที่ต้องการให้ CPU

## เหตุผลที่ cache ช่วยเพิ่มประสิทธิภาพ
ถึงแม้ CPU ขอเพียงคำเดียว แต่ระบบจะดึงมาเป็นทั้ง **block** เพราะมีโอกาสสูงที่ CPU จะเรียกใช้

- ข้อมูลตำแหน่งเดิมอีก
- ข้อมูลใกล้เคียงกันใน block เดียวกัน

นี่เป็นผลจาก locality of reference

## โครงสร้างพื้นฐานของ Cache/Main Memory
- Main memory แบ่งเป็นหลาย **blocks**
- Cache แบ่งเป็นหลาย **lines**
- แต่ละ line เก็บ
  - ข้อมูลจำนวน K words
  - **tag** เพื่อบอกว่าข้อมูลนี้มาจาก block ไหนใน main memory
  - control bits เช่น dirty bit หรือ valid bit

เพราะจำนวน block ใน main memory มีมากกว่าจำนวน line ใน cache มาก จึงต้องมีวิธีระบุว่าขณะนั้น line แต่ละเส้นกำลังเก็บ block ไหนอยู่

## ขั้นตอนการอ่านข้อมูล
เมื่อ CPU ส่ง read address เข้ามา

- ถ้า block ที่มี address นั้นอยู่ใน cache  
  - อ่านจาก cache ทันที
- ถ้าไม่อยู่  
  - ดึง block จาก main memory มาใส่ใน cache line ที่กำหนด
  - จากนั้นจึงส่งข้อมูลให้ CPU

## หลายระดับของ Cache
ระบบสมัยใหม่มักมี cache มากกว่า 1 ระดับ เช่น

- **L1**: เล็กที่สุด เร็วที่สุด ใกล้ CPU มากที่สุด
- **L2**: ใหญ่ขึ้น ช้ากว่า L1
- **L3**: ใหญ่ขึ้นอีก และช้ากว่า L2

แนวทางนี้ช่วยลดเวลาเฉลี่ยในการเข้าถึงหน่วยความจำได้มาก เพราะ CPU มักเจอข้อมูลที่ระดับบนก่อน

## ใจความสำคัญของหัวข้อนี้
- Cache คือหน่วยความจำขนาดเล็กและเร็วที่เก็บสำเนาบางส่วนของ main memory
- จุดประสงค์คือ ลดเวลาเฉลี่ยในการเข้าถึงข้อมูล
- ใช้หลัก locality เพื่อให้ block ที่เพิ่งถูกเรียกมีโอกาสถูกใช้อีก
- ระบบปัจจุบันมักใช้ cache หลายระดับเพื่อเพิ่มประสิทธิภาพรวมของระบบ
