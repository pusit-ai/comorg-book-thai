Intel x86 Memory Management (304) — การจัดการหน่วยความจำ
การจัดการหน่วยความจำ (Memory Management) ในสถาปัตยกรรม Intel x86 เป็นกลไกสำคัญที่ทำให้ระบบปฏิบัติการสามารถควบคุมการเข้าถึงหน่วยความจำของโปรแกรมต่าง ๆ ได้อย่างมีประสิทธิภาพและปลอดภัย โดยเฉพาะในโหมด 32 บิต (x86-32 หรือ IA-32) ซึ่งรองรับพื้นที่แอดเดรสสูงสุด 4 GB

1. แนวคิดพื้นฐาน

หน่วยความจำในระบบ x86 ไม่ได้ถูกใช้งานแบบตรง ๆ (Physical Memory) แต่จะผ่านการแปลงที่อยู่ (Address Translation) จาก

Logical Address (ที่โปรแกรมใช้งาน)
→ แปลงเป็น
Linear Address
→ แล้วจึงแปลงเป็น
Physical Address (ที่อยู่จริงใน RAM)

กระบวนการนี้ช่วยให้สามารถแยกพื้นที่หน่วยความจำของแต่ละโปรแกรมออกจากกัน ป้องกันการเข้าถึงข้อมูลโดยไม่ได้รับอนุญาต

2. Segmentation (การแบ่งส่วนหน่วยความจำ)

Segmentation เป็นวิธีการจัดการหน่วยความจำแบบดั้งเดิมของ x86 โดยแบ่งหน่วยความจำออกเป็นส่วน ๆ (Segment) เช่น

Code Segment (CS)
Data Segment (DS)
Stack Segment (SS)

แต่ละ Segment จะมี:

Base Address (ตำแหน่งเริ่มต้น)
Limit (ขนาดของ Segment)

เมื่อโปรแกรมเรียกใช้งานหน่วยความจำ จะใช้ค่า : Segment Selector + Offset

CPU จะนำไปคำนวณเป็น Linear Address
📌 ข้อดี:

-แบ่งหน่วยความจำเป็นส่วนตามหน้าที่

📌 ข้อเสีย:

-จัดการยาก และไม่ยืดหยุ่น

3. Paging (การแบ่งหน้า)

Paging เป็นวิธีหลักที่ใช้ในระบบสมัยใหม่ โดยแบ่งหน่วยความจำออกเป็นหน่วยขนาดคงที่ เรียกว่า Page (ปกติ 4 KB)

Virtual Memory → แบ่งเป็น Page
Physical Memory → แบ่งเป็น Frame

การแปลงที่อยู่จะใช้โครงสร้างที่เรียกว่า Page Table

โครงสร้าง Paging (2-Level Paging)
Page Directory
Page Table
Offset

รูปแบบของ Linear Address : | Page Directory | Page Table | Offset |
📌 ข้อดี:

-ใช้งานง่าย
-รองรับ Virtual Memory
-ป้องกันโปรแกรมชนกัน

4. Virtual Memory (หน่วยความจำเสมือน)

Virtual Memory คือการทำให้โปรแกรมมองเห็นหน่วยความจำมากกว่าที่มีจริง โดยใช้พื้นที่ในฮาร์ดดิสก์ (เช่น swap หรือ page file) มาช่วย

📌 ประโยชน์:

-รันโปรแกรมใหญ่ได้
-แยกพื้นที่แต่ละโปรแกรม
-เพิ่มความปลอดภัยของระบบ

5. Protection (การป้องกันหน่วยความจำ)

ระบบ x86 มีการกำหนดสิทธิ์การเข้าถึงหน่วยความจำ เช่น:

-Read (อ่าน)
-Write (เขียน)
-Execute (รันคำสั่ง)

รวมถึงระดับสิทธิ์ (Privilege Levels หรือ Rings):

-Ring 0 → Kernel (สิทธิ์สูงสุด)
-Ring 3 → User Program

📌 ช่วยป้องกัน:

-โปรแกรมเข้าถึง kernel
-โปรแกรมชนกัน

6. สรุป

การจัดการหน่วยความจำใน Intel x86 เป็นระบบที่ผสมผสานระหว่าง Segmentation และ Paging โดยในระบบปัจจุบันนิยมใช้ Paging เป็นหลัก เพื่อให้เกิดความยืดหยุ่น ปลอดภัย และรองรับ Virtual Memory ได้อย่างมีประสิทธิภาพ
