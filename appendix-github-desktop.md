# ภาคผนวก: คู่มือ GitHub Desktop สำหรับนิสิตปี 1 (Step-by-Step)

[← กลับไปสารบัญหลัก](README.md) | [ภาคผนวก: แก้ปัญหาที่พบบ่อย ←](appendix-troubleshooting.md)

---

**GitHub Desktop** คือโปรแกรม GUI (Graphic User Interface) ที่ช่วยให้ใช้งาน Git และ GitHub ได้โดยไม่ต้องพิมพ์คำสั่งใน Terminal เหมาะสำหรับนิสิตที่ต้องการความสะดวกและเห็นภาพการทำงานที่ชัดเจน

---

## 1. การติดตั้งและตั้งค่าครั้งแรก

1.  **ดาวน์โหลด:** เข้าไปที่ [desktop.github.com](https://desktop.github.com/) แล้วติดตั้งให้เรียบร้อย
2.  **Login:** เปิดโปรแกรมไปที่ **File > Options** (Windows) หรือ **GitHub Desktop > Settings** (macOS)
3.  เลือกแท็บ **Accounts** แล้วกด **Sign in** ด้วยบัญชี GitHub ของท่าน

---

## 2. การดึงโปรเจกต์ลงเครื่อง (Clone)

หากหัวหน้ากลุ่มแจ้ง URL ของ Repository แล้ว (เช่น `https://github.com/หัวหน้า/comorg-book-thai`) ให้ทำดังนี้:

1.  ใน GitHub Desktop กดปุ่ม **File > Clone Repository...**
2.  เลือกแท็บ **URL** แล้ววางลิงก์ที่หัวหน้าส่งมา
3.  เลือก **Local Path** (โฟลเดอร์ในเครื่องที่ต้องการเก็บงาน) แล้วกด **Clone**

---

## 3. วงจรการทำงานปกติ (Workflow สำหรับลูกทีม)

เมื่อต้องการเริ่มทำงานแต่ละครั้ง ให้ทำตามลำดับดังนี้:

### ขั้นที่ 1: ดึงงานล่าสุด (Pull)
ก่อนเริ่มพิมพ์งานทุกครั้ง ต้องมั่นใจว่าเรามีงานล่าสุดของเพื่อน:
1.  กดปุ่ม **Fetch origin** ที่แถบด้านบน
2.  หากมีปุ่ม **Pull origin** ปรากฏขึ้นมา ให้กดเพื่อดึงงานล่าสุดลงเครื่อง

### ขั้นที่ 2: สร้างสาขาใหม่ (Branch)
**สำคัญมาก:** ห้ามทำงานบนสาขา `main` โดยตรง ให้สร้างสาขาของหัวข้อที่ตัวเองรับผิดชอบ:
1.  คลิกที่ปุ่ม **Current Branch** (ปกติจะเป็น `main`)
2.  กดปุ่ม **New Branch**
3.  ตั้งชื่อตามหัวข้อ เช่น `section-4.1` แล้วกด **Create Branch**
4.  หากมีปุ่ม **Publish branch** ให้กดเพื่อแจ้ง GitHub ว่าเราสร้างสาขานี้แล้ว

### ขั้นที่ 3: แก้ไขไฟล์งาน
1.  เปิดโฟลเดอร์งาน (กดปุ่ม **Open in Visual Studio Code** ในโปรแกรมได้เลย)
2.  แก้ไขไฟล์หรือเพิ่มไฟล์ตามที่ได้รับมอบหมาย (เช่น `chapter-4/4-1.md`)
3.  บันทึกไฟล์ (Save) ให้เรียบร้อย

### ขั้นที่ 4: บันทึกและส่งงาน (Commit & Push)
1.  กลับมาที่ GitHub Desktop จะเห็นรายการไฟล์ที่เปลี่ยนไปในแถบด้านซ้าย
2.  ที่ช่อง **Summary** (ด้านล่างซ้าย) ให้พิมพ์อธิบายสั้นๆ เช่น `เพิ่มเนื้อหา 4.1`
3.  กดปุ่ม **Commit to [ชื่อสาขา]**
4.  กดปุ่ม **Push origin** เพื่อส่งงานขึ้นเว็บ

---

## 4. การขอรวมงาน (Pull Request)

เมื่อ Push งานขึ้นไปแล้ว ให้ทำต่อบนหน้าเว็บ GitHub:

1.  ในโปรแกรม GitHub Desktop จะมีปุ่มสีฟ้าขึ้นว่า **Create Pull Request** ให้กดปุ่มนั้น
2.  ระบบจะเปิดเบราว์เซอร์พาไปหน้า GitHub
3.  ตรวจสอบว่างานของเราถูกต้อง แล้วกด **Create pull request**
4.  แจ้งหัวหน้ากลุ่มให้มาตรวจและ Merge งานเข้าสู่สาขาหลัก

---

## 5. การสลับกลับไปทำงานหัวข้อใหม่

เมื่อหัวหน้า Merge งานเก่าเสร็จแล้ว และเราต้องการเริ่มหัวข้อใหม่:
1.  ใน GitHub Desktop คลิกที่ **Current Branch** แล้วเลือกกลับไปที่ `main`
2.  กด **Fetch origin** และ **Pull origin** เพื่อรับงานล่าสุด (รวมถึงงานที่เราเพิ่งส่งไป)
3.  เริ่มทำตาม [ขั้นตอนที่ 2 (สร้างสาขาใหม่)](#ขั้นที่-2-สร้างสาขาใหม่-branch) อีกครั้ง

---

### สรุปปุ่มที่ต้องใช้บ่อย
- **Fetch origin:** เช็คว่ามีงานใหม่จากเพื่อนไหม
- **Pull origin:** ดึงงานใหม่ลงเครื่อง
- **Commit:** บันทึกงานในเครื่องเราเอง
- **Push:** ส่งงานที่บันทึกแล้วขึ้น GitHub
- **Current Branch:** สลับหรือสร้างพื้นที่ทำงานใหม่
